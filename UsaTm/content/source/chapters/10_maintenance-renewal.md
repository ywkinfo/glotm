# 등록 후 유지관리와 갱신 체계

등록 후 운영의 핵심은 "달력을 지키는 것"과 "실제 사용 중인 포트폴리오만 유지하는 것"이다. 많은 기업이 등록증을 받은 뒤 안심하지만, 미국에서는 유지 제출과 audit 리스크 때문에 오히려 등록 이후 관리 품질이 더 중요해진다.

## 유지관리 달력 설계

유지 제출 시점은 개별 사건의 문제가 아니라 포트폴리오 운영의 문제다. 출원번호별 스프레드시트만 두는 것보다, 책임자와 검토 기준이 명확한 캘린더 체계를 두는 편이 좋다. 신규 등록이 많아질수록 중앙 달력, 대리인 알림, 내부 승인 라인이 함께 있어야 한다.

Section 8/9 같은 핵심 유지 제출은 물론이고, 5년 연속 사용이 쌓인 등록은 Section 15를 선택적 강화 수단으로 함께 검토할 수 있다. 중요한 것은 일정 자체보다 일정마다 누가 사용 사실을 확인하고 누가 제출 판단을 내리는지까지 연결하는 것이다.

## 사용 포트폴리오 정리

등록된 goods/services 전부를 계속 유지하려고 하면 audit와 증거 부담이 커진다. 실제 사용 중인 범위를 기준으로 정리하고, 더 이상 사용하지 않는 항목은 전략적으로 축소하는 판단도 필요하다. 유지관리는 수비적 업무가 아니라 register quality를 관리하는 업무다.

실무에서는 현재 사용 중인 항목, 축소 검토 항목, 증거 보강이 필요한 항목을 나눠 두는 편이 좋다. 그래야 유지제출 직전에 갑자기 범위를 줄이거나, 반대로 사용 근거가 약한 항목을 그대로 남기는 실수를 줄일 수 있다.

## audit 대비와 증거 주기

audit 대응력은 증거 창고의 품질로 갈린다. 제품 사진, 포장 샘플, 캡처 화면, 판매 시점 자료, owner-user 관계 문서를 주기적으로 모아 두면 훨씬 안정적이다. 반대로 제출 시점에 급히 수집하면 사용 시점과 표장 버전 정합성이 흔들릴 가능성이 높다.

가장 실무적인 방식은 분기 또는 반기 기준으로 핵심 등록의 사용증거를 점검하는 것이다. 등록 시점의 specimen만 남겨 두는 것이 아니라, 실제 현재 사용을 보여 줄 자료를 계속 축적해야 유지관리와 audit에 대응하기 쉽다.

## 유지관리 운영표

등록 후 운영은 달력과 자료, 승인 구조를 같은 표에서 보는 편이 좋다.

| 항목 | 최소 관리 기준 |
| --- | --- |
| registration number | 포트폴리오 보드에 즉시 반영 |
| next filing window | 6개월 전부터 알림 시작 |
| evidence owner | 브랜드팀 / 사업팀 / 법무 중 담당 지정 |
| current use status | strong / moderate / weak |
| scope review | Keep / Narrow / Drop 후보 표시 |

## audit red flag

audit는 갑자기 생기는 일이 아니라, 평소 red flag를 방치한 결과로 드러나는 경우가 많다.

| red flag | 왜 문제인가 | 기본 대응 |
| --- | --- | --- |
| specimen 이후 최신 사용 자료가 없음 | 현재 사용 설명이 약해짐 | 분기 증거 점검 |
| owner-user 관계 문서가 낡음 | 사용 주체 설명이 흔들림 | licence/QC 문서 갱신 |
| 실제 판매 범위보다 지정 범위가 과도함 | audit 부담과 취약점 증가 | scope review |
