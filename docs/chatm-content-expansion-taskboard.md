# ChaTm 콘텐츠 보강 작업 분배표

## 공통 원칙

- generated JSON은 직접 수정하지 않는다.
- 모든 공개본 수정은 `ChaTm/content/source/chapters/`와 `ChaTm/content/source/manifest.json`에서 시작한다.
- 기한, 수수료, 절차명, 기관명, 시스템명은 `ChaTm/content/research/cn_tm_fact_verification_log.md` 확인 전 본문 단정 서술 금지
- 각 장에는 최소 1개의 체크리스트, 비교표, 판단 기준표, 운영 메모를 포함한다.

## 리더

- 목표 구조 12~14장 고정
- Sprint 1 범위 고정
- 머지 순서 통제
- 각 레인의 블로커 제거

## 원고 레인 A

대상:

- 제1장. 중국 상표제도 구조와 판단 프레임
- 제2장. 브랜드 구조와 중국어 표기 전략
- 제3장. 검색, 분류, 서브클래스 충돌 분석
- 제4장. 출원 경로 선택: 직접출원 vs 마드리드

완료 기준:

- 장별 도입부, 판단 기준, 체크리스트/표, 리스크 메모 포함
- H3 이상 소제목 최소 5개
- 검색 엔트리 증가에 기여하는 실질 섹션 작성

## 원고 레인 B

대상:

- 제5장. 출원서 작성 실무
- 제6장. 심사, 공고, 이의와 거절 대응
- 제7장. 등록 후 유지, 갱신과 사용 증거
- 제8장. 비사용취소, 무효와 권리 안정성

완료 기준:

- 단계별 실무 흐름과 판단 포인트 명확화
- 제출 자료, 기록 방식, 운영 캘린더 관련 표/체크리스트 포함
- 검증 필요 사실은 본문에 숨기지 말고 fact log 후보로 분리

## 원고 레인 C

대상:

- 제9장. 라이선스, 유통, 제조 구조와 상표 통제
- 제10장. 분쟁 대응: 행정, 사법, 플랫폼, 세관
- 제11장. 도메인, 저작권, 부정경쟁과의 결합 전략
- 제12장. 모니터링, 포트폴리오 운영, 내부통제(RACI)
- 부록

완료 기준:

- 집행 경로를 분기표나 선택 기준표로 정리
- 계약/운영/집행의 연결고리를 명확히 설명
- 부록에 체크리스트, 타임라인, 공식 링크, FAQ 초기본 포함

## 사실 검증 레인

즉시 확장할 로그 항목:

- 공식 기관명과 영문/국문 표기
- 출원, 공고, 이의, 거절 대응 주요 기한
- 갱신, 사용 증거, 비사용취소 관련 핵심 기간
- 지정상품/분류/서브클래스 관련 공식 용어
- 플랫폼, 행정, 사법, 세관 집행 경로 명칭
- 직접출원 vs 마드리드 비교 시 공식 절차명

완료 기준:

- 각 항목에 마지막 확인일, 1차 출처, 검증 메모 존재
- 본문 승격 가능/보류 여부 명시

## QA / 배포 준비 레인

병렬 준비:

- 장 제목과 `manifest.json` 일치 여부 점검
- 헤딩 구조와 표 형식 점검
- `/china` 리더 QA 시나리오 정리
- release-readiness 체크리스트 초안 작성

순차 검증:

1. `npm run content:china`
2. `npm test`
3. `npm run build`
4. `npm run build:pages:glotm`

완료 기준:

- 콘텐츠 조립 실패 없음
- 루트 검증 명령 통과
- `/china` 홈/챕터/검색/continue reading 스모크 기준 충족
