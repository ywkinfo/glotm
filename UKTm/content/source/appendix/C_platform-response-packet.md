# Appendix C. Platform Response Packet

플랫폼 신고는 채널별 폼이 달라도, 내부에서 준비해야 할 기본 자료는 거의 같다. 아래 패킷을 먼저 묶어 두면 Amazon, eBay, Meta, Google, marketplace 운영사, 호스팅사 대응 속도가 빨라진다.

## 기본 패킷

| 자료 | 필수 여부 | 메모 |
| --- | --- | --- |
| 권리자명 / 대리인명 | 필수 | 등록부와 같은 표기 사용 |
| 등록번호 / 출원번호 | 필수 | UK right인지 comparable right인지 구분 |
| 침해 URL / seller ID / listing ID | 필수 | 스크린샷만 제출하지 말고 URL도 보관 |
| 비교 이미지 | 필수 | genuine vs suspected item |
| 최초 발견일 | 필수 | 반복 침해 seller 추적용 |
| 영업·안전 리스크 메모 | 권장 | 리콜, 안전, 브랜드 훼손 여부 |

## 채널별 추가 질문

| 채널 | 추가로 자주 요구되는 자료 | 내부 확인 포인트 |
| --- | --- | --- |
| Marketplace | 브랜드 등록 여부, seller 반복 위반 이력 | 동일 seller의 다른 listing도 함께 조사 |
| SNS / 광고 | 계정 핸들, 광고 ID, 랜딩 URL | paid ad인지 organic post인지 구분 |
| 결제 / 호스팅 | merchant 정보, domain/hosting 연결 자료 | 도메인 대응과 병행 여부 검토 |
| 도메인 분쟁 | WHOIS/lookup, use screenshot, confusion explanation | Nominet DRS 또는 법원 트랙 선택 |

## 제출 전 체크리스트

- 침해 유형이 상표, passing off, counterfeit, grey market 중 무엇인지 구분했는가
- 요청 구제가 삭제, 비활성화, 계정 정지, 재등록 방지 중 무엇인지 명확한가
- 후속 escalation owner가 지정돼 있는가
- 동일 사실관계를 세관·도메인·유통계약 이슈와 함께 묶어볼 필요는 없는가
