# 등록 후 유지관리와 갱신 체계

등록 이후의 영국 상표 운영은 `갱신`, `register maintenance`, `실사용 점검` 세 축으로 돌아간다. 등록증을 받은 뒤 아무 조치를 하지 않아도 되는 기간처럼 느껴지지만, 실제로는 owner 구조와 사용 구조가 계속 변하기 때문에 관리 품질이 크게 갈린다.

영국 등록은 10년마다 갱신되며, 갱신 수수료는 첫 클래스 £200, 추가 클래스당 £50이다. 이 사실 자체보다 중요한 것은 갱신 이벤트를 포트폴리오 정비 이벤트로 쓰는 습관이다.

## pilot-lane maintenance owner board

UKTm의 유지관리 장은 full-depth handbook보다 “무엇을 놓치면 draft 공개본의 신뢰가 흔들릴까”를 빠르게 다시 보는 용도에 가깝다. 그래서 아래 네 줄을 먼저 적어 두면 pilot lane에서도 유지관리 공백을 크게 줄일 수 있다.

| 등록 후 잠글 것 | 왜 필요한가 | owner | 다음 handoff |
| --- | --- | --- | --- |
| renewal horizon | 10년 주기를 실제 calendar row로 바꿈 | IP + 재무 | 갱신 준비표 |
| comparable right 여부 | Brexit 이후 이중 관리 포인트 식별 | IP | renewal card |
| owner / actual user | licence, distributor, group use 설명 | 법무 + 사업팀 | evidence vault |
| 다음 점검일 | smoke QA 수준에서도 최소 점검 루틴 유지 | IP | 분기 triage |

## 유지관리 카드에 들어갈 최소 필드

유지관리 카드는 단순 renew date 목록이 아니라 권리 상태 설명서에 가까워야 한다. owner, 사용 채널, 핵심 상품군, 라이선스 여부, comparable mark 여부를 같이 적어야 다음 담당자도 바로 이해할 수 있다.

| 필드 | 왜 필요한가 |
| --- | --- |
| 등록번호 / 출원번호 | 플랫폼 신고·세관 대응과 연결 |
| 권리자명 | 계약·assignment 점검 기준 |
| 핵심 상품군 / 서비스 | 사용증거와 갱신 필요성 판단 기준 |
| 사용 채널 | D2C, retail, distributor, marketplace 구분 |
| comparable mark 여부 | Brexit 이후 이중 관리 포인트 식별 |
| 다음 갱신일 | 12개월 전부터 리마인더 시작 |

## 변경 관리가 갱신보다 먼저다

실무에서는 갱신 자체보다 변경 누락이 더 자주 문제를 만든다. 주소가 바뀌었는데 register가 그대로거나, 실제 user 구조가 바뀌었는데 licence story가 정리되지 않은 경우가 대표적이다.

그래서 월간 또는 분기 리뷰 때마다 `owner`, `address`, `actual user`, `contract structure`가 바뀌었는지 함께 보는 편이 좋다. 이 작업이 되어 있으면 갱신 직전의 급한 정리가 크게 줄어든다.

## 갱신 전 점검표

| 점검 항목 | 질문 | 결과 |
| --- | --- | --- |
| 사업 필요성 | 이 권리가 다음 10년에도 필요한가 | Keep / Consider drop |
| 사용 상태 | 실제 사용 증거를 설명할 수 있는가 | Strong / Moderate / Weak |
| 계약 연계 | licence, distribution, assignment와 연결되는가 | Yes / No |
| 분쟁 활용도 | 플랫폼·도메인·세관 대응에 쓰이는가 | High / Medium / Low |
| 비용 합리성 | 클래스 수를 그대로 유지할 필요가 있는가 | Keep / Narrow |

## 실무 체크리스트

- 갱신 달력을 사업 launch 캘린더와 같은 보드에서 보고 있는가
- comparable UK right를 별도 관리 항목으로 두는가
- renew decision 전에 evidence vault를 같이 검토하는가
- 권리 유지/포기 결정을 누가 최종 승인하는지 정했는가

## 분기 운영 quick triage

pilot lane에서는 완전한 portfolio review보다 `정상 / 보완 / 공백` 세 줄로 빠르게 나누는 편이 더 실용적이다.

| 상태 | 의미 | 기본 액션 |
| --- | --- | --- |
| 정상 | renewal row, comparable mark 여부, recent proof가 최신 | 현 상태 유지 |
| 보완 | evidence는 있으나 owner-user 또는 comparable memo가 약함 | 다음 점검 전 linkage 보강 |
| 공백 | renewal row 또는 recent proof가 비어 있음 | 즉시 maintenance card 재구성 |

이 triage 표가 있으면 UKTm은 smoke QA와 draft 공개본 유지라는 현재 lane 목적에 더 잘 맞게 작동한다.
