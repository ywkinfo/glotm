# 등록 후 유지관리와 갱신 체계

등록 이후의 영국 상표 운영은 서류 보관 업무가 아니라 권리 가치를 유지하는 일이다. 등록만 끝내고 달력, 권리자 정보, 사용 구조를 방치하면 실제로 필요할 때 권리를 충분히 활용하지 못하게 된다.

이 장은 등록 후 체계를 "캘린더 + 기록 + 승인 구조" 세 축으로 정리한다. 이 세 축이 있어야 갱신과 변경 관리가 안정된다.

## 갱신 주기와 기한 구조

영국 상표 등록의 존속 기간과 갱신 체계는 아래와 같다. 기한을 놓치면 권리가 소멸할 수 있으므로, 갱신 일정은 사업 캘린더와 통합해서 관리해야 한다.

| 항목 | 내용 |
|------|------|
| 최초 존속 기간 | 출원일로부터 10년 |
| 갱신 주기 | 10년마다 반복 |
| 갱신 신청 가능 시점 | 만료일 전 6개월부터 |
| 늦은 갱신 허용 기간 | 만료일 후 6개월 이내 (추가 수수료 발생) |
| 미갱신 시 결과 | 권리 소멸, 단 일정 기간 내 복권 신청 가능 |

갱신 수수료는 류(class) 수에 따라 달라지며, 출판 직전 UKIPO 공식 수수료표에서 확인해야 한다. 외부 대리인을 쓰는 경우 대리인 수수료가 별도로 발생한다.

## 유지관리 달력 만들기

등록 후에는 갱신 시점만 보는 것이 아니라, 관련 계약 갱신, 제품 리뉴얼, 유통 구조 변경 일정까지 함께 보는 편이 좋다. 상표 유지관리 달력이 사업 일정과 분리돼 있으면 누락이 생기기 쉽다.

실무적으로는 상표별 카드에 아래 항목을 함께 붙여 두는 방식이 가장 재사용성이 높다.

| 카드 항목 | 내용 |
|----------|------|
| 등록 번호 | UKIPO 등록 번호 |
| Owner | 현재 권리자 법인명 |
| 표장 형태 | 단어 / 도형 / 결합 |
| 지정 류 및 핵심 상품군 | 등록 명세 요약 |
| 사용 채널 | 온라인 / 오프라인 / 수출 |
| 갱신 만료일 | 다음 갱신 기한 |
| 담당 대리인 | 외부 영국 대리인 연락선 |
| 마지막 사용 증거 갱신일 | Evidence vault 최신 날짜 |

## 변경 관리 기준 세우기

등록 후 흔히 발생하는 변화는 owner 구조 변경, 주소 변경, 라이선스 구조 변경, 주력 상품군 변화다. 이 변화가 레지스터와 실사용 구조에 어떤 영향을 주는지 정기적으로 보는 편이 좋다.

특히 사업팀이 채널을 넓히거나 패키지를 바꿀 때 법무가 뒤늦게 알면, 유지관리보다 더 큰 정합성 문제가 생길 수 있다. 변경 관리 알림선을 미리 정해 두는 편이 안전하다.

## 갱신 전 점검 순서

갱신 자체는 기계적으로 보일 수 있지만, 실제로는 "이 권리를 앞으로도 유지할지" 다시 평가하는 시점이다.

| 점검 항목 | 확인 질문 |
|----------|----------|
| 사업 필요성 | 이 표장이 현재도 핵심 자산인가 |
| 사용 상태 | 최근 5년간 영국에서 실제 사용이 있는가 |
| 계약 연계 | 라이선스·유통 계약에 연결되어 있는가 |
| 분쟁 활용 가능성 | 집행·경고장에 실제로 사용되고 있는가 |
| 포트폴리오 중복 | 유사한 다른 권리로 커버 가능한가 |

사용이 약한 권리, 전략성이 낮은 권리, 관리 부담이 큰 권리는 포트폴리오 차원에서 다시 볼 필요가 있다. 단순 납부 리스트가 아니라 포트폴리오 정리 이벤트로 다루는 것이 바람직하다.
