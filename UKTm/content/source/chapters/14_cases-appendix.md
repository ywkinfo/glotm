# 사례, 체크리스트, 실행 부록

마지막 장의 목적은 법리를 더 추가하는 것이 아니라, 앞선 13개 장을 실제 업무로 전환하는 것이다. 영국 상표 운영은 사건이 생길 때마다 처음부터 다시 생각하면 느려지므로, 반복 가능한 체크리스트와 케이스 메모가 필요하다.

이 장은 팀이 바로 재사용할 수 있는 실행 템플릿 성격으로 읽는 편이 좋다. 완벽한 사례집보다 빠르게 꺼내 쓸 수 있는 운영 메모가 더 중요하다.

## 대표 시나리오 1: 런치 직전 충돌 발견

런치 직전 충돌이 보이면 가장 먼저 해야 할 일은 panic이 아니라 시나리오 정리다. 출원을 유지할지, 표장을 수정할지, 출시 순서를 바꿀지, 협상을 병행할지를 24시간 안에 좁혀야 한다.

이때 유용한 질문은 네 가지다. 표장이 정말 핵심 자산인가, 일정 조정이 가능한가, 협상 여지가 있는가, 이미 외부 커뮤니케이션이 시작되었는가. 이 질문에 답하면 대응 방향이 급격히 선명해진다.

| 체크 항목 | 확인 |
|----------|------|
| 충돌 표장의 실제 사용 여부 확인 | |
| 우리 출원/등록 상태 확인 | |
| 런치 일정 조정 가능 여부 파악 | |
| 협상 가능 여부 사전 판단 | |
| 대외 커뮤니케이션 중단 여부 검토 | |
| 24시간 이내 대응 방향 확정 | |

## 대표 시나리오 2: 등록 후 온라인 침해 확산

등록 후 침해가 여러 채널에서 동시에 보이면, 채널별 개별 대응보다 우선순위와 공통 자료 패킷을 먼저 정리해야 한다. 그렇지 않으면 팀이 각 채널을 따로 처리하다 지치게 된다.

핵심은 가장 큰 매출 손실이 나는 채널, 브랜드 훼손이 큰 채널, 재발 가능성이 높은 채널 순서로 대응하는 것이다.

## 대표 시나리오 4: 영국 대리인 교체

장기 보유 포트폴리오의 대리인을 교체하는 경우다. 이 상황은 단순 연락처 변경이 아니라 권리 이전 오류, 기한 누락, 증거 인계 문제가 생길 수 있다.

## 운영 체크리스트

아래 체크리스트는 분기별 포트폴리오 리뷰에서 사용한다.

### 출원·심사 단계

- [ ] 출원 전 owner, user, 승인자, 외부 대리인 창구가 정리되어 있는가
- [ ] 검색 결과가 red / amber / green 구조로 정리되어 있는가
- [ ] 출원 전략 문서 (목표 시장, 우선 표장, fallback)가 최신 상태인가
- [ ] 진행 중 심사 이슈의 대응 표 (논점·권장조치·마감·승인자)가 있는가

### 유지관리 단계

- [ ] 핵심 표장별 evidence vault가 최신 상태인가 (최근 분기 업데이트)
- [ ] 등록 후 5년 경과 권리 목록과 사용 증거 상태를 확인했는가
- [ ] 향후 12개월 내 갱신 예정 권리 목록이 달력에 있는가
- [ ] 갱신 전 포트폴리오 재평가 (유지·포기·축소)를 했는가
- [ ] Owner·주소·라이선스 구조 변경이 레지스터에 반영되어 있는가
