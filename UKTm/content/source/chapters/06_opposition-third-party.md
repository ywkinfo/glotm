# 공고, Opposition, Third Party Challenge 대응

영국에서 공고 이후 단계는 `등록 확률`보다 `외부 이해관계자와의 실제 충돌 관리`가 중요해지는 구간이다. opposition은 법적 절차이지만 동시에 협상, 공존, 범위 조정, launch 재설계가 함께 일어나는 운영 이벤트이기도 하다.

실무적으로는 공고가 되면 그날 바로 예상 반대자 후보와 내부 승인선을 정리해야 한다. 실제 opposition이 들어온 뒤부터 자료를 모으기 시작하면 cooling-off를 제대로 활용하기 어렵다.

## opposition 타임라인을 한 표로 본다

영국에서는 공고 후 2개월 안에 opposition이 제기될 수 있고, TM7A notice of threatened opposition을 넣으면 상대방은 추가 1개월을 확보할 수 있다. 정식 opposition이 시작되면 cooling-off를 통해 협상 기간을 확보할 수 있으므로, 사전 경계와 사후 협상 타임라인을 분리해서 봐야 한다.

| 시점 | 공식 이벤트 | 실무 액션 |
| --- | --- | --- |
| Publication day | Trade Marks Journal 공고 | 예상 반대자·유통 파트너·launch 일정 재점검 |
| Day 0-2 months | TM7 opposition 가능 기간 | 방어 포인트와 범위 양보 한도 정리 |
| Day 0-2 months | TM7A notice of threatened opposition 가능 | 예고가 들어오면 협상 브리프와 fallback 문구 준비 |
| Opposition served 후 | applicant response 준비 | 내부 승인자·대리인·사업팀 call 고정 |
| Cooling-off 초기 9개월 | 합의 탐색 | coexistence, 명세 조정, 사용 제한 조건 검토 |
| 추가 연장 가능 | 최대 18개월까지 | 장기 협상인지, 중도 종료 후 본안으로 갈지 판단 |

## opposition이 오면 먼저 무엇을 정하는가

첫 질문은 "이 사건에서 반드시 지켜야 할 권리 범위가 무엇인가"다. 전체 등록을 지키는 것이 목표인지, 핵심 클래스 몇 개만 남아도 되는지에 따라 전략이 완전히 달라진다.

둘째 질문은 "이 사건이 launch를 늦추는가"다. 영국 출시 일정이 이미 고정돼 있으면 법률 전략보다 사업 우선순위가 더 중요해질 수 있다.

| 시나리오 | 우선 전략 | 메모 |
| --- | --- | --- |
| 핵심 하우스마크 + 영국 출시 임박 | 방어 중심, 범위 양보 최소화 | CEO/경영진 escalation 조건 명확화 |
| 보조 브랜드 + 일정 유연 | 일부 양보 또는 재출원 검토 | 비용 대비 효율 우선 |
| 장기 파트너십 브랜드 | coexistence 가능성 탐색 | 계약·유통 조건과 같이 검토 |

## cooling-off를 협상 창으로 쓴다

cooling-off는 단순한 지연이 아니라 협상 기간이다. 이 기간에 무엇을 얻고 포기할 수 있는지 정리하지 않으면, 시간만 지나고 본안 대응 부담은 그대로 남는다.

효율적인 팀은 cooling-off 시작 시점에 `권리 범위`, `허용 가능한 사용 조건`, `출시 일정`, `대체 표장 존재 여부`를 한 페이지로 만든다. 협상 메모가 없으면 대리인과 내부 팀이 같은 말을 반복하게 된다.

## opposition 운영 체크리스트

| 점검 항목 | owner | 상태 |
| --- | --- | --- |
| 핵심 방어 범위를 정했는가 | 법무/IP |  |
| 양보 가능한 클래스·문구를 적었는가 | 브랜드팀 / 사업팀 |  |
| cooling-off 전략 메모가 있는가 | 법무 / 외부 대리인 |  |
| launch 일정과 PR 일정에 영향이 있는가 | 사업팀 |  |
| 종료 후 learnings를 다음 search memo에 반영했는가 | PM / 법무 |  |

## TM7A 수신 직후 체크

| 시간대 | 먼저 할 일 |
| --- | --- |
| 24시간 내 | 예상 반대자, 핵심 클래스, launch 영향 기록 |
| 48시간 내 | 양보 가능 문구와 fallback 범위 정리 |
| 72시간 내 | 대리인 질의, cooling-off 목표, 내부 승인자 확정 |

## 협상 종료 조건

| 상황 | 종료 판단 |
| --- | --- |
| 핵심 범위를 지킬 수 있음 | coexistence 또는 제한 합의 검토 |
| 핵심 범위를 지킬 수 없음 | 본안 대응 또는 refile 검토 |
| launch 일정이 크게 흔들림 | 경영진 escalation 후 조기 종료 |
