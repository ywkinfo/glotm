# 포트폴리오 운영, 내부 통제, RACI

상표 운영이 반복적으로 흔들리는 조직은 법률 지식이 부족해서라기보다, 책임 구조가 느슨한 경우가 많다. 출원, 심사, 사용증거, 집행, 계약 검토가 각 부서에 흩어져 있으면 작은 문제도 늦게 드러난다.

이 장의 목표는 복잡한 조직도를 만드는 것이 아니라, 영국 상표 운영에서 최소한 비어 있지 않아야 하는 책임 축을 정리하는 것이다.

## 필수 역할 정의

최소한 아래 다섯 축이 명확해야 대부분의 실무 흐름이 정리된다.

| 역할 | 핵심 책임 | 일반적 담당 부서 |
|------|----------|--------------|
| 브랜드 Owner | 표장 결정, 사용 기준 승인, 포트폴리오 우선순위 | 브랜드팀 |
| 법무 Owner | 출원·심사·분쟁 관리, 계약 검토, 외부 대리인 지시 | 법무팀 |
| 사업 Owner | 시장 진입 판단, 예산 승인, 런치 일정 확정 | 사업팀 |
| 외부 대리인 Contact | UKIPO 절차 실행, 대응 문안 작성, 기한 관리 | 영국 IP 전문 대리인 |
| 최종 승인자 | 전략적 양보 결정, Escalation 승인, 합의 승인 | 경영진 또는 법무 책임자 |

문제는 역할이 없는 것보다 `겹치지만 아무도 책임지지 않는 상태`다. 따라서 역할명보다 실제 승인권과 응답시간을 먼저 정리하는 편이 좋다.

## 영국 상표 운영 RACI 표

아래 RACI 표는 최소 운영 단위다. 조직 구조에 맞게 조정해서 사용한다.

- **R** (Responsible): 실제 수행 담당
- **A** (Accountable): 최종 결과 책임, 승인권자
- **C** (Consulted): 의견 제공, 검토 참여
- **I** (Informed): 결과 공유 대상

| 업무 | 브랜드팀 | 법무팀 | 사업팀 | 외부 대리인 | 최종 승인자 |
|------|---------|-------|-------|-----------|-----------|
| 출원 전 검색 범위 결정 | C | R | C | R | A |
| 출원서 작성·제출 | C | A | I | R | I |
| 심사 대응 문안 작성 | C | A | I | R | I |
| 심사 이슈 전략 결정 (양보 vs 방어) | C | R | C | C | A |
| 이의신청 대응 전략 | C | R | C | R | A |
| Cooling-off 협상 | C | R | C | C | A |
| 갱신 실행 | I | A | I | R | I |
| 갱신 유지 여부 판단 | C | R | R | C | A |
| 사용증거 수집 | R | A | R | I | I |
| Evidence vault 관리 | R | A | I | I | I |
| 라이선스 계약 검토 | C | R | R | C | A |
| 플랫폼 침해 신고 | R | A | I | C | I |
| 경고장 발송 | C | R | C | R | A |
| 소송 결정 | C | C | R | C | A |
| 포트폴리오 리뷰 (분기) | R | R | R | C | A |

## 운영 캘린더와 보고 체계

상표 운영은 사건이 생겼을 때만 보는 항목이 아니라 정기적으로 리뷰해야 하는 포트폴리오다.

| 주기 | 리뷰 항목 |
|------|----------|
| 월간 | 신규 출원 현황, 심사 이슈 진행, 플랫폼 침해 신고 결과 |
| 분기 | Evidence vault 상태, 갱신 예정 목록, 분쟁 현황 요약 |
| 반기 | 포트폴리오 전략성 점검, non-use 리스크 권리 목록 |
| 연간 | 전체 포트폴리오 리뷰, 외부 대리인 성과 평가, 예산 계획 |

보고 체계는 길 필요가 없다. 핵심 권리 변화, 위험 사건, 다음 승인 필요 항목만 짧게 보여 주는 방식이 경영진 이해에 더 효과적이다.

## RACI를 실무 문서로 바꾸기

RACI는 표만 만들고 끝내면 의미가 없다. 실제 제출 승인서, 증거 보관 규칙, 분쟁 escalation 기준, 플랫폼 대응 템플릿까지 연결돼야 살아 있는 운영 문서가 된다.

이 장의 RACI를 다음 장 체크리스트와 Appendix 템플릿에 연결해서 읽으면, 실행 가능한 운영 규칙이 완성된다.
