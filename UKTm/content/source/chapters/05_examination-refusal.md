# 심사, 절차 보정, 거절 대응

영국 심사 단계는 출원서 입력값을 실제 register 문구로 다듬는 단계다. UKIPO는 형식 오류만 보는 것이 아니라 식별력, 명세 명확성, 충돌 관련 이슈까지 폭넓게 점검하므로, 출원이 접수됐다고 해서 바로 안정권이라고 보면 안 된다.

운영상 중요한 것은 `심사 이슈를 얼마나 빨리 분류해 내부 승인으로 연결하느냐`다. 반응이 느리면 보정으로 끝날 사안을 불필요하게 길게 끌게 된다.

## 심사 흐름과 예시 기간

정확한 심사 속도는 workload와 사건 성격에 따라 달라지지만, UKIPO 공식 안내를 기준으로 보면 출원 후 초기 examination report가 몇 주 안에 나오는 케이스가 일반적이다. 실무에서는 런치 일정이 있는 사건이라면 4주 단위로 추적표를 두는 편이 안전하다.

| 단계 | 일반적 질문 | 실무 메모 |
| --- | --- | --- |
| 접수 직후 | 기본 입력값이 접수 가능한가 | filing receipt와 내부 tracker 즉시 정합성 확인 |
| examination report | 명세·절대적 거절사유·기타 이슈가 있는가 | issue type을 같은 날 분류 |
| 보정 / 의견서 | 좁힐지 방어할지 | launch와 자산 중요도를 같이 봄 |
| 공고 전 정리 | Trade Marks Journal에 올릴 준비가 됐는가 | opposition 가능성 브리프 연결 |

## 심사 이슈 3유형으로 분류한다

실무에서는 심사 이슈를 세 가지로 나누면 대부분의 의사결정이 빨라진다. 형식/명세 정합성, 식별력·기술성, 선행 권리 및 충돌 관련 논점이다.

| 유형 | 전형적 예시 | 기본 대응 |
| --- | --- | --- |
| 형식·명세 정합성 | 모호한 goods/services, 클래스 배치 오류 | 보정 위주, `Search UK trade mark classes` 재확인 |
| 절대적 거절사유 | 기술적·설명적 표현, 식별력 약함 | 논리 방어와 범위 조정 비교 |
| 충돌 관련 이슈 | 선행 권리와의 거리 설명 필요 | 출원 전략 재검토 또는 공고 이후 협상 대비 |

## TM5 / 국제 포트폴리오 정합성 체크

다국가 포트폴리오를 운영하는 조직이라면 영국 명세가 다른 국가 명세와 지나치게 어긋나지 않는지 봐야 한다. 특히 TM5 ID List나 글로벌 템플릿을 쓰는 팀은 `영국에서 수용되는 용어인지`를 최종 확인하는 단계가 필요하다.

이 포인트는 법적 의무라기보다 운영 리스크 관리에 가깝다. 글로벌 명세를 그대로 붙였다가 영국에서만 과도한 보정이 생기면 이후 use evidence 관리도 분절되기 쉽다.

## examination response handoff

영국 심사는 issue 자체보다 response handoff가 느릴 때 비용이 커진다. 따라서 examination report를 받으면 아래 세 row를 같은 날 채우는 편이 launch 중심 팀에 더 잘 맞는다.

### same-day issue triage

| 이슈 | owner | 그날 바로 남길 것 |
| --- | --- | --- |
| wording / class issue | 법무/IP + 대리인 | 좁히는 안 초안 |
| distinctiveness issue | 브랜드팀 + 법무/IP | mark importance, fallback 존재 여부 |
| conflict-facing comment | 법무/IP + 사업팀 | launch 영향, 협상 필요성 |

### narrow vs argue 질문표

| 질문 | Narrow 쪽으로 기우는 신호 | Argue 쪽으로 기우는 신호 |
| --- | --- | --- |
| 핵심 자산인가 | 보조 브랜드, 한정 SKU | core mark, 장기 투자 자산 |
| launch 일정이 촉박한가 | Yes, 빠른 통과가 우선 | No, 설명력 확보가 우선 |
| use evidence story가 있는가 | 약함 | 강함 |
| 다른 국가 정합성이 중요한가 | 낮음 | 높음 |

### 공고 전 launch 영향 메모

심사 대응은 공고 전 handoff와 붙어 있다. 공고까지 가는 동안 어떤 business impact가 생기는지 네 줄만 적어 두면 opposition 장과도 연결이 빨라진다.

| 항목 | 적을 내용 |
| --- | --- |
| affected SKU / channel | 어떤 제품과 채널이 영향을 받는가 |
| copy / packaging hold | 멈춰야 할 제작물이나 카피가 있는가 |
| fallback wording | 보정 시 살릴 최소 wording |
| next review date | 대리인 회신 전 다시 볼 날짜 |

## 실무 예시

예를 들어 클래스 자체는 맞지만 goods/services 표현이 지나치게 마케팅 문구에 가까운 경우, UKIPO 단계에서는 좁은 보정으로 해결되는 일이 많다. 반대로 브랜드 핵심 자산이 식별력 이슈를 받는다면, 단기 출시 일정만 보고 바로 좁히기보다 장기 보호가치까지 같이 따져야 한다.

이때 내부 의사결정 메모는 길 필요가 없다. `문제`, `선택지`, `추천`, `마감` 네 줄만 선명하게 적어도 실행 속도가 크게 오른다.
