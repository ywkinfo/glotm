# 출원서 준비와 명세 작성

영국 출원 실무의 성패는 표장 선택만이 아니라 출원서 입력값의 정합성에서 갈린다. owner 정보, 우선 표장 형태, goods/services, 내부 승인 라인이 흐리면 이후 심사 대응과 등록 후 관리가 모두 불편해진다.

이 장은 출원서 작성 자체보다 "어떤 입력값을 누구 책임으로 고정할 것인가"에 초점을 둔다. 입력값이 안정되면 외부 대리인과의 왕복도 훨씬 줄어든다.

## owner와 사용 구조 정리

출원 전 owner와 actual user 구조를 먼저 정리해야 한다. 지배회사, 유통법인, 라이선시, 공동 프로젝트 법인 중 누가 권리를 가져갈지 명확하지 않으면 등록 후 계약과 증거 관리까지 꼬이게 된다.

브랜드팀이 표장을 정해도 법무와 재무가 owner 구조를 검토하지 않으면 나중에 정정 비용이 커질 수 있다. 출원서 작성 전에 소유 구조 메모를 한 번 만드는 편이 좋다.

## Nice 분류와 goods and services 명세 작성

UKIPO는 Nice 국제분류(45류) 체계를 사용한다. 명세는 지정 상품/서비스를 해당 류(class)에 맞게 기재해야 하며, 분류가 틀리면 심사에서 정정 요청을 받게 된다.

| 명세 작성 원칙 | 설명 |
|--------------|------|
| 실사용과 근접하게 | 카탈로그·판매 페이지 표현과 비교해 기재 |
| 확장 예정 범위 반영 | 12개월 내 출시 예정 제품군 포함 |
| 과도하게 넓은 표현 피하기 | 나중에 non-use 방어 부담 증가 |
| 내부 카테고리명 그대로 쓰지 않기 | 시장 설명 가능한 표현으로 정리 |
| 복수 류 필요 시 분리 출원 검토 | 류당 수수료 구조 확인 |

명세 문장은 launch 직전 영업자료, 제품 카탈로그, 이커머스 카테고리와도 비교해 보는 편이 좋다. 출원서만 따로 예쁘게 정리돼 있고 실제 사용 표현과 멀어지면 유지관리 단계에서 다시 비용이 든다.

## 제출 전 체크리스트

제출 직전에는 아래 항목을 한 번에 확인해야 한다. 이 단계의 체크가 약하면 나중에 수정이 필요한 입력 오류가 누적된다.

| 항목 | 확인 내용 | 담당 |
|------|----------|------|
| 표장 형태 | 단어 / 도형 / 결합 / 색채 여부 | 브랜드팀 |
| owner 표기 | 법인명 정확성, 등록 주소 | 법무·재무 |
| 대리인 정보 | UK 공인 대리인 여부 | 법무 |
| Nice 분류 | 류(class) 번호와 명세 일치 | 법무 + 브랜드팀 |
| 우선권 주장 | 기초 출원 번호·날짜 (6개월 이내) | 법무 |
| 내부 승인 | 사업 대표 또는 법무 책임자 승인 | 승인자 |
| 외부 대리인 리뷰 | 영국 대리인 최종 검토 여부 | 외부 대리인 |

실무자는 제출 승인자와 제출 실행자를 분리해 두는 편이 안전하다. 승인자는 사업 판단을, 실행자는 형식 정확성을 책임지게 두면 속도와 품질이 함께 유지된다.
