# 출원서 준비와 명세 작성

영국 출원서는 단순 입력폼이 아니라 이후 심사, 라이선스, 사용증거, 갱신 품질을 결정하는 기준 데이터다. 특히 owner 정보와 goods/services 표현이 흔들리면 심사 때 보정이 늘고, 등록 후 evidence story도 약해진다.

영국에서는 `Search UK trade mark classes`와 UKIPO classification guide를 먼저 보고 명세를 다듬는 편이 효율적이다. 내부 상품 카테고리 명을 그대로 복사하기보다, 실제 판매·마케팅 문구와 register 문구가 자연스럽게 이어지는지를 보는 것이 중요하다.

## owner와 실제 사용 구조를 먼저 잠근다

권리자는 누가 되고 실제 사용자는 누가 될지 출원 전에 정해야 한다. 지주회사, 운영법인, 유통법인, 라이선시가 따로 있을 경우에는 `왜 이 법인을 owner로 두는지`까지 메모해 둬야 이후 계약과 증거 정리가 쉽다.

| 입력값 | 확인 질문 | 누가 확인할지 |
| --- | --- | --- |
| 권리자명 | 등록 후 계약과 세금 구조까지 무리 없는가 | 법무 / 재무 |
| 주소 | official correspondence와 일치하는가 | 법무 / 외부 대리인 |
| 표장 형태 | word / logo / composite 중 무엇인가 | 브랜드팀 / 법무 |
| 실제 사용자 | owner와 user가 다르면 licence story가 설명 가능한가 | 법무 / 사업팀 |

## Nice 분류와 영국 명세 표현 원칙

영국은 Nice 분류 체계를 쓰지만, 실제 심사에서는 용어의 명확성과 수용 가능성이 더 중요하다. 그래서 내부 제품명보다 UKIPO가 이해할 수 있는 설명 용어로 바꾸는 작업이 필요하다.

| 내부 표현 예시 | 더 나은 영국 명세 표현 방향 | 메모 |
| --- | --- | --- |
| all-in-one creator kit | 소매 상품인지, SaaS인지, 교육 서비스인지 분해 | 기능별로 클래스가 갈릴 수 있음 |
| brand community membership | downloadable content, educational services, loyalty services 등으로 분리 | 모호한 마케팅 문구는 피함 |
| wellness ecosystem | goods/services를 실제 제공 단위로 다시 씀 | 광범위한 콘셉트어는 심사 때 약함 |

## 명세 초안 검토표

명세는 길이보다 구조가 중요하다. 실제로는 클래스별로 `지금 사용`, `곧 사용`, `당장 필요 없음` 세 구획으로 나누면 훨씬 빠르게 다듬어진다.

| 클래스 | 초안 문구 | 실제 사용 시점 | 리스크 메모 | 유지 여부 |
| --- | --- | --- | --- | --- |
|  |  | 현재 / 12개월 내 / 미정 | 과도하게 넓음 / 모호함 / 적절 | Keep / Narrow / Remove |
|  |  | 현재 / 12개월 내 / 미정 | 과도하게 넓음 / 모호함 / 적절 | Keep / Narrow / Remove |

## 제출 전 체크리스트

제출 직전에는 승인자와 실행자를 분리해 놓는 편이 안전하다. 승인자는 사업적 범위와 owner 구조를 확인하고, 실행자는 입력 정확성과 제출 준비를 본다.

| 점검 항목 | 확인자 | 상태 |
| --- | --- | --- |
| 표장 버전이 최신인가 | 브랜드팀 |  |
| owner와 address가 확정됐는가 | 법무 |  |
| classes와 명세 문구가 `Search UK trade mark classes` 기준으로 정리됐는가 | 법무 / 외부 대리인 |  |
| 우선출원·국제출원 연계 여부가 정리됐는가 | 법무 |  |
| 제출 승인자와 제출 실행자가 분리됐는가 | PM / 법무 |  |

## 실무 예시

예를 들어 영국 이커머스 launch를 앞둔 소비재 브랜드라면, 상품 판매용 클래스와 온라인 retail services를 한 번에 넣고 싶어질 수 있다. 이때 실제 판매주체와 서비스 제공주체가 다르면 owner 구조를 먼저 정리하지 않으면 심사 이후 라이선스와 evidence story가 꼬이기 쉽다.

## 명세 브리프 전달 패킷

외부 대리인에게 명세 검토를 맡길 때는 아래 패킷이 같이 가는 편이 좋다.

| 묶음 | 포함 내용 |
| --- | --- |
| owner pack | 권리자, 실제 사용자, address |
| mark pack | word/logo/composite 구분, 최신 버전 |
| class pack | core goods/services, 인접 범위, 제거 후보 |
| launch pack | 출시 일정, 채널, 후속 확장 계획 |

## 표현 수정 기준

| 상황 | 기본 수정 방향 |
| --- | --- |
| 내부 마케팅 문구가 모호함 | 실제 goods/services 설명 용어로 교체 |
| 너무 넓은 개념어가 들어감 | core 범위 중심으로 좁힘 |
| owner-user 구조가 복잡함 | licence story 설명 가능한 범위로 재정리 |
