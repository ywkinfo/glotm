# 라이선스, 유통, 공동브랜딩 운영

영국에서 상표는 register에만 존재하는 권리가 아니라 계약 구조 속에서 살아 움직이는 자산이다. 라이선스, 유통계약, 공동브랜딩이 늘어날수록 `누가 어떤 표장으로 무엇을 할 수 있는지`를 정교하게 관리해야 실제 권리 가치가 유지된다.

특히 영국 register에는 licence를 기록할 수 있으므로, 운영상 중요한 거래라면 recorded licence와 unrecorded licence의 차이를 실무적으로 구분해야 한다. 법적 세부 효과는 계약 구조에 따라 다르지만, 최소한 register transparency와 내부 통제 수준은 달라진다.

## recorded licence와 unrecorded licence를 구분한다

| 구분 | recorded licence | unrecorded licence |
| --- | --- | --- |
| register 반영 | Yes | No |
| 대외 투명성 | 높음 | 낮음 |
| 내부 운영 | register와 계약 데이터가 연결됨 | 계약팀 자료에만 남기 쉬움 |
| 적합한 상황 | 장기·핵심 라이선스, 실사용 설명이 중요한 구조 | 단기·제한적 프로젝트, 빠른 실행이 우선일 때 |

## 유통 계약은 상표 운영 문서이기도 하다

유통 계약은 매출 조건만이 아니라 상표 사용 품질 기준을 외부 채널에 내리는 도구다. reseller와 marketplace 운영 파트너가 어떻게 브랜드를 표시할지 통제하지 않으면, 실제 사용 모습이 register 전략과 멀어질 수 있다.

| 점검 포인트 | 왜 중요한가 |
| --- | --- |
| 사용 허용 표장 버전 | 로고 변형과 word mark 혼용 방지 |
| 패키지·상품명 승인 절차 | register와 실제 사용 정합성 유지 |
| online listing 규칙 | 플랫폼 침해 대응 시 genuine listing 식별 가능 |
| 종료 후 사용 중단 의무 | 계약 종료 후 residual use 관리 |

## 공동브랜딩은 naming rule을 먼저 만든다

캠페인명, 협업명, 한정판 라벨은 짧게 쓰고 빨리 사라지는 것처럼 보여도 디지털 흔적은 오래 남는다. 따라서 공동브랜딩 프로젝트에서는 출원 여부보다 먼저 `누가 naming을 승인하고, 종료 후 누가 archive를 정리할지`를 정해야 한다.

## 실무 예시

예를 들어 영국 distributor가 자체 landing page를 만들고 프로모션용 공동 로고를 붙이는 구조라면, 판매는 정상이어도 실제 사용 형태가 통제되지 않을 수 있다. 이 경우 recorded licence 여부, listing 승인 절차, 종료 후 page 정리 규칙을 한 세트로 묶어야 다음 분쟁에서 설명이 쉽다.

또한 공동브랜딩 종료 뒤 남는 SNS 게시물과 marketplace asset은 계약 종료일과 함께 정리 기준을 두는 편이 좋다. 오프라인 행사는 끝났더라도 온라인 흔적은 침해나 혼동 자료로 오래 남기 때문이다.

## 라이선스 / 유통 점검 체크리스트

- recorded licence가 필요한 핵심 거래를 구분했는가
- 실제 사용 승인자와 계약 승인자가 같은지, 다른지 명확한가
- reseller / distributor가 쓰는 상품명·listing 문구를 검토하는가
- 종료 후 잔존 재고와 디지털 흔적 처리 규칙이 있는가
