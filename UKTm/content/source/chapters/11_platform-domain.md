# 플랫폼, 도메인, 온라인 침해 대응

영국 시장의 상표 문제는 오프라인보다 온라인에서 먼저 보이는 경우가 많다. 마켓플레이스, 소셜 커머스, 검색 광고, 도메인, 셀러 계정 문제가 동시에 얽히면 전통적인 경고장만으로는 속도가 맞지 않는다.

이 장은 온라인 침해를 별도 운영 레인으로 관리하는 방법을 다룬다. 핵심은 법적 타당성과 실행 속도를 동시에 확보하는 것이다.

## 온라인 침해 유형과 대응 채널

온라인 침해는 유형에 따라 대응 채널이 다르다. 모든 케이스에 같은 방법을 쓰면 효율이 떨어진다.

| 침해 유형 | 주요 채널 | 대응 방법 |
|-----------|----------|----------|
| 브랜드 사칭 (가짜 공식몰) | Amazon, eBay, Shopify | 플랫폼 IP 침해 신고 |
| 위조품 판매 | 마켓플레이스 | 플랫폼 신고 + 세관 연계 |
| 검색 광고 키워드 도용 | Google Ads | Google 상표 정책 신고 |
| 소셜미디어 계정 사칭 | Instagram, Facebook, TikTok | 플랫폼 IP 신고 절차 |
| .uk / .co.uk 도메인 분쟁 | Nominet (DRS) | Nominet 분쟁해결절차 신청 |
| 기타 gTLD 도메인 분쟁 | ICANN (UDRP) | UDRP 중재 신청 |
| 비공식 유통·리셀링 | 마켓플레이스, 직거래 플랫폼 | 플랫폼 신고 + 법적 경고장 병행 |

### Nominet과 .uk 도메인 분쟁

영국 도메인(.uk, .co.uk, .org.uk 등)의 분쟁은 Nominet의 Dispute Resolution Service(DRS)를 통해 처리된다. UDRP와 별도로 운영되는 영국 전용 절차이며, 상표권을 가진 경우 부당 등록된 .uk 도메인을 이전 또는 취소 요청할 수 있다.

DRS 신청을 위해서는 해당 도메인 등록이 권리를 침해하고 있음을 입증해야 한다. 플랫폼 신고보다 처리 기간이 길 수 있으나, 도메인 자체의 소유권 이전을 받아낼 수 있다는 점에서 장기적으로 중요한 채널이다.

## 플랫폼별 운영 기준 만들기

플랫폼 신고는 각 채널의 요구 자료와 처리 속도가 다르기 때문에, 권리 자료 패킷을 표준화해 두는 편이 좋다.

| 기본 패킷 항목 | 내용 |
|--------------|------|
| 상표 등록 증명 | UKIPO 등록 번호, 등록 상품·서비스 |
| 권리자 정보 | 법인명, 담당자, 연락처 |
| 대표 침해 예시 | URL, 스크린샷 (날짜 포함) |
| 공식 판매 채널 정보 | 공식 스토어 링크 또는 판매 정책 |
| 이전 조치 이력 | 동일 침해자 또는 유사 케이스 처리 기록 |

플랫폼 대응이 느려지는 가장 큰 이유는 사실 부족보다 자료 형식 불일치다. 채널별 요구사항을 미리 템플릿화해 두면 대응 속도가 확실히 빨라진다.

## 도메인과 검색 광고 이슈

도메인과 검색 광고는 상표 침해가 항상 동일한 형태로 드러나지 않는다. 혼동 유발, 사칭, 리셀링, 비교 광고, 키워드 집행 등 맥락이 다르므로 한 가지 대응 방식만 고집하면 효율이 떨어진다.

실무자는 먼저 문제를 `브랜드 사칭`, `유입 가로채기`, `비공식 유통`, `콘텐츠 남용` 정도로 나눠 보는 편이 좋다. 분류가 되면 대응 채널도 같이 정리된다.

## 온라인 대응 기록 남기기

온라인 침해는 한 번 처리하고 끝나는 경우가 드물다. 계정이 바뀌거나 판매처가 옮겨가므로 사건 단위 기록이 쌓여야 재발 대응이 빨라진다.

사건 카드에는 URL, 스크린샷, 신고 일시, 처리 결과, 재발 여부를 남기는 편이 좋다. 이 기록은 이후 플랫폼 에스컬레이션과 외부 대리인 브리프에 바로 재사용된다.
