# 집행, 경고장, 분쟁 대응

영국 상표 집행은 등록 이후 따로 붙는 업무가 아니라, 출원·사용·플랫폼 운영이 실제 시장 문제와 만나는 지점이다. seller takedown, 유통계약 위반, 경고장, IPEC 소송 검토가 연속적으로 이어질 수 있으므로 triage 체계가 중요하다.

핵심은 사건을 발견한 날 바로 `포럼`, `구제 수준`, `속도`를 정하는 것이다. 느린 triage는 작은 침해를 장기 분쟁으로 키우고, 너무 빠른 escalation은 불필요한 비용을 만든다.

## IPEC와 High Court 선택 기준

| 포럼 | 적합한 사건 | 장점 | 주의점 |
| --- | --- | --- | --- |
| IPEC small claims | 저가치 온라인 침해, 간단한 구제 요청 | 비교적 간이·저비용, £10,000 이하 분쟁에 적합 | 복잡한 사실관계에는 한계 |
| IPEC multi-track | 중간 규모 침해, 빠른 금지와 예측 가능한 비용이 중요할 때 | damages 최대 £500,000, recoverable costs cap 참고 가능 | 복잡도가 높아지면 준비 부담 증가 |
| High Court | 고액, 복합 사실관계, 광범위한 구제 필요 | 증거조사와 구제 폭이 넓음 | 시간·비용 부담이 큼 |

## 경고장 전에 사건을 분류한다

같은 상표 침해라도 seller 삭제가 우선인지, 유통계약 집행이 우선인지, 법원 포럼 검토가 우선인지 다르다. 따라서 법률명칭보다 운영 분류가 먼저다.

| 사건 유형 | 첫 액션 | 다음 액션 |
| --- | --- | --- |
| marketplace seller | 증거 확보 + 플랫폼 패킷 투입 | 반복 seller면 IPEC 검토 |
| 유통 파트너 무단 사용 | 계약 검토 + cease and desist | 재고 회수 / 품질 통제 |
| counterfeit / safety risk | 증거 확보 + 세관/플랫폼 병행 | 강한 경고장 또는 법원 |
| 도메인·계정 선점 | registry/policy route 검토 | Nominet DRS 또는 법원 |

## 대응 템포를 정해 둔다

영국 집행은 빠른 사건과 깊은 사건을 분리해야 한다. seller takedown처럼 48시간 안에 움직여야 하는 문제와, 공동브랜딩 종료 후 잔존 사용처럼 사실관계 정리가 먼저인 문제를 같은 속도로 다루면 둘 다 어정쩡해진다.

| 템포 | 대상 사건 | owner |
| --- | --- | --- |
| 24-48시간 | safety issue, counterfeit suspicion, 대형 seller | 법무 / 사업팀 / 플랫폼 owner |
| 3-5영업일 | 경고장, 유통 위반, 광고 문구 침해 | 법무 / 외부 대리인 |
| 2주+ | IPEC/High Court 전략, settlement negotiation | 법무 / 경영진 |

## dispute escalation quick pack

영국 집행은 발견 당일 1페이지 memo가 있으면 훨씬 빨라진다. 포럼 선택보다 먼저 아래 세 row를 적어 두면 경고장, 협상, 법원 검토를 같은 사건 보드에서 움직이기 쉽다.

### desired remedy memo

| 원하는 구제 | 언제 먼저 두나 |
| --- | --- |
| listing 삭제 | seller 확산을 즉시 끊어야 할 때 |
| 사용 중단 약속 | 유통 파트너 또는 협상 여지가 있을 때 |
| injunction 검토 | 영국 시장 피해가 빠르게 커질 때 |
| settlement frame | 비용 회수보다 조기 종료가 중요할 때 |

### cease and desist issue packet

경고장은 문구보다 입력값이 먼저다. 아래 네 줄을 미리 묶어 두면 외부 대리인과 내부 팀이 같은 사실관계를 보게 된다.

| packet 항목 | 왜 필요한가 |
| --- | --- |
| infringing use sample | 어떤 사용을 멈추게 하려는지 명확화 |
| right basis | 등록번호, owner, UK-only 근거 |
| target outcome | 삭제, 수정, 협상, 금지 중 무엇인지 |
| deadline / escalation | 미응답 시 다음 포럼과 시점 |

### settlement vs filing gate

모든 사건을 즉시 소송으로 넘길 필요는 없다. 반대로 합의로 오래 끌수록 비용이 커지는 사건도 있으니, gate 질문을 미리 적어 두는 편이 좋다.

| 질문 | settlement 쪽 신호 | filing 쪽 신호 |
| --- | --- | --- |
| 상대방이 반복 침해자인가 | No | Yes |
| 원하는 구제가 제한적인가 | Yes | No |
| 영국 launch 피해가 즉각적인가 | No | Yes |
| 증거와 owner story가 정리됐는가 | 협상 후 보강 가능 | 이미 정리돼 있어 바로 제출 가능 |

## 집행 체크리스트

- 침해 유형을 seller issue, contract issue, domain issue, court matter로 나눴는가
- 원하는 구제가 삭제, 중단, 합의, 금지명령 중 무엇인지 정했는가
- IPEC와 High Court 중 어느 포럼이 더 맞는지 초기 판단을 남겼는가
- 세관·플랫폼·도메인 이슈를 같은 사건 보드에서 추적하는가
