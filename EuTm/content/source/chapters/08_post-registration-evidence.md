# 제8장. 등록 후 사용, 갱신, 증거 관리

등록 이후에는 사용 흔적과 갱신 달력이 권리 유지의 핵심이 된다. 이 장의 목적은 본사와 현지 조직이 함께 보는 운영 문서를 만드는 데 있다. 유럽 권역에서는 등록 자체보다, 어떤 증거를 어떤 주기로 모으고 어느 범위까지 권리를 유지할지가 더 중요해지는 경우가 많다.

## 등록 후 가장 먼저 잠글 것

등록 직후 아래 항목을 하나의 운영 시트에 넣는 편이 좋다.

- 등록번호와 등록일
- 권리자 정보
- covered goods/services
- 핵심 사용 국가
- evidence owner
- 갱신 담당자
- 영국 병행 권리 여부

이렇게 해 두면 등록증을 받은 뒤에도 권리가 실제 운영 체계에 연결된다.

## genuine use 리스크를 어떻게 볼 것인가

EUIPO cancellation guidance는 등록 후 연속 5년 동안 genuine use가 없으면 revocation risk가 생길 수 있음을 분명히 한다. 또한 사용이 일부 goods/services에만 입증되면 보호 범위가 그 sub-category 수준으로 제한될 수 있다.

이 의미는 단순하다. 등록증이 있다고 해서 전체 지정 범위를 영원히 보유하는 것이 아니라, 실제 사용 구조가 계속 register와 연결돼 있어야 한다는 뜻이다.

## 어떤 증거를 모아야 하는가

유럽 권역 운영에서는 아래 자료를 주기적으로 축적하는 편이 좋다.

- 제품 포장, 라벨, 태그
- 판매 페이지와 주문 화면
- 배포된 브로슈어, 카탈로그, 캠페인 화면
- 인보이스, 출고, 유통 관련 자료
- 국가별 사용 시점이 보이는 캡처
- 라이선스 또는 유통 구조 메모

핵심은 “이 표장이 이 상품/서비스에 대해 이 시장에서 실제 사용됐다”를 나중에 설명할 수 있어야 한다는 점이다.

## 증거 저장 구조

사용증거는 파일이 아니라 시스템으로 관리하는 편이 좋다. 최소 기준은 다음과 같다.

- 원본과 제출용 파일 분리
- 국가, 채널, 날짜, 표장 버전이 파일명에 표시됨
- 본사와 현지팀이 같은 저장 규칙을 사용함
- owner와 actual user 관계가 메모로 남아 있음

나중에 cancellation, invalidity, opposition proof of use, 유통 분쟁이 생기면 결국 누가 어떤 자료를 얼마나 빨리 꺼낼 수 있는지가 품질을 좌우한다.

## 갱신 운영

EUTM는 10년 단위로 갱신되며 계속 연장 가능하다. EUIPO 운영 기준으로는 만료 전 6개월이 basic renewal period이고, 그 시점을 넘기면 추가 수수료가 붙는 6개월 grace period가 이어진다. 운영상으로는 이 구조를 달력에 먼저 고정해 두는 것이 중요하다.

- 갱신일을 중앙 달력에 넣는 것
- EUIPO 알림만 믿지 않는 것
- 갱신 시점에 유지할 goods/services 범위를 같이 점검하는 것

EUIPO guidance상 등록 만료 전 알림이 제공될 수 있지만, 사무소 알림이 오지 않았다고 해서 만료 책임이 사라지지는 않는다. 따라서 내부 달력과 대리인 달력을 동시에 두는 편이 안전하다.

## 포트폴리오 정리 원칙

모든 goods/services를 끝까지 유지하는 것이 항상 좋은 전략은 아니다. 실제 사용 중인 범위와 앞으로 유지할 범위를 나눠 보는 것이 중요하다.

- 핵심 매출 범위: 유지 우선
- 사용이 약한 범위: 증거 보강 또는 축소 검토
- 더 이상 전략 가치가 없는 범위: 정리 검토

이 작업을 갱신 직전이 아니라 평소에 해 두면 포트폴리오 품질이 훨씬 좋아진다.

## evidence vault 카드 예시

증거를 모으는 것과 증거를 꺼내 쓰는 것은 다르다. 카드 구조를 두면 국가·채널·표장 버전 기준으로 바로 찾을 수 있다.

| 카드 항목 | 예시 |
| --- | --- |
| mark | house mark / product mark |
| 국가 | EU 공통 / 핵심국 / UK |
| 채널 | D2C, retail, distributor, marketplace |
| 대표 증거 | 포장, 판매 페이지, invoice, brochure |
| 마지막 점검일 | 분기 또는 반기 날짜 |

## 갱신 12개월 전 점검표

갱신 이벤트는 비용 결제보다 포트폴리오 정리의 창으로 쓰는 편이 좋다.

| 시점 | 점검할 것 |
| --- | --- |
| 12개월 전 | 유지 필요성, 핵심국, UK 병행 구조 |
| 6개월 전 | evidence vault 강도, goods/services 축소 후보 |
| 1개월 전 | 결제, 대리인, 내부 승인 완료 여부 |

## 분기 운영 루틴

등록 후 운영은 갱신 직전에만 움직이면 늦다. 실제로는 분기 단위 루틴으로 증거와 owner map을 계속 갱신하는 편이 훨씬 안전하다.

| 주기 | 점검할 것 | owner |
| --- | --- | --- |
| 매분기 | 신규 판매국, 신규 채널, 제품명 변경 여부 | 사업팀 + IP |
| 반기 | evidence vault 누락, 사용 약한 goods/services 후보 | IP |
| 연 1회 | renewal horizon, UK 병행 권리, 대리인 라인 | IP + 외부대리인 |

이 루틴의 목적은 자료를 많이 모으는 것이 아니라, genuine use를 나중에 설명할 수 있는 상태를 유지하는 것이다. 분기 단위로 owner를 고정해 두면 cancellation이나 proof of use 요청이 왔을 때 "누가 자료를 갖고 있는가"부터 다시 찾지 않아도 된다.

## owner-user-evidence 연결표

유럽 권역에서는 권리자(owner)와 실제 사용자(user)가 다른 구조가 자주 나온다. 본사 명의 권리를 현지 판매법인, distributor, marketplace seller가 실제로 움직이는 경우, 관계 설명이 안 되면 증거는 많아도 설득력이 약해진다.

| 항목 | 남겨야 할 메모 | 빠지면 생기는 문제 |
| --- | --- | --- |
| owner | 등록 명의자, 권리 보유 목적 | 권리 귀속 설명이 흔들림 |
| actual user | 판매 주체, 채널 운영 주체 | genuine use 연결고리가 약해짐 |
| linkage | 라이선스, 유통계약, 그룹사 관계 | 증거가 누구의 사용인지 불명확해짐 |
| evidence location | 포장, invoice, listing, brochure 저장 위치 | 제출 속도가 크게 느려짐 |

이 표를 운영 시트에 붙여 두면 제10장의 distribution 이슈와도 연결된다. 유럽 등록 이후의 핵심은 자료를 쌓는 일이 아니라, 그 자료가 어느 권리 구조를 설명하는지 즉시 말할 수 있게 만드는 일이다.

## EuTm용 체크리스트

- 등록 직후 운영 시트를 만들었는가
- 5년 genuine use 리스크를 달력에 연결했는가
- 증거를 국가·채널·날짜 기준으로 저장하는가
- 영국 병행 권리의 갱신 흐름을 따로 보고 있는가
- goods/services를 유지할 범위와 축소 후보로 구분했는가
- owner 변경이나 유통 구조 변경이 증거 시트에 반영되는가

## 한 줄 결론

유럽 등록 이후 운영의 핵심은 등록증 보관이 아니라, genuine use와 갱신을 같은 시스템으로 관리해 권리 범위를 계속 설명 가능하게 만드는 일이다.
