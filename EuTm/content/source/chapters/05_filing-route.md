# 제5장. 출원 경로와 서류 설계

같은 표장이라도 출원 경로와 서류 정리가 다르면 후속 대응 난도가 달라진다. 이 장의 목적은 유럽 권역 출원 단계에서 owner, representative, goods/services, filing route, priority를 한 번에 정리하는 운영 표준을 만드는 데 있다.

## 출원 전에 먼저 잠글 것

유럽 출원에서는 filing form 자체보다 입력값 정합성이 더 중요하다. 최소한 아래 항목을 먼저 잠가 두는 편이 좋다.

- 표장 최종 버전
- owner 명칭과 주소
- 사용할 goods/services 문구
- EUTM / national / UK / Madrid 중 어떤 경로인지
- 우선권 주장 여부
- 외부 대리인 또는 미국/현지 법무와의 커뮤니케이션 담당자

이 값이 정리되지 않으면 출원 자체는 가능해도 이후 보정, 이의, 사용증거 관리에서 같은 문제가 반복된다.

## owner와 대표 구조

유럽 권역 출원은 본사 단독 출원, 현지 법인 출원, 국제 포트폴리오 명의, 라이선스 구조 등 ownership 시나리오가 다양하다. 따라서 출원 단계에서 다음을 같이 본다.

- 법인명 표기가 내부 계약과 일치하는가
- 실제 사용 주체와 owner가 어떤 관계인지
- 어느 권리 단위에 어떤 대리인을 붙일 것인지
- 이후 양도나 라이선스가 예상되는지

대표 구조도 중요하다. 장기적으로는 누가 외부 대리인과 대화하고, 누가 goods/services 수정 승인권을 가지며, 누가 결제를 담당하는지 명확해야 한다.

## goods/services 설계 원칙

EUIPO는 goods/services 선택의 명확성과 accepted terms 활용을 중요하게 본다. 실무적으로는 goods/services를 아래처럼 나누는 편이 좋다.

- 현재 바로 사용하는 core line
- 12~24개월 안의 realistic adjacent line
- 아직 제품 정의가 없는 future line

출원에는 첫 두 범위 중심으로 들어가는 편이 안전하다. 미래 가설까지 과도하게 넣으면 나중에 사용 입증이나 포트폴리오 정리에 부담이 커진다.

## EUTM filing과 national filing 실무 차이

EUTM filing은 하나의 권리 단위로 EU 전체를 다룰 때 강점이 있다. 반면 national filing은 국가별 문서와 대리인 흐름이 더 세분화된다. 따라서 filing stage에서 아래를 적어 두면 좋다.

- 왜 EUTM인지 또는 왜 national인지
- 어느 국가를 우선순위로 삼는지
- 영국은 별도로 병행하는지
- 일부 국가 실패 시 전체 전략에 어떤 영향이 있는지

이 메모가 없으면 filing decision이 단순한 기계적 제출로 남는다.

## core-state filing control board

validate lane에서 제5장이 흔들리는 순간은 권리 선택 결론은 있는데, 그 결론을 어떤 filing packet row로 내려야 하는지 안 적혀 있을 때다. 특히 EUTM 단독, core-state national filing, UK split, mixed structure가 같이 보이는 경우에는 "무엇을 어느 packet에 넣을 것인가"를 먼저 고정하는 편이 좋다.

| launch 구조 | filing packet에서 먼저 분리할 것 | 왜 미리 적어야 하는가 |
| --- | --- | --- |
| EUTM 중심 launch | EU-wide goods/services, central owner, central representative | 광역 권리의 설명력을 filing 단계에서 먼저 고정하기 위해 |
| 특정 회원국 리스크 집중 | core-state risk memo, local counsel line, 별도 goods/services note | 한 국가 리스크 때문에 전체 packet이 다시 흔들리는 일을 줄이기 위해 |
| EU + UK 병행 | UK row, UK representative 필요 여부, split evidence start note | filing 직후 제8장 calendar와 연결이 어긋나지 않게 하기 위해 |
| mixed structure | house mark와 product mark의 route 분리 | 한 브랜드의 리스크가 다른 브랜드 packet까지 끌고 가지 않게 하기 위해 |

이 보드를 두면 제5장은 form 입력 장이 아니라, 제2장에서 잠근 rights structure를 실제 filing packet 단위로 번역하는 장이 된다.

## EUTM Easy Filing / Filing form과 Fast Track 메모

EUIPO는 EUTM Easy Filing과 EUTM Filing form을 구분해 제공하고, Fast Track 조건도 별도로 둔다. 실무상 의미는 간단하다. 단순한 구조와 accepted goods/services, 적절한 결제 흐름을 갖추면 더 부드럽게 처리될 수 있다는 점이다.

따라서 유럽 출원에서는 다음을 점검하는 편이 좋다.

- 단순 케이스인지, 복수 이해관계자가 얽힌 케이스인지
- goods/services를 accepted terms 중심으로 설계할 수 있는지
- 결제와 제출 책임자가 명확한지
- priority나 representative 구조 때문에 일반 filing form이 필요한지

정교한 사건이라면 “빠른 양식”보다 “설명 가능한 양식”이 더 중요하다.

## priority와 Madrid 연계

유럽 filing 단계에서 priority나 Madrid 연계가 있는 경우는 별도 관리가 필요하다. 우선권을 붙일지, 기본권리를 어떤 office에 둘지, Madrid를 병행할지 여부를 filing note에 남겨 두는 편이 좋다.

EUIPO guideline 기준으로 priority는 first filing 후 6개월 안에 주장할 수 있고, priority claim을 뒷받침하는 자료는 filing date 기준 3개월 안에 정리해 두는 편이 안전하다. 따라서 priority를 붙일 사건이라면 filing pack의 timing row에 `first filing date / 6개월 만기 / priority document owner`를 같이 적어 두는 편이 좋다.

WIPO 기준으로 Madrid를 쓰려면 national 또는 regional application/registration이 전제된다. 따라서 유럽 경로를 먼저 정리하지 않은 상태에서 Madrid만 앞세우면 오히려 구조가 불명확해질 수 있다.

### Madrid base-right preflight 표

`Madrid linkage`를 filing 단계에서 쓰려면, Madrid를 “EU filing 대체재”처럼 적는 대신 어떤 base right에 기대는지와 그 base right가 실제 launch 구조와 맞는지를 먼저 적는 편이 좋다.

| 질문 | yes일 때 남길 메모 | no일 때 바로 보이는 리스크 |
| --- | --- | --- |
| regional or national base right가 이미 정리돼 있는가 | base application/registration row와 owner를 packet에 기록 | Madrid만 있고 filing 기준권이 흐려짐 |
| EU / UK launch가 같은 포트폴리오 문법으로 움직이는가 | central management memo를 유지 | UK split과 EU filing이 서로 다른 일정으로 움직일 수 있음 |
| goods/services가 base right와 크게 어긋나지 않는가 | accepted terms와 launch 범위를 같이 잠금 | filing 직후 설명 불일치가 커짐 |
| representative / approval line이 중앙에서 잠겨 있는가 | Madrid follow-up owner를 별도 표기 | 국가별 follow-up이 다시 분산됨 |

이 preflight 표를 두면 Madrid는 권리 선택을 흐리는 옵션이 아니라, 이미 잠근 filing 구조를 국제 포트폴리오로 어떻게 연결할지 보여 주는 옵션으로 읽히게 된다.

## 제출 전 QC

제출 직전에는 아래 항목을 한 번에 본다.

- owner 명칭과 주소가 일치하는가
- 표장 버전이 최종본인가
- goods/services 문구가 accepted terms 또는 설명 가능한 문구인가
- 영국 병행 여부가 반영됐는가
- priority 또는 Madrid 연계가 문서에 남아 있는가
- 대표자와 결제 책임자가 정해졌는가

## filing pack 체크리스트

filing 단계는 form 입력보다 pack 준비가 중요하다. 아래 pack이 있으면 EUTM, national, UK 어느 경로로 가도 다시 쓰기 쉽다.

| pack 항목 | 포함 내용 |
| --- | --- |
| owner pack | 법인명, 주소, ownership note |
| mark pack | 최종 표장 버전, 사용형 메모 |
| goods/services pack | core, adjacent, accepted terms |
| route pack | EUTM / national / UK / Madrid 선택 이유 |
| timing pack | priority 여부, launch 일정, 결제 승인 |

## 대표자/대리인 handoff 규칙

외부 대리인 handoff는 메일 전달보다 내부 선택 이유를 넘기는 일이 더 중요하다.

| 질문 | 미리 답해 둘 것 |
| --- | --- |
| 누가 대리인과 직접 소통하는가 | 본사 IP / 지역 counsel / PM |
| goods/services 수정 승인권은 누구에게 있는가 | 사업팀 / 법무 / 공동 |
| 영국 병행 여부를 누가 잠그는가 | 권리 선택 회의 owner |
| priority/Madrid 이슈는 누가 확인하는가 | 포트폴리오 리드 |

## accepted terms / fast-track triage

유럽 filing 단계에서는 “빠른 양식”을 쓰는지보다 accepted terms와 제출 구조가 설명 가능한지가 더 중요하다. 아래 표를 두면 filing form 선택이 실무 판단으로 바뀐다.

| 신호 | 단순 filing 쪽 | 일반 filing 쪽 |
| --- | --- | --- |
| goods/services | accepted terms 중심 | custom 설명이 많음 |
| ownership 구조 | 단순 | 복수 이해관계자 또는 복잡한 representative 구조 |
| UK/Madrid 연계 | 없음 또는 단순 | 병행 memo가 많음 |
| 내부 승인 | 이미 잠김 | 아직 조정 중 |

이 표를 쓰면 “빠르게 제출”과 “설명 가능한 제출” 사이의 선택 기준이 분명해진다.

## owner / representative control sheet

출원 직전 흔들리는 것은 문구보다 owner와 representative 라인이다. control sheet를 같이 두는 편이 좋다.

| 항목 | 잠글 것 |
| --- | --- |
| owner | 등록 명의자, 권리 보유 목적 |
| representative | 외부 대리인, 지역 counsel, 내부 창구 |
| approval line | goods/services, 비용, UK 병행 승인자 |
| evidence follow-up owner | 제8장 evidence vault 책임자 |

이 시트를 filing pack에 붙이면, 제5장이 제출 단계뿐 아니라 이후 renewal/evidence 운영의 입구 역할도 하게 된다.

## filing 14-day readiness board

권리 선택이 끝난 뒤 2주 안에 filing 입력값을 잠그지 못하면, EUTM / national / UK 병행 결론이 다시 흔들린다. 그래서 route pack과 owner sheet를 시간표로 같이 보는 편이 좋다.

| 시점 | 반드시 끝낼 것 | owner |
| --- | --- | --- |
| D+1 ~ D+3 | owner / representative / approval line 최종본 | IP |
| D+4 ~ D+7 | goods/services accepted terms와 core/adjacent 범위 잠금 | IP + 사업팀 |
| D+8 ~ D+10 | UK 병행 row, priority, Madrid linkage 메모 | IP |
| D+11 ~ D+14 | external counsel handoff와 결제/제출 준비 | PM + IP |

이 보드가 있으면 filing stage가 “형식 작성”보다 “설명 가능한 input freeze”로 바뀌고, 제8장 evidence calendar와도 자연스럽게 연결된다.

## route pack lock board

유럽 filing 단계에서 실제로 흔들리는 것은 form이 아니라, route pack의 어느 줄이 비어 있는지다. 제5장에서는 적어도 아래 네 묶음을 route pack 안에 고정하는 편이 좋다.

| route pack 묶음 | 반드시 잠글 것 | 다음 장으로의 연결 |
| --- | --- | --- |
| rights pack | EUTM / national / UK / Madrid 결론 | 제2장 rights memo와 일치 여부 확인 |
| terms pack | accepted terms, core / adjacent 범위 | 제8장 weak goods/services review |
| representative pack | external counsel, approval line, payment owner | 제7장 opposition line과 일치 여부 |
| timing pack | priority, UK split row, launch 일정 | renewal / evidence calendar |

이 보드를 두면 제5장은 제출용 문구 장이 아니라, route pack을 실제 운영 표준으로 고정하는 장이 된다.

## filing-to-evidence handoff

EuTm에서는 filing이 잘 됐는지보다, 그 filing 구조가 나중에도 설명 가능한 evidence 구조로 이어지는지가 더 중요하다. 그래서 제5장 끝에는 evidence handoff를 명시해 두는 편이 좋다.

| filing 단계 출력값 | 제8장으로 넘길 것 | 이유 |
| --- | --- | --- |
| owner / representative control sheet | owner-user-evidence 연결표 | 등록 후 설명력을 유지하기 위해 |
| core / adjacent goods/services | weak goods/services review 후보 | sub-category limitation 리스크를 미리 보기 위해 |
| UK split row | EU / UK 분기 캘린더 | launch timing과 evidence 시작점이 다르기 때문에 |

이 handoff가 있으면 filing pack이 제출에서 끝나지 않고, 제8장 evidence system의 첫 버전으로 이어진다.

## 한 줄 결론

유럽 출원 실무의 핵심은 폼 작성 자체보다, 어떤 권리 단위로 누구 명의로 어떤 범위를 제출하는지 내부 설명이 먼저 가능한 상태를 만드는 데 있다.
