## 2. 브랜드 / 이름 질문

### 2-1. 먼저 잠글 이름은 무엇인가

- [ ] 핵심 표장만 먼저 낼 것인가, 로고 결합까지 같이 갈 것인가
- [ ] 영문명과 한글명 중 중국에서 먼저 노출될 이름은 무엇인가
- [ ] 소비자, 플랫폼, 영업팀이 각각 어떤 이름으로 부를 가능성이 큰가
- [ ] 지금 이름을 바꾸면 사업 설명이 더 쉬워지는가, 더 어려워지는가

- [ ] 발음이 비슷한 다른 이름이 이미 널리 쓰이고 있는가
- [ ] 뜻은 다르지만 소비자 인식이 비슷해질 위험이 있는가

## 3. 중국어 명칭 질문

### 3-1. 중국어 이름은 누가 정하는가

- [ ] 내부에서 먼저 정했는가, 아니면 현지 대리인 제안에 맡길 것인가
- [ ] 음역, 의역, 별칭 중 어떤 방식이 현재 사업에 맞는가
- [ ] 중국어 이름이 브랜드 핵심 의미를 유지하는가
- [ ] 중국어 이름을 따로 등록할 필요가 있는가

**작성 메모**

____________________________________________________________________

### 3-2. 중국어 이름은 실제로 쓰일 수 있는가

- [ ] 소비자가 자연스럽게 부를 수 있는가
- [ ] 검색창에서 찾기 쉬운가
- [ ] 포장, 상세페이지, 광고 소재에 넣어도 어색하지 않은가
- [ ] 파트너와 대리점이 임의로 다른 이름을 만들 가능성은 없는가

**작성 메모**

____________________________________________________________________

### 3-3. 방어용 변형이 필요한가

- [ ] 짧은 기간 안에 충돌할 가능성이 높은 변형이 있는가
- [ ] 중국어 약칭, 별칭, 축약형이 별도로 필요할 수 있는가
- [ ] 현재 예산 안에서 방어표장까지 묶을 수 있는가
- [ ] 방어 범위를 넣는 이유를 한 문장으로 설명할 수 있는가

**작성 메모**

## 4. 지정상품 / 서브클래스 질문

### 4-1. 코어 범위는 무엇인가

- [ ] 지금 판매 중인 SKU는 무엇인가
- [ ] 3개월 안에 출시할 상품은 무엇인가
- [ ] 꼭 먼저 확보해야 할 클래스는 무엇인가
- [ ] 같은 클래스 안에서도 충돌이 몰릴 구간은 어디인가

**작성 메모**

____________________________________________________________________

### 4-2. 확장 범위는 어디까지인가

- [ ] 12~24개월 안에 현실적인 확장 제품이 있는가
- [ ] 확장 범위가 실제 사업 계획과 연결되는가
- [ ] 당장 쓰지 않지만 충돌 방지 목적이 분명한가
- [ ] 너무 넓게 잡아 설명이 약해질 위험은 없는가

**작성 메모**

____________________________________________________________________

### 4-3. 서브클래스 리스크는 점검됐는가

- [ ] 상품 문구와 실제 판매 구조가 연결되는가
- [ ] 클래스 번호만 맞고 실제 사용과 안 맞는 구간은 없는가
- [ ] 유통 채널 기준으로 따로 봐야 할 항목이 있는가
- [ ] 나중에 다시 같은 출원을 반복할 가능성이 있는가

## 5. owner / user 질문

### 5-1. 누가 권리를 갖는가

- [ ] owner는 누구인가
- [ ] 실제 user는 누구인가
- [ ] 본사, 중국 법인, 제조 파트너, 유통사 중 누가 무엇을 맡는가
- [ ] owner와 user가 다르다면 그 이유를 설명할 수 있는가

**작성 메모**

____________________________________________________________________

### 5-2. 누가 운영을 통제하는가

- [ ] 패키지와 판매 화면을 누가 관리하는가
- [ ] 대리인과 누가 커뮤니케이션하는가
- [ ] 사용 증거를 누가 모으는가
- [ ] 내부 문서와 실제 운영이 같은가

**작성 메모**

- [ ] 현지 커스터마이징이 많이 필요한가
- [ ] 본사 기준의 중앙 관리가 더 중요한가

**작성 메모**

____________________________________________________________________

### 6-2. 조직이 끝까지 운영할 수 있는가

- [ ] 내부 의사결정 속도가 출원 일정과 맞는가
- [ ] 수정 요청이 오면 누가 최종 결정을 내리는가
- [ ] 지금 경로를 밀 때 비용보다 운영 리스크가 더 크지 않은가

## 7. 제출 전 최종 확인

### 7-1. 마지막 질문

- [ ] 중국에서 실제로 어떤 이름으로 먼저 쓰게 될지 정해졌는가
- [ ] 중국어 이름을 지금 확정하지 않으면 누가 대신 정하게 되는가
- [ ] 코어 범위와 방어 범위를 구분해서 설명할 수 있는가
- [ ] owner와 user를 문서로 설명할 수 있는가
- [ ] 현재 조직이 이 출원 경로를 끝까지 운영할 수 있는가

### 7-2. 최종 판정

- [ ] GO, 바로 제출 가능
- [ ] REVISE, 일부 조정 후 제출
- [ ] HOLD, 내부 정리 후 재검토
- [ ] ABANDON, 현재 안건 중단
