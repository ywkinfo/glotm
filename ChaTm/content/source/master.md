## 서문

#### 왜 중국 상표는 별도 운영 체계로 봐야 하는가

중국은 한국 기업에게 가장 중요한 생산기지이자 소비시장 중 하나다. 그러나 상표 운영 관점에서는 단순히 “큰 시장”이 아니라, 출원 속도와 권리 충돌, 판매채널 확장, 플랫폼 대응이 한꺼번에 압축되는 시장에 가깝다.

한국이나 다른 국가에서 브랜드를 먼저 쓰고 있다는 사실만으로 중국에서 자동으로 보호가 되지는 않는다. 중국에서는 선출원 구조, 지정상품 설계, 사용 구조, 집행 경로를 일찍 정리하지 않으면 사업 확장 단계에서 리스크가 한 번에 드러날 수 있다.

#### 이 가이드가 다루는 질문

- 어떤 브랜드를 중국에서 먼저 선점해야 하는가
- 중국어 표기와 영문 표기를 어떻게 함께 설계해야 하는가
- 출원 후 공고, 이의, 거절 대응은 어떤 흐름으로 준비해야 하는가
- 등록 후 사용 구조와 증거를 어떤 체계로 쌓아야 하는가
- 분쟁이 생기면 행정, 사법, 플랫폼, 세관 중 어디서부터 움직여야 하는가

## 제1장. 중국 상표제도 구조와 판단 프레임

중국 상표제도는 한 번의 신청 절차가 아니라, 출원 설계와 등록 후 운영, 분쟁 대응을 하나의 연속된 운영 시스템으로 봐야 한다.

#### 관할 기관과 역할

중국 상표 행정 실무의 핵심 접점은 `CNIPA`다. 제도 총괄 정보는 [CNIPA](https://www.cnipa.gov.cn/)와 [SAMR](https://www.samr.gov.cn/) 공지에서 확인하고, 국제출원 연계 여부는 [WIPO](https://www.wipo.int/) 자료와 함께 본다.

##### 실무에서 구분해야 할 세 개의 층위

1. 출원과 심사
2. 등록 후 유지와 사용 관리
3. 분쟁 발생 시 행정, 법원, 플랫폼, 세관 대응

#### 중국에서 먼저 무너지는 판단

중국 진출 초기에는 보통 다음 오판이 반복된다.

- 한국 등록이 있으니 중국에서도 늦지 않다고 보는 판단
- 영문 브랜드만 확보하면 중국어 표기는 나중에 정리해도 된다는 판단
- 현지 유통 파트너가 브랜드 사용을 대신 설명해 줄 것이라는 기대

##### 브랜드 구조를 처음부터 나눠서 보는 기준

- 핵심 표장: 반드시 선점할 명칭, 로고, 결합표장
- 중국어 표기: 소비자 인식과 검색 경로를 고려한 현지명
- 방어 표장: 짧은 기간 안에 충돌 가능성이 높은 변형

###### 최소 질문

브랜드명 하나를 정할 때도 “누가 쓰는가, 어떤 상품에 붙는가, 어떤 채널에서 먼저 노출되는가”를 같이 물어야 한다.

## 제2장. 출원 설계와 선행 검토

중국 출원은 서류 제출 자체보다 출원 전 설계의 정확도가 훨씬 중요하다. 지정상품 범위와 명칭 구조가 어긋나면 등록 이후 사용과 집행에서도 같은 문제가 반복된다. 그래서 이 장은 단순한 제출 절차보다, 무엇을 먼저 판단하고 어떤 자료를 잠가야 하는지를 중심으로 읽는 편이 맞다.

실무적으로 필요한 산출물도 분명하다. 검색 화면이나 대리인 메모를 모으는 것보다, 어떤 표장을 중국에서 먼저 선점할지, 영문과 중국어 표기를 어떻게 설계할지, 어떤 상품 범위를 코어로 둘지, 직접 출원과 국제출원 중 어느 경로가 현재 조직 구조에 맞는지를 정리한 결정 메모가 더 중요하다. 즉, 이 장의 출발점은 신청서 작성이 아니라 “무엇을 어떻게 낼지”를 내부에서 한 장으로 정리하는 데 있다.

#### 선행 검색의 목적

검색은 단순 중복 확인이 아니라 다음 세 가지를 보기 위한 절차다.

- 동일 또는 유사한 선행표장의 존재
- 중국어 음역 또는 의역 충돌 가능성
- 실제 사업 계획과 지정상품 설계의 불일치

좋은 검색의 결과는 "있다/없다"가 아니라 결론 방향이다. 같은 브랜드를 그대로 밀고 갈 수 있는지, 중국어 표기를 따로 조정해야 하는지, 상품 범위를 줄이면 살릴 수 있는지, 아니면 지금 접는 편이 전체 비용을 줄이는지를 함께 적어야 한다. 중국은 선출원 구조와 현지 표기 문제가 초기에 어긋나면 나중에 수정 비용이 더 커지기 쉽다.

실무적으로는 다음 네 가지 방향 중 하나로 정리하는 편이 좋다.

- **go**: 현재 표장과 범위로 그대로 밀고 갈 수 있는 경우
- **revise**: 중국어 표기, 지정상품, 범위를 손보면 살릴 수 있는 경우
- **hold**: 내부 사업 구조나 사용자 구조가 아직 덜 정리된 경우
- **abandon**: 현지 충돌 위험이 높아 다른 표기나 다른 브랜드가 더 합리적인 경우

이 정리가 있어야 다음 단계의 지정상품 설계와 출원 경로 판단도 흔들리지 않는다.

### 표장 구조와 중국어 표기 설계

중국에서는 영문 표장만 확보하면 충분하다고 보기 어렵다. 실제 소비자 인식과 검색 경로, 유통 현장에서는 중국어 표기와 영문 표기가 서로 다른 역할을 할 수 있기 때문이다. 그래서 출원 전부터 적어도 세 층으로 나눠 보는 편이 좋다.

- 핵심 표장: 반드시 선점해야 하는 영문명, 로고, 결합표장
- 중국어 표기: 음역, 의역, 별칭 중 실제 시장에서 쓸 가능성이 높은 이름
- 방어 표장: 짧은 기간 안에 충돌 가능성이 높은 변형

중국어 표기는 단순 번역 문제가 아니다. 소비자가 실제로 어떤 이름으로 부를지, 플랫폼 검색에서 어떤 표기가 먼저 걸릴지, 파트너와 대리점이 어떤 이름을 현장에서 쓸지를 같이 봐야 한다. 늦게 정리할수록 브랜드와 유통 현장이 서로 다른 이름을 쓰기 시작한다.

| 점검 항목 | 질문 |
| --- | --- |
| 음역 | 영문명을 소리대로 옮겼을 때 충돌 위험이 큰가 |
| 의역 | 뜻을 살린 현지명이 브랜드 정체성을 훼손하지 않는가 |
| 병행 사용 | 영문과 중국어를 동시에 쓸 때 포장과 판매 화면이 일관적인가 |
| 검색 경로 | 소비자가 실제로 어떤 표기로 검색할 가능성이 큰가 |

### 지정상품과 서브클래스 리스크

중국에서는 국제분류만 맞춘다고 끝나지 않는다. 실제 거래 구조와 충돌 가능성을 보려면 서브클래스 관점의 설계가 필요하다. 즉, 클래스 번호를 맞춘 뒤에도 실제 판매 SKU, 유통 채널, 핵심 제품군과 연결되는지를 한 번 더 확인해야 한다.

실무적으로는 코어 범위와 인접 확장 범위를 나눠 보는 편이 좋다. 현재 판매하거나 즉시 출시할 상품군은 우선 확보 대상이고, 12~24개월 안에 현실적인 확장 범위는 사업 로드맵과 연결될 때만 넣는 편이 안전하다. 너무 넓게 잡으면 이후 사용과 집행에서 설명이 약해지고, 너무 좁게 잡으면 중국 시장 확장 단계에서 비슷한 출원을 반복하게 된다.

| 범위 유형 | 의미 | 실무 판단 |
| --- | --- | --- |
| Core | 현재 판매 또는 즉시 출시 예정인 핵심 상품군 | 우선 확보 대상 |
| Adjacent | 12~24개월 안에 현실적인 확장 범위 | 사업 로드맵과 연결될 때만 포함 |
| Defensive | 당장 쓰지 않지만 충돌 방지 목적이 큰 범위 | 이유가 명확할 때만 제한적으로 검토 |

##### 출원 전 체크리스트

| 점검 항목 | 확인 질문 |
| --- | --- |
| 표장 구조 | 영문, 한글, 중국어 표기를 각각 따로 출원할지 |
| 상품 범위 | 실제 판매 SKU와 지정상품이 연결되는지 |
| 사용 주체 | 본사, 중국 법인, 대리점 중 누가 실제 사용자로 남는지 |
| 우선순위 | 시장 진입 전에 반드시 확보할 클래스가 무엇인지 |
| 경로 선택 | 직접 출원과 국제출원 중 현재 운영 구조에 맞는지 |

### 출원 주체와 사용 주체를 먼저 맞춘다

표장과 상품 범위만 보는 것으로는 부족하다. 누가 owner이고 누가 실제 user인지, 누가 패키지와 판매 화면을 통제하는지, 누가 대리인과 커뮤니케이션하는지도 같이 잠가 둬야 한다. owner와 user가 다를 수 있다는 사실 자체가 문제는 아니고, 그 차이를 문서와 실제 운영에서 설명하지 못하는 상태가 문제다.

#### 직접 출원과 국제출원의 판단

직접 출원과 마드리드 출원은 비용만으로 고르면 안 된다. 중국 단일 시장의 속도, 지정상품 커스터마이징 필요성, 내부 의사결정 속도를 함께 봐야 한다.

중국이 최우선 시장이라면 등록 가능성 검토와 지정상품 커스터마이징이 더 중요해질 수 있다. 반대로 다국가 확장이 동시에 진행될 때는 국제출원이 관리 효율 측면에서 유리한 경우도 있다. 실무적으로는 어느 경로가 더 고급인지보다, 어느 경로가 지금 조직이 끝까지 감당할 수 있는지로 판단하는 편이 맞다.

| 판단 항목 | 직접 출원 쪽이 유리한 신호 | 국제출원 쪽이 유리한 신호 |
| --- | --- | --- |
| 시장 우선순위 | 중국 단일 시장이 핵심 | 여러 국가가 동시에 중요 |
| 상품 설계 | 현지 커스터마이징이 중요 | 본사 기준의 통합 관리가 중요 |
| 속도 | 빠른 왕복과 세밀한 조정 필요 | 중앙 관리 효율이 우선 |
| 내부 구조 | 중국 전담 판단 체계가 있음 | 국가별 운영을 묶어야 함 |

##### 실무 메모

출원 전 마지막 질문은 비교적 단순하다. 이 표장을 중국에서 어떤 이름으로 먼저 쓰게 될지, 중국어 표기를 지금 확정하지 않으면 누가 현장에서 대신 정하게 될지, 코어 상품군과 방어 범위를 구분해 설명할 수 있는지, owner와 user를 문서로 설명할 수 있는지, 현재 조직이 이 출원 경로를 끝까지 운영할 수 있는지를 묻는 것이다. 이 질문에 답이 흐리면 아직 출원 설계가 덜 끝난 상태다.

## 제3장. 심사, 공고, 이의와 거절 대응

출원 이후는 기다리는 구간이 아니다. 이 장의 목적은 통지서를 받았을 때 무엇을 먼저 읽고, 어떤 순서로 판단하고, 무엇을 남겨야 하는지 정리하는 데 있다. 같은 결과라도 대응 방식에 따라 살릴 수 있는 표장과 접어야 하는 표장이 갈리므로, 문구 해설보다 흐름을 먼저 잡아야 한다.

### 결과를 받으면 먼저 세 가지를 나눈다

심사 결과나 통지서를 받으면 처음부터 전체 문장을 해석하려 들지 않는다. 먼저 아래 세 가지를 분리한다.

- **무엇이 문제인지**: 형식 문제인지, 식별력 문제인지, 선행권리 충돌인지, 상품 범위 문제인지
- **어떤 범위가 걸렸는지**: 표장 전체인지, 일부 지정상품인지, 특정 표현만인지
- **지금 필요한 행동이 무엇인지**: 설명 보완인지, 범위 조정인지, 유지 대응인지, 내부 중단 판단인지

이렇게 나누면 통지서가 길어도 흔들리지 않는다. 실무상 중요한 것은 문서의 길이가 아니라, 우리가 지금 어디를 고쳐야 하는지다.

#### 읽는 순서

1. 통지 유형을 먼저 확인한다.
2. 문제 문구를 그대로 옮겨 적는다.
3. 영향받는 표장과 상품 범위를 표시한다.
4. 대응 가능성과 내부 우선순위를 짧게 적는다.
5. 바로 답할지, 내부 검토 후 답할지 정한다.

| 확인 항목 | 보는 방법 | 메모 기준 |
| --- | --- | --- |
| 통지 유형 | 거절, 보정 요구, 의견 제출, 공고 관련 통지인지 구분 | 유형을 먼저 적는다 |
| 문제 지점 | 표장, 상품, 선행표장, 절차 중 무엇인지 표시 | 한 줄로 요약한다 |
| 영향 범위 | 전부인지 일부인지, 한 클래스인지 여러 클래스인지 확인 | 범위를 좁혀 쓴다 |
| 내부 과제 | 대리인 대응, 사업 조정, 증거 확보, 중단 판단 중 무엇인지 정리 | 다음 행동으로 연결한다 |

### 거절과 보정 요구는 같은 방식으로 읽지 않는다

거절통지와 보정 요구는 둘 다 대응이 필요하지만, 읽는 순서가 다르다. 거절은 먼저 왜 막혔는지의 논리를 읽어야 하고, 보정 요구는 무엇을 덜어내거나 덧붙여야 하는지를 먼저 봐야 한다.

#### 거절통지를 읽을 때

거절통지는 보통 다음 네 가지 축으로 나눠 본다.

- 선행표장과의 충돌 여부
- 식별력 또는 일반명칭성 문제
- 지정상품이 너무 넓거나 부정확한 문제
- 서류상 설명 부족 또는 내부 불일치

핵심은 “전체를 다시 낼지”가 아니라 “어떤 논리로 남길 수 있는지”다. 같은 거절이라도 표장 전체를 지키는 사건과 일부 상품만 지키는 사건은 다르게 다뤄야 한다.

#### 보정 요구를 읽을 때

보정 요구는 대개 다음 두 가지 중 하나다.

- 형식이나 설명을 바로잡으면 넘어갈 수 있는 경우
- 내용상 범위 축소나 자료 보완이 필요한 경우

보정은 빨리 처리할수록 좋지만, 빠르기만 하고 정리가 안 되면 나중에 같은 문제가 다시 나온다. 따라서 제출 전 짧은 내부 메모를 남기는 편이 좋다.

### 내부 triage 기준

| 판단 질문 | 예 | 초기 판단 |
| --- | --- | --- |
| 문제를 고치면 살릴 수 있는가 | 설명 보완, 지정상품 조정, 증거 보강 | 대응 우선 |
| 일부만 살리고 일부는 버릴 수 있는가 | 코어 상품만 남기기 | 부분 대응 |
| 유지 비용이 사업 가치보다 큰가 | 핵심이 아닌 변형 표장 | 중단 검토 |
| 새 출원이 더 단순한가 | 다른 표기나 다른 범위가 더 명확 | 교체 검토 |

### 공고와 이의는 공개 단계로 읽는다

공고는 결과를 공개된 상태로 넘기는 구간이다. 즉, 내부적으로만 끝나는 단계가 아니라 제3자가 반응할 수 있는 창이 열린다. 그래서 공고 이후에는 단순 대기보다 모니터링과 대응 준비가 함께 가야 한다.

#### 공고의 의미

공고는 “이제 누군가가 볼 수 있다”는 뜻이다. 내부적으로는 다음을 준비한다.

- 이의 가능성
- 보완 자료
- 대리인과의 연락 순서
- 사업팀이 알아야 할 리스크 수준

공고를 받았다고 바로 긴 문서를 만들 필요는 없다. 먼저 어떤 권리가 공개 상태가 되었는지, 어떤 표장이 노출되었는지, 누가 볼 수 있는지를 정리하면 충분하다.

#### 이의가 들어오면 먼저 볼 것

이의는 감정적으로 반응할 사건이 아니다. 먼저 아래를 확인한다.

1. 이의 사유가 무엇인지
2. 상대가 어떤 권리를 근거로 하는지
3. 우리 표장과 상품 중 무엇을 겨냥하는지
4. 반박보다 조정이 나은 부분이 있는지

이의 단계에서는 전부를 지키려다 더 나빠질 수 있다. 그래서 방어가 필요한 부분과 내려놓을 부분을 분리해야 한다.

### 대응 옵션은 네 가지로 정리한다

실무적으로는 결과를 받은 뒤 다음 네 방향 중 하나로 정리하면 된다.

- **go**: 현재 표장과 범위로 밀고 간다
- **revise**: 표장, 상품, 설명을 손보면 살릴 수 있다
- **hold**: 내부 구조가 덜 정리되어 잠시 멈춘다
- **abandon**: 충돌이 크고 비용이 더 크면 접는다

이 구분은 결정을 빠르게 만들기 위한 것이다. 답을 길게 쓰는 것보다, 어떤 방향으로 갈지 먼저 통일하는 편이 낫다.

| 옵션 | 쓰는 때 | 내부 판단 포인트 |
| --- | --- | --- |
| go | 충돌이 약하고 설명이 충분한 경우 | 지금 그대로 가도 되는가 |
| revise | 일부 표현이나 상품만 조정하면 되는 경우 | 수정 후 일관성이 생기는가 |
| hold | 정보가 부족하거나 사업 방향이 흔들리는 경우 | 지금 서두르지 않는 것이 나은가 |
| abandon | 충돌이 크고 대체안이 더 좋은 경우 | 계속 붙잡는 게 손해인가 |

### 내부 의사결정 흐름은 짧게, 그러나 남겨야 한다

결정은 길 필요가 없다. 대신 남아야 한다. 심사 대응은 회의 기억으로 버티면 안 된다. 사건번호와 논리, 판단 이유가 남지 않으면 다음 통지에서 같은 질문을 다시 하게 된다.

#### 권장 흐름

1. 수령 직후: 통지 유형과 핵심 사유를 한 줄로 요약한다.
2. 당일 또는 조속히: 표장 유지, 범위 조정, 중단 가능성을 나눠 본다.
3. 대리인 검토 후: 제출 논리와 자료 목록을 확정한다.
4. 제출 후: 무엇을 냈는지, 무엇을 남겼는지 기록한다.
5. 후속 단계: 같은 문제의 재발 여부를 점검한다.

#### 판단 메모에 꼭 들어갈 항목

- 사건번호
- 대상 표장
- 대상 지정상품 또는 서비스
- 통지 유형
- 핵심 사유
- 선택한 대응 옵션
- 제출 여부와 제출일
- 다음 확인 시점

### 기록은 이메일이 아니라 사건 카드로 남긴다

이메일 체인만 있으면 나중에 누가 왜 그렇게 판단했는지 사라진다. 최소한의 사건 카드가 있어야 팀이 같은 사건을 다시 읽을 수 있다.

#### 사건 카드 템플릿

| 항목 | 기입 예시 | 기록 원칙 |
| --- | --- | --- |
| 사건번호 | CNIPA 접수번호 또는 통지 번호 | 원문 그대로 적는다 |
| 통지 유형 | 거절 / 보정 요구 / 이의 관련 | 하나로 분류한다 |
| 문제 요약 | 선행표장 충돌, 상품 범위 과다 등 | 한 문장으로 쓴다 |
| 대응 방향 | go / revise / hold / abandon | 결정어를 사용한다 |
| 제출 자료 | 의견서, 보정안, 내부 메모 | 무엇을 냈는지 적는다 |
| 내부 담당 | 법무, 브랜드, 사업, 대리인 | 책임 주체를 적는다 |
| 후속 과제 | 재검토, 추가 증거, 모니터링 | 다음 행동을 적는다 |

#### 기록 습관

- 문서 제목을 통일한다.
- 버전이 바뀌면 날짜를 붙인다.
- 대리인 의견과 내부 판단을 분리한다.
- 최종 제출본만 남기지 말고 검토본도 보관한다.

이 습관이 있어야 나중에 유사 사건이 왔을 때 재사용이 된다. 상표 실무는 한 번의 승부가 아니라 반복 운영이다.

### 첫 응답 체크리스트

통지서를 받으면 바로 할 일은 많지 않다. 먼저 다음을 확인한다.

- [ ] 통지 유형을 분류했다
- [ ] 핵심 사유를 한 줄로 적었다
- [ ] 영향받는 표장과 상품 범위를 표시했다
- [ ] 내부 담당자를 정했다
- [ ] 대리인과의 왕복이 필요한지 판단했다
- [ ] 제출 전 추가 자료가 필요한지 봤다
- [ ] 사건 카드 초안을 만들었다
- [ ] 다음 마감 또는 확인 시점을 적었다

#### 첫 응답 원칙

- 바로 결론부터 내리지 않는다.
- 먼저 살릴 수 있는 범위를 본다.
- 코어와 비코어를 분리한다.
- 기록 없이 구두로 끝내지 않는다.

## 제4장. 등록 후 유지와 사용 증거

등록이 끝이 아니다. 이 시점부터는 브랜드를 어떻게 계속 쓰고, 어떻게 증명하고, 어떻게 점검할지 관리해야 한다. 중국에서는 권리자 정보, 실제 사용 주체, 판매 화면, 포장, 인보이스, 플랫폼 이력, 대리인 커뮤니케이션이 따로 놀기 쉽다. 이 장은 그 흐름을 한 번에 묶는 운영 가이드다.

### 등록 후에 바로 정리할 것

등록 직후 가장 먼저 할 일은 사용 구조를 문서로 고정하는 일이다. 누가 상표를 소유하고, 누가 실제로 쓰고, 누가 화면과 패키지를 관리하고, 누가 외부 대리인과 연락하는지 흐리게 두면 나중에 증거가 있어도 설명이 약해진다.

#### 운영 원칙

1. 권리자와 사용자를 먼저 적는다.
2. 실제 노출 화면과 판매 자료를 먼저 모은다.
3. 파일명과 폴더 규칙을 먼저 정한다.
4. 분기별로, 반기별로 다시 본다.
5. 새 SKU나 새 채널은 등록상태와 함께 검토한다.

이 장의 목표는 법률 이론을 늘리는 것이 아니라, 브랜드팀이나 IP팀이 바로 돌릴 수 있는 관리 루틴을 만드는 데 있다.

### owner와 user를 분리해서 본다

중국 실무에서는 본사, 중국 법인, 제조 파트너, 유통사, 플랫폼 운영 대행사가 서로 다른 역할을 맡는다. 문제는 누가 실제 사용 흔적을 남기는지, 누가 외부에 설명 가능한 책임을 지는지다.

| 역할 | 운영상 의미 | 남겨둘 자료 |
| --- | --- | --- |
| owner | 권리를 보유하고 전체 방향을 통제하는 주체 | 등록 정보, 내부 권리대장, 위임 구조 |
| user | 실제로 표장을 붙여 쓰는 주체 | 포장, 라벨, 화면 캡처, 계약서 |
| operator | 판매 페이지, 캠페인, 유통 자료를 운영하는 주체 | 운영 메모, 배너 파일, 출시 일정 |
| counterparty | 대리인, 제조사, 유통사처럼 외부에서 협업하는 주체 | 계약서, 발주서, 커뮤니케이션 기록 |

#### 운영 규칙

- owner와 user가 다르면, 그 차이를 설명하는 메모를 따로 둔다.
- 동일 표장을 여러 법인이 같이 쓰면, 각자의 역할과 채널을 분리해 적는다.
- 사용 주체가 바뀌면 증거 파일도 같이 업데이트한다.
- 브랜드 운영 화면과 계약 구조가 어긋나지 않게 맞춘다.

### 무엇을 증거로 남길 것인가

증거는 많이 모으는 게 아니라, 나중에 바로 찾을 수 있게 남기는 게 중요하다. 등록 후 운영에서 자주 필요한 항목은 아래 정도면 충분하다.

| 증거 범주 | 예시 | 저장 포인트 |
| --- | --- | --- |
| 제품 노출 | 상품 사진, 라벨, 패키지, 사용설명서 | 출시 단위별로 묶기 |
| 판매 노출 | 플랫폼 페이지, 상세페이지, 검색 결과 화면 | 날짜가 보이게 저장 |
| 거래 흔적 | 인보이스, 발주서, 출고 자료 | 상품명과 표장이 같이 보이게 저장 |
| 운영 흔적 | 캠페인 배너, SNS 게시물, 전시회 자료 | 캠페인명과 채널을 같이 적기 |
| 권리 구조 | 라이선스, 유통 계약, 위임 메모 | 사용 권한의 흐름을 보여주기 |
| 집행 흔적 | 경고장, 신고 기록, 세관·플랫폼 대응 메모 | 사건번호와 조치 결과를 같이 저장 |

#### 좋은 증거의 기준

- 표장이 실제로 보인다.
- 날짜나 시점이 남는다.
- 어떤 상품인지 알 수 있다.
- 누가 사용했는지 연결된다.
- 화면만 있고 맥락이 없는 자료는 부족하다.

### Evidence Vault 파일링 SOP

증거 보관소는 복잡할 필요가 없다. 대신 규칙은 단순해야 한다. 새 자료가 들어오면 같은 방식으로 쌓여야 하고, 다른 사람이 봐도 다시 찾을 수 있어야 한다.

#### 권장 폴더 구조

```text
Brand_Evidence/
  2026/
    Q1/
      product/
      sales/
      platform/
      contracts/
      enforcement/
```

#### 파일링 규칙

- 파일명은 `브랜드_채널_날짜_내용` 식으로 통일한다.
- 원본과 가공본을 섞지 않는다.
- 화면 캡처는 가능하면 날짜가 보이게 저장한다.
- 계약서는 최종본과 핵심 조항 메모를 함께 둔다.
- 하나의 사건에는 사건 메모 1개를 만든다.

#### 입력 절차

1. 자료 수집
2. 중복 제거
3. 범주 태깅
4. 날짜 정리
5. owner/user 연결 표시
6. 폴더 저장
7. 사건 메모 업데이트

#### 사건 메모에 적을 것

- 브랜드명
- 상품 또는 채널
- 사용 주체
- 수집 날짜
- 자료 출처
- 다음 점검일

### 분기별 점검 루틴

분기 점검은 아직 쓰고 있는가를 확인하는 가벼운 체력검사다. 여기서 놓치면 나중에 증거가 끊긴다.

| 점검 항목 | 확인 질문 | 조치 |
| --- | --- | --- |
| 핵심 표장 노출 | 실제 판매 화면에 계속 보이는가 | 캡처 보강, 누락 채널 보완 |
| 중국어 표기 | 등록 내용과 다른 이름이 쓰이지 않는가 | 표기 통일 메모 발행 |
| 상품 범위 | 새 SKU가 기존 범위를 벗어나지 않는가 | 범위 재검토 |
| 사용자 구조 | 실제 user가 바뀌지 않았는가 | 계약·메모 동시 수정 |
| 자료 축적 | 최근 분기 자료가 충분히 쌓였는가 | 누락 자료 수집 |

#### 분기 점검 산출물

- 1페이지 점검 메모
- 누락 자료 목록
- 보완이 필요한 채널 목록
- 다음 분기 행동 항목

### 반기별 점검 루틴

반기 점검은 운영 구조를 다시 맞추는 단계다. 이때는 증거 자체보다, 권리자 정보와 외부 운영 구조가 현재 사업과 맞는지 본다.

#### 반기 점검 항목

- 권리자 정보가 현재 법인 구조와 맞는가
- 대리인 연락망이 살아 있는가
- 라이선스나 유통 구조가 실제 운영과 일치하는가
- 집행 이력과 경고 메모가 한 곳에 모여 있는가
- 새 지역, 새 채널, 새 상품군이 기존 포트폴리오와 충돌하지 않는가

#### 반기 점검 결과는 세 갈래로 나눈다

- 유지: 구조와 증거가 정상
- 보완: 자료는 있으나 정리가 부족
- 재설계: 사용자 구조나 채널 구조를 다시 짜야 함

### 자주 무너지는 지점

실무에서 자주 보이는 실패 패턴은 생각보다 반복적이다. 대부분은 등록보다 운영 관리가 느슨해서 생긴다.

#### 1. 화면만 있고 거래 흔적이 없다

프로모션 캡처는 많은데, 실제 판매나 유통을 보여주는 자료가 없다. 이런 경우는 보여주기 좋지만 설명력은 약하다.

#### 2. owner와 user가 뒤섞인다

본사 명의로 등록해 놓고, 실제 사용은 현지 법인이나 파트너가 하는데 설명 메모가 없다. 나중에 분쟁이 나면 누가 무엇을 했는지 다시 맞춰야 한다.

#### 3. 중국어 표기가 제각각이다

상세페이지, 포장, 계약서, 플랫폼에서 서로 다른 이름을 쓴다. 브랜드가 하나인데 운영 언어가 여러 개가 된다.

#### 4. 새 SKU가 그냥 추가된다

상품팀이 새 제품을 빨리 넣고, IP팀은 나중에 본다. 등록상 범위와 실제 판매 범위가 멀어지면 설명이 어려워진다.

#### 5. 증거가 폴더는 있는데 못 찾는다

자료는 많지만 파일명과 날짜가 뒤죽박죽이다. 실제로 필요한 날에는 찾는 데 시간이 더 든다.

### 유지 운영 체크리스트

아래 항목은 최소한으로 돌려야 한다.

| 체크리스트 | 완료 기준 |
| --- | --- |
| owner/user 구조 정리 | 역할 메모가 최신 상태다 |
| 핵심 표장 캡처 | 최근 분기 판매 화면이 저장돼 있다 |
| 제품·패키지 증거 | 최신 포장 또는 라벨 자료가 있다 |
| 거래 자료 | 인보이스 또는 출고 흔적이 있다 |
| 플랫폼 자료 | 주요 채널 화면이 저장돼 있다 |
| 계약 자료 | 사용 권한을 보여주는 문서가 있다 |
| 사건 메모 | 이슈와 조치가 한 장으로 정리돼 있다 |
| 다음 점검일 | 다음 분기와 반기 일정이 적혀 있다 |

#### 한 번에 끝내는 유지 루틴

- 이번 분기 새로 나온 상품을 적는다.
- 그 상품의 표장 노출 자료를 저장한다.
- 판매 화면과 거래 자료를 같이 묶는다.
- owner/user 변경 여부를 본다.
- 누락 자료를 채운다.
- 다음 점검일을 적는다.

## 제5장. 분쟁 대응과 집행

중국에서 상표 분쟁은 행정기관, 인민법원, 플랫폼, 세관이 서로 다른 속도와 장단점을 가진 채 병행될 수 있다. 그래서 단일 절차보다 사건의 유형에 맞는 출구를 먼저 고르는 판단이 중요하다.

### 분쟁이 생기면 먼저 유형을 나눈다

같은 침해처럼 보여도 첫 대응은 다르다. 아래 네 가지를 먼저 분리해서 본다.

| 분쟁 유형 | 흔한 신호 | 우선 확인할 것 |
| --- | --- | --- |
| 모조품·위조 유통 | 동일 또는 매우 유사한 표장, 낮은 품질, 다수 판매처 | 판매 화면, 주문 흔적, 물류 경로, 실물 사진 |
| 유사표장 출원·등록 | 공고 단계 진입, 식별력이나 유사성 문제 | 상대 출원 상태, 지정상품, 공고 여부 |
| 유통 파트너 갈등 | 계약은 있는데 사용 방식이 어긋남 | 라이선스/유통 계약, 실제 사용 주체, 판매 통제 |
| 국경 통과 이슈 | 수출입 단계에서 동일·유사 표장이 포착됨 | 세관 신고 가능성, 물품 식별 정보, 권리 증빙 |

핵심은 분쟁 이름이 아니라 지금 막아야 하는 대상이 무엇인지다. 온라인 판매인지, 출원 자체인지, 창고 물류인지, 통관인지부터 갈라야 한다.

### 채널은 네 개, 목적이 다르다

#### 선택 구조

| 채널 | 주된 목적 | 강한 점 | 약한 점 |
| --- | --- | --- | --- |
| 행정 | 빠른 현장 대응, 판매 차단, 초기 압박 | 속도, 접점이 넓음 | 손해배상 논리에는 한계가 있을 수 있음 |
| 법원 | 권리 판단의 정교화, 금지·배상 논리 | 기록과 판단의 무게 | 준비가 더 필요하고 속도가 느릴 수 있음 |
| 플랫폼 | 온라인 판매 차단, 상품·계정 조치 | 체감 속도, 반복 조치 관리 | 플랫폼별 형식에 맞는 자료가 필요함 |
| 세관 | 수출입 단계 차단, 물류 통제 | 국경 단계에서 막을 수 있음 | 물품 식별과 권리 정보 정리가 중요함 |

#### 실무 선택 기준

1. **지금 멈춰야 하는가**: 판매가 계속되면 피해가 커지는가
2. **증거가 충분한가**: 등록증, 사용 증거, 침해 화면, 실물 자료가 있는가
3. **상대가 누구인가**: 판매자, 제조자, 유통사, 플랫폼 계정, 통관 물품 중 누구를 겨냥할 것인가
4. **목표가 무엇인가**: 즉시 차단, 경고, 합의, 무효화 압박, 배상 중 어디에 가까운가

행정과 법원은 서로 대체재가 아니라 조합 대상이다. 온라인 판매를 먼저 멈추고, 그 다음에 권리 구조를 정교하게 가져가는 식의 순서가 자주 맞는다.

### 흔한 상황별 첫 행동

#### 1. 온라인에서 모조품이 보인다

- 판매 화면과 상품 상세를 즉시 보관한다
- 주문 정보, 판매자 정보, 링크, 계정명을 같이 모은다
- 가능하면 실물 확보 루트를 함께 만든다
- 플랫폼 신고와 법무 검토를 동시에 돌린다

#### 2. 비슷한 이름이 공고에 올라왔다

- 상대 출원 번호와 지정상품을 정리한다
- 우리 권리와의 충돌 지점을 한 문장으로 적는다
- 이의, 의견서, 무효 가능성을 내부에서 먼저 비교한다
- 사업부에는 "브랜드를 바꿀지"가 아니라 "어떤 결과를 원하는지"를 묻는다

#### 3. 유통 파트너가 자기 이름처럼 쓰고 있다

- 계약서와 실제 사용 화면을 나란히 놓는다
- 누가 패키지, 광고, 판매 페이지를 통제했는지 확인한다
- 상대방이 권한 범위를 넘었는지, 단순 운영 실수인지 가른다
- 경고장 전에 내부 사실관계를 먼저 잠근다

#### 4. 세관 단계에서 잡을 수 있을 것 같다

- 제품 사진, 포장 특징, 모델명, 로고 요소를 표준화한다
- 권리증과 권리자 정보를 최신 상태로 묶는다
- 물류팀과 법무가 같은 파일을 보게 만든다
- 통관 차단 가능성은 "있다/없다"보다 식별 가능성이 핵심이다

### 첫 대응 패키지는 작지만 정확해야 한다

첫 대응용 자료는 많을수록 좋은 게 아니다. 바로 쓸 수 있는 형태여야 한다.

#### 필수 구성

| 항목 | 넣을 것 |
| --- | --- |
| 권리 정보 | 등록번호, 권리자, 지정상품, 관련 표장 이미지 |
| 사용 증거 | 실제 사용 화면, 포장, 라벨, 인보이스, 판매 페이지 |
| 침해 증거 | 캡처, 링크, 계정명, 날짜, 실물 사진, 물류 흔적 |
| 관계 자료 | 계약서, 유통 구조, 위임 관계, 내부 메모 |
| 대응 메모 | 원하는 결과, 추천 채널, 다음 행동 |

#### 좋은 패키지의 기준

- 상대가 무엇을 했는지 한 번에 읽힌다
- 우리 권리가 왜 연결되는지 바로 보인다
- 플랫폼, 행정, 법원, 세관 중 어디에 올려도 재사용 가능하다
- 내부 승인 없이도 다음 단계 준비가 가능하다

### 내부에서는 먼저 올릴 사람을 정한다

분쟁 대응은 법무만의 일이 아니다. 최소한 아래 다섯 축이 같이 움직여야 한다.

- **법무**: 채널 선택, 문구, 권리 논리
- **사업/영업**: 고객·파트너 영향, 협상 여지
- **물류/SCM**: 통관, 반출입, 실물 식별
- **브랜드/마케팅**: 표장 사용 방식, 화면 정합성
- **현지 대리인**: 행정 절차, 제출 형식, 현장 접점

#### 내부 에스컬레이션 기준

아래 중 하나라도 맞으면 바로 상향한다.

- 브랜드가 핵심 매출 채널과 직접 연결된다
- 상대 행위가 반복적이거나 확산 중이다
- 증거가 흩어져 있어 나중에 못 모을 가능성이 높다
- 플랫폼, 행정, 소송 중 어느 쪽이든 일정 조율이 필요하다
- 현지 법인과 본사 판단이 다를 수 있다

에스컬레이션은 "큰일이라서"가 아니라 결정 지연 비용이 더 클 때 한다.

### 첫 응답 체크리스트

| 체크 | 질문 |
| --- | --- |
| 유형 분류 | 모조품, 출원, 파트너, 통관 중 어디인가 |
| 채널 선택 | 지금은 행정, 법원, 플랫폼, 세관 중 무엇이 맞는가 |
| 증거 확보 | 화면, 실물, 계약, 권리증빙이 한 묶음으로 있는가 |
| 목표 설정 | 차단, 경고, 합의, 배상 중 우선순위가 정리됐는가 |
| 내부 승인 | 법무, 사업, 물류, 본사가 같은 방향인가 |
| 후속 루트 | 다음 액션과 담당자가 정해졌는가 |

## 한 줄 결론

심사, 공고, 이의, 거절 대응은 각각 다른 이름의 같은 일이다. 통지를 받으면 먼저 유형과 사유를 나누고, go / revise / hold / abandon 중 하나로 정리한 뒤, 사건 카드와 증거 파일에 남겨야 한다. 등록 후에는 owner/user 구조와 증거를 같은 규칙으로 쌓고, 분쟁이 나면 유형과 채널을 먼저 고르면 된다. 중국 상표 실무에서 버티는 쪽은 말을 많이 하는 팀이 아니라, 빨리 정리하고 꾸준히 기록하는 팀이다.

## 부록. 실무 도구 모음

이 부록은 바로 복사해서 쓸 수 있는 작업 도구다. 읽고 끝내는 장이 아니라, 출원 전 판단, 증거 보관, 분쟁 첫 대응을 같은 방식으로 묶어 두는 장이다.

### A. 출원 전 질문지

출원 전에는 예쁜 설명보다 같은 판단 기준이 먼저다. 아래 질문에 답이 흐리면, 제출보다 정리가 먼저다.

#### A-1. 기본 정보

| 항목 | 작성 |
| --- | --- |
| 프로젝트명 |  |
| 검토일 |  |
| 작성자 |  |
| 최종 판단자 |  |
| 목표 시장 |  |
| 예정 제출 시점 |  |

#### A-2. 먼저 잠글 이름

- 핵심 표장만 먼저 낼지, 로고 결합까지 같이 갈지 정한다.
- 영문명과 현지명 중 중국에서 먼저 노출될 이름을 고른다.
- 소비자, 플랫폼, 영업팀이 각각 어떤 이름으로 부를지 적는다.
- 이름을 바꾸면 설명이 쉬워지는지, 더 어려워지는지 본다.

#### A-3. 중국어 명칭

- 내부에서 정할지, 현지 제안에 맡길지 정한다.
- 음역, 의역, 별칭 중 무엇이 현재 사업에 맞는지 고른다.
- 중국어 이름이 브랜드 핵심 의미를 유지하는지 본다.
- 중국어 이름을 별도로 등록할지 판단한다.
- 검색창, 상세페이지, 광고 소재에 넣어도 어색하지 않은지 확인한다.

#### A-4. 지정상품과 범위

- 지금 판매 중인 SKU를 적는다.
- 가까운 시점에 나올 상품을 적는다.
- 먼저 확보해야 할 클래스와 서브클래스를 적는다.
- 12~24개월 안의 현실적인 확장 범위를 적는다.
- 지금 범위가 너무 넓거나 너무 좁지 않은지 본다.

#### A-5. owner / user

- 권리자와 실제 사용자를 구분해 적는다.
- 본사, 중국 법인, 제조 파트너, 유통사 중 누가 무엇을 맡는지 적는다.
- 패키지, 판매 화면, 사용 증거를 누가 관리하는지 적는다.
- owner와 user가 다르면 그 이유를 설명한다.

#### A-6. 출원 경로

- 중국 단일 시장이 최우선인지 먼저 본다.
- 여러 국가를 함께 묶어야 하는지 본다.
- 현지 커스터마이징이 많이 필요한지 본다.
- 내부가 끝까지 운영할 수 있는 경로인지 본다.

#### A-7. 최종 판정

- **GO**: 지금 표장과 범위로 제출 가능

### B. Evidence Vault 운영 규칙

등록 후 증거는 많이 쌓는 게 아니라, 나중에 바로 꺼낼 수 있게 쌓는 게 핵심이다. 규칙은 단순해야 오래 간다.

#### B-1. 목적

- 표장 사용 증거를 같은 규칙으로 저장한다.
- 분기별, 반기별 점검 때 누락을 빨리 찾는다.
- 분쟁이나 심사 때 바로 재사용할 수 있게 둔다.

#### B-2. 무엇을 넣는가

- 제품 노출 자료
- 판매 화면 자료
- 거래·출고 자료
- 계약·권리 구조 자료
- 플랫폼 운영 자료
- 집행·경고 대응 자료
- 내부 메모와 점검 기록

#### B-3. 폴더와 파일명

연도, 분기, 범주 순서만 쓴다. 폴더를 늘리는 데 에너지를 쓰지 말고 찾기 쉽게 유지한다.

```text
Brand_Evidence/
  2026/
    Q1/
      product/
      sales/
      platform/
      contracts/
      enforcement/
      internal-memo/
```

파일명은 아래 순서로 고정한다.

```text
브랜드_채널_YYYY-MM-DD_자료유형_내용_v01
```

#### B-4. 파일명 원칙

- 날짜는 `YYYY-MM-DD`로 쓴다.
- 띄어쓰기는 쓰지 않는다.
- 버전은 `v02`, `v03`처럼 올린다.
- 캡처, 원본, 메모는 이름을 다르게 둔다.

#### B-5. 메타데이터

| 항목 | 적는 내용 |
| --- | --- |
| 브랜드명 | 실제 사용 중인 표장 |
| 상품 또는 채널 | 어떤 제품, 어떤 판매 경로인지 |
| 사용 주체 | owner, user, operator 중 누구인지 |
| 수집 날짜 | 자료를 확보한 날짜 |
| 자료 출처 | 플랫폼, 계약서, 촬영, 내부 전달 등 |
| 관련 사건번호 | 이의, 경고, 신고, 대응 번호 |
| 보관 위치 | 폴더와 파일명 |
| 다음 점검일 | 다음 분기 또는 반기 점검 날짜 |

#### B-6. 입력 순서

1. 자료를 모은다.
2. 중복 파일을 지운다.
3. 범주를 정한다.
4. 날짜를 맞춘다.
5. owner/user 연결을 표시한다.
6. 폴더에 저장한다.
7. 사건 메모를 갱신한다.

#### B-7. 점검 질문

분기 점검에서는 현재도 쓰고 있는지를 본다.

- 핵심 표장이 판매 화면에 남아 있는가
- 중국어 표기가 등록 내용과 다른가
- 새 SKU가 기존 범위를 벗어났는가
- 사용자 구조가 바뀌었는가
- 최근 분기 자료가 충분한가
- 누락된 채널이 있는가

반기 점검에서는 운영 구조 자체를 다시 맞춘다.

- 권리자 정보가 현재 법인 구조와 맞는가
- 대리인 연락망이 살아 있는가
- 라이선스와 실제 운영이 일치하는가
- 집행 이력과 경고 메모가 한곳에 모여 있는가
- 새 지역, 새 채널, 새 상품군이 기존 포트폴리오와 충돌하지 않는가

#### B-8. 흔한 실수

- 화면만 모으고 거래 흔적을 안 남긴다.
- owner와 user를 섞어 쓴다.
- 중국어 표기가 계속 바뀐다.
- 새 SKU를 나중에 생각한다.
- 폴더는 있는데 못 찾는다.

#### B-9. 운영 체크리스트

- [ ] owner/user 구조를 적었다
- [ ] 핵심 표장 캡처를 저장했다
- [ ] 제품·패키지 증거를 넣었다
- [ ] 거래 자료를 넣었다
- [ ] 플랫폼 자료를 넣었다
- [ ] 계약 자료를 넣었다
- [ ] 사건 메모를 업데이트했다
- [ ] 다음 점검일을 적었다

#### C-5. 즉시 조치

- 캡처와 링크를 먼저 잠근다.
- 실물 또는 식별 정보가 사라질 수 있으면 먼저 확보한다.
- 외부로 새는 커뮤니케이션을 막는다.
- 본사 승인 전 단독 메시지를 막는다.
- 플랫폼, 현지 대리인, 법무 검토를 동시에 돌린다.

#### C-6. 상향 기준

아래 중 하나라도 맞으면 바로 올린다.

- 브랜드가 핵심 매출 채널과 직접 연결된다.
- 상대 행위가 반복되거나 확산 중이다.
- 증거가 흩어져 있어 오늘 안에 잠가야 한다.
- 플랫폼, 행정, 법원, 세관 중 복수 채널 검토가 필요하다.
- 현지 법인과 본사의 판단이 다를 수 있다.
