# 제11장. 플랫폼, 도메인, 저작권, 부정경쟁 결합 대응

중국에서 브랜드 분쟁은 상표권만으로 닫히지 않는 경우가 많다. 플랫폼 판매 페이지, 도메인, 저작권 요소, 부정경쟁 논리가 함께 얽히면 채널별로 다른 대응 수단을 써야 한다. 이 장의 목적은 권리 유형을 나열하는 것이 아니라, 어떤 채널에서 어떤 권리 묶음이 실무적으로 먹히는지 정리하는 데 있다.

## 플랫폼·도메인 통합 triage 보드

온라인 침해 신호가 들어오면 채널별로 흩어 보기 전에 먼저 유형을 분류한다. 같은 운영 주체가 여러 채널에서 동시에 움직이는 경우가 많기 때문이다.

| 신호 유형 | 1차 확인 항목 | 예상 대응 경로 | 긴급도 |
| --- | --- | --- | --- |
| 플랫폼 유사 스토어명 | 표장 일치 여부, 판매 상품 범위 | 플랫폼 신고 | 높음 |
| 도메인에 브랜드명 포함 | 실제 사용 맥락, 연결 사이트 내용 | 도메인 이의 또는 경고장 | 보통 |
| 상세페이지 이미지·문구 복제 | 복제 요소, 정품 차이점 | 저작권 또는 부정경쟁 병행 | 보통 |
| SNS 계정명 + 판매 연결 | 팔로워 수, 판매 링크 연결 여부 | 플랫폼 신고 + 계정 이의 | 높음 |
| 플랫폼·도메인·SNS 동시 발견 | 운영 주체 동일 여부 확인 | 통합 대응 패킷 준비 | 최고 |

## 권리 묶음 선택 판단표

어떤 권리를 중심으로 대응할지는 침해 유형과 채널에 따라 달라진다. 한 번에 모두 주장하는 것보다 먼저 통하는 레버리지를 정하는 편이 속도와 효율 모두에서 낫다.

| 침해 유형 | 1차 권리 | 보조 권리 | 메모 |
| --- | --- | --- | --- |
| 동일·유사 표장 판매 | 상표권 | — | 패킷 완성도가 핵심 |
| 상세페이지 이미지 복제 | 저작권 | 상표권 | 저작물 요소 확인 필요 |
| 유사 도메인 사용 | 상표권 | 부정경쟁 | 사용 맥락 설명 필요 |
| 표기 혼동 유발 마케팅 | 부정경쟁 | 상표권 | 소비자 혼동 증거 필요 |
| 정품인 척 포장 변형 | 상표권 + 저작권 | 부정경쟁 | 통합 패킷 필요 |

### 온라인 침해는 채널별 규칙부터 읽는다

플랫폼 분쟁은 법리보다 제출 패킷과 속도가 먼저인 경우가 많다. 따라서 법무팀이 매번 새로 쓰기보다 공통 자료를 준비하고, 채널별 요구 사항만 바꾸는 편이 효율적이다.

실무적으로는 다음 자료를 공통 패킷으로 관리하는 편이 좋다.

- 권리자 정보
- 등록 자료
- 정품 식별 자료
- 침해 화면 캡처
- 연락 창구 정보

#### 플랫폼 신고 전에 먼저 확보할 캡처

플랫폼 대응은 나중에 법리를 길게 쓰는 것보다, 처음 캡처를 어떻게 남겼는지가 결과를 좌우하는 경우가 많다. 판매 페이지 주소, 상품명, 가격, 판매자 표시, 정품과 다른 핵심 지점을 한 묶음으로 남겨 두면 후속 제출이 훨씬 단순해진다.

특히 판매 페이지가 자주 바뀌는 채널이라면, 침해 표기와 판매 주체가 같이 보이는 화면을 먼저 확보하는 습관이 중요하다.

#### 채널 대응 맵

| 채널 | 먼저 보는 자료 | 실무 포인트 |
| --- | --- | --- |
| 플랫폼 | 판매 페이지, 판매자 정보, 정품 비교 자료 | 속도와 패킷 완성도가 중요 |
| 도메인 | 도메인 사용 맥락, 혼동 가능성, 상대방 행태 | 상표 외 맥락 설명 필요 |
| 콘텐츠/디자인 | 이미지, 설명문, 저작물 요소 | 저작권 또는 부정경쟁 논리 병행 가능 |
| SNS/홍보 | 계정명, 프로필, 판매 연계 여부 | 즉시성 높은 모니터링 필요 |

### 도메인 문제는 상표만으로 끝나지 않는다

도메인 분쟁은 이름이 비슷하다는 이유만으로 해결되지 않는다. 실제 사용 맥락과 악의성, 소비자 혼동 가능성, 사업상 연계성까지 같이 보게 된다. 따라서 도메인 대응은 등록증만 제출하는 방식보다, 현재 사용 상황과 상대방의 행태를 같이 정리하는 편이 설명력이 높다.

특히 중국어 표기와 영문 표기가 함께 쓰이는 브랜드는 어느 이름이 소비자에게 더 강하게 인식되는지도 같이 정리해야 한다. 도메인과 플랫폼 문제는 종종 같은 사건에서 동시에 보이므로, 별도 관리보다 묶어서 triage 하는 편이 낫다.

#### 도메인과 플랫폼 판매자가 같이 움직일 때 어떻게 볼까

같은 이름 구조가 도메인, 플랫폼 스토어, SNS 계정에 함께 나타나면 사건을 따로 떼기보다 하나의 운영 주체 가능성부터 보는 편이 좋다. 이렇게 묶어 보면 혼동 구조와 상대방 행태를 더 짧게 설명할 수 있다.

실무에서는 채널별로 권리 주장을 다르게 하더라도, 내부 사건 보드에서는 동일 주체 가능성 메모를 공통으로 두는 방식이 재사용에 유리하다.

### 저작권과 부정경쟁은 보조 레버리지다

상표권 하나로 설명이 약할 때는 제품 이미지, 상세페이지 구성, 카피 문구, 포장 요소 등 저작권 또는 부정경쟁 논리가 같이 작동할 수 있다. 핵심은 모든 권리를 다 주장하는 것이 아니라, 어떤 채널에서 가장 설명력이 높은 레버리지를 고르는 것이다.

#### 상세페이지 이미지 복제는 어떤 권리 묶음으로 볼까

상세페이지에서 표장 자체만이 아니라 제품 이미지, 구성 요소, 설명문이 함께 복제되면 상표만으로 설명하는 것보다 저작권 또는 부정경쟁 논리를 같이 검토하는 편이 자연스러울 수 있다. 중요한 것은 권리 수를 늘리는 것이 아니라, 채널 담당자가 이해하기 쉬운 설명 묶음을 고르는 일이다.

따라서 대응 메모에는 어떤 요소가 복제됐는지, 출처 혼동이 어디서 생기는지, 어느 권리 묶음이 가장 간단한지 짧게 정리해 두는 편이 좋다.

#### 결합 대응 체크리스트

- 상표권 자료만으로 충분한가
- 이미지나 설명문이 복제된 부분이 있는가
- 소비자 혼동이 제품 출처 외 요소에서도 생기는가
- 도메인과 플랫폼, SNS가 같은 운영 주체로 묶이는가
- 채널별로 가장 빨리 먹히는 자료 패킷은 무엇인가

### 이 장의 운영 메모

온라인 분쟁은 권리의 수보다 운영 구조가 중요하다. 상표, 도메인, 저작권, 부정경쟁을 따로 나누기보다, 어떤 채널에서 어떤 자료 묶음이 가장 빨리 통하는지를 기준으로 정리하는 편이 속도와 재사용성 모두에서 유리하다.

## channel-first evidence packet 표준안

온라인 사건은 권리 묶음보다 채널 묶음으로 정리하는 편이 재사용성이 높다. 같은 seller 또는 운영 주체가 플랫폼, 도메인, SNS를 동시에 쓰는 경우가 많기 때문이다.

| 패킷 항목 | 플랫폼 사건 | 도메인 사건 | 결합 사건 |
| --- | --- | --- | --- |
| 권리 자료 | 상표 등록 자료 | 상표 + 사용 맥락 메모 | 상표 + 저작권 + 혼동 메모 |
| 침해 캡처 | 판매 페이지, seller 정보 | landing page, redirect 화면 | 각 채널 캡처 묶음 |
| 정품 비교 자료 | 필수 | 보조 | 필수 |
| 운영 주체 추정 메모 | seller 정보 중심 | domain holder/연계 페이지 중심 | cross-channel 연결 메모 |
| 원하는 결과 | takedown, listing 삭제 | transfer/중지/경고 | 순차 대응 계획 |

### 패킷 표준안이 필요한 이유

채널마다 양식이 달라도, 내부에서는 같은 순서로 자료를 모아야 반복 대응 속도가 빨라진다. 법무팀이 매번 처음부터 설명문을 쓰지 않도록 packet order를 고정하는 것이 중요하다.

## domain + marketplace 동일주체 의심 체크포인트

플랫폼 seller와 도메인 운영자가 같은 주체인지 확인하면, separate incident로 보이던 사건이 하나의 혼동 구조로 정리되는 경우가 많다.

| 체크포인트 | 무엇을 보나 | 메모 |
| --- | --- | --- |
| 동일 표기 사용 | 영문/중문 브랜드명이 같은가 | 동일 naming pattern 확인 |
| 연락처 중복 | 이메일, 전화, WeChat 등 | seller/domain linkage 추정 |
| 이미지 재사용 | 같은 제품 사진 또는 배너 | 저작권 보조 논리 가능 |
| 링크 흐름 | 도메인에서 플랫폼으로 이동하는가 | 판매 funnel 확인 |
| 가격/상품군 유사성 | 동일 SKU 또는 유사 세트인가 | 출처 혼동 설명 강화 |

### 동일주체 메모를 남기는 방식

확정적 표현보다 "동일 운영 주체 가능성" 메모를 남기는 편이 안전하다. 어떤 채널에서 확인된 사실인지, 어떤 부분은 추정인지 구분해 두면 후속 법무 판단이 쉬워진다.

## unfair competition을 언제 전면에 세우나

상표권만으로 설명이 약하거나, 소비자 혼동이 제품명보다 판매 방식·표시 방식에서 강하게 생기는 경우에는 부정경쟁 논리가 전면으로 올라올 수 있다.

| 상황 | 부정경쟁 논리가 유용한 이유 | 보조 자료 |
| --- | --- | --- |
| 정품인 척 출처를 흐리는 판매 페이지 | 출처 혼동 설명이 직접적 | seller narrative, comparison capture |
| 도메인과 SNS에서 브랜드 identity를 흉내 냄 | 채널 전체 혼동을 설명 가능 | 프로필, landing page, buyer comments |
| 패키지·카피·페이지 구성을 통째로 복제 | 상표 외 요소의 종합 혼동 설명 가능 | before/after capture |

### 저작권과 부정경쟁을 같이 쓰는 판단 기준

둘을 같이 쓰는 것이 목적이 아니라, 채널 담당자가 가장 짧게 이해할 수 있는 설명을 고르는 것이 목적이다. 이미지 복제와 출처 혼동이 동시에 크면 두 논리를 병행할 수 있지만, 어느 하나만으로 충분하면 packet을 과도하게 늘릴 필요는 없다.

## 48시간 incident board

온라인 사건은 초기 48시간 안에 분류와 packet 방향을 정해 두면 후속 속도가 크게 달라진다.

| 시간대 | 해야 할 일 | owner |
| --- | --- | --- |
| 0-6시간 | 캡처 확보, 채널 분류, seller/domain linkage 메모 | 브랜드 보호 담당 |
| 6-24시간 | 권리 묶음 선택, packet 초안 정리 | 법무 |
| 24-36시간 | takedown / 경고장 / domain action 우선순위 결정 | 법무 + 사업팀 |
| 36-48시간 | 제출 또는 first notice 발송 | 법무 / 외부 대리인 |

### 48시간 안에 반드시 남겨야 하는 것

- 채널별 URL과 캡처 시점
- seller/domain 동일주체 가능성 메모
- 어떤 권리 묶음을 primary로 삼을지
- 정품 비교 포인트
- 다음 액션과 owner

## channel cleanup 이후 재발 모니터링

한 번 takedown이 됐다고 사건이 끝나는 것은 아니다. seller가 이름만 바꾸거나 도메인 redirect만 바꿔 다시 등장하는 경우가 있으므로, cleanup 이후 재발 모니터링 기준을 남겨 두는 편이 좋다.

| 재발 신호 | 다시 볼 것 | 메모 |
| --- | --- | --- |
| seller명만 바뀜 | 이미지·가격·SKU 유사성 | 동일주체 가능성 재확인 |
| 도메인 redirect 변경 | landing page history | 도메인 사건 연장선인지 확인 |
| SNS 계정만 남음 | 프로필명, 링크, bio | channel residue 판단 |

### cleanup 후 14일 체크

1. 이전 URL과 seller명을 재검색한다.
2. 동일 이미지/카피가 새 listing에 재등장하는지 본다.
3. domain redirect와 SNS link-in-bio를 다시 확인한다.
4. 재등장 시 기존 incident id에 이어 붙인다.

## owner map for online enforcement

| 역할 | 기본 owner | 왜 필요한가 |
| --- | --- | --- |
| 캡처 확보 | 브랜드 보호 담당 | 최초 packet 품질 좌우 |
| 권리 묶음 선택 | 법무 | 상표/저작권/부정경쟁 우선순위 결정 |
| 채널 제출 | 디지털 운영 또는 외부 대리인 | 속도와 양식 대응 |
| 재발 모니터링 | 브랜드팀 | seller/domain residue 추적 |

### 이 장의 quick note

온라인 사건은 법리보다 packet order와 owner map이 먼저다. 같은 seller가 다른 채널로 이동해도 같은 incident id로 이어 붙일 수 있게 기록 구조를 고정하는 편이 좋다.
