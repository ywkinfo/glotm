# 제5장. 출원서 작성 실무와 지정상품 설계

출원서 품질은 입력값에서 갈린다. 출원인 구조, 표장 버전, 지정상품 문구, 사용 주체 메모가 서로 맞지 않으면 심사뿐 아니라 등록 후 운영에서도 같은 문제가 반복된다. 그래서 출원서 작성은 문서 입력이 아니라 권리 구조를 고정하는 단계로 봐야 한다.

## 출원인과 실제 사용 주체를 먼저 맞춘다

중국 사업에서는 본사, 중국 법인, OEM 파트너, 유통사가 함께 움직이는 경우가 많다. 이때 출원인과 실제 사용 주체가 어긋나면 이후 증거 관리와 분쟁 대응에서 설명 비용이 커진다. 따라서 출원 전에 "누가 권리를 통제하고, 누가 실제로 쓰는가"를 한 문장으로 설명할 수 있어야 한다.

실무상 필요한 것은 법인명만 적는 것이 아니라 권리 통제 구조를 정리하는 것이다. 라이선스나 유통 구조가 예정돼 있다면, 출원서 설계 단계에서부터 그 사실을 내부 메모에 붙여 두는 편이 좋다.

## 표장 버전을 고정한다

같은 브랜드라도 문자표장, 결합표장, 중국어 표기형, 로고형이 섞이면 포트폴리오가 금방 복잡해진다. 그래서 출원서 작성 단계에서는 이번 출원이 "무엇을 보호하려는지"를 명확히 해야 한다. 영문 워드마크를 가져가는지, 중국어 표기를 따로 가져가는지, 결합표장까지 같이 갈지 먼저 정해야 한다.

이 선택은 디자인 선호가 아니라 운영 우선순위 문제다. 실제 포장과 플랫폼 상품 페이지, 대리점 자료에서 무엇이 가장 자주 쓰이는지 기준으로 보는 편이 안전하다.

## 제출 직전보다 먼저 만들어야 할 출원 데이터 패킷

출원서 작성이 늦어지는 이유는 법률 검토보다 입력 데이터가 흩어져 있기 때문인 경우가 많다. 그래서 실제로는 출원 직전보다 한 단계 앞에서 "이번 건의 입력값"을 하나의 패킷으로 묶는 편이 효율적이다.

### 출원 데이터 패킷 표준안

| 묶음 | 포함 항목 | 왜 필요한가 |
| --- | --- | --- |
| 권리자 정보 | 출원인 명칭, 주소, 연락창구 | 권리 통제 구조 고정 |
| 표장 정보 | 영문/중문/로고 버전, 파일명 | 어떤 버전을 보호하는지 명확화 |
| 상품군 정보 | 코어/인접 범위, 내부 SKU 메모 | 지정상품 설계와 연결 |
| 운영 메모 | 실제 사용자, 대리인, 후속 출원 계획 | 등록 후 사용·증거 구조 연결 |

이 패킷을 먼저 만들면 출원서 작성은 단순 입력이 되고, 나중에 같은 브랜드의 후속 출원이나 방어 표장 검토 때도 재사용이 가능하다.

## 지정상품은 넓게보다 설명 가능하게 쓴다

지정상품 설계는 넓게 적는다고 유리하지 않다. 실제 판매 SKU, 가까운 확장 계획, 향후 사용 증거로 설명 가능한 범위를 기준으로 잡아야 한다. 너무 넓으면 나중에 불사용 주장이나 운영 혼선이 생기고, 너무 좁으면 시장 확장 때 다시 공백이 생긴다.

2026-03-31 기준 fact verification log에서 확인한 내용에 따르면, 중국 출원에서는 `类似商品和服务区分表` 표준명칭과 공개된 수리가능 상품·서비스 명칭을 기준으로 분류를 설계하는 쪽이 안전하다. 비표준 항목을 쓰더라도 분류 원칙에 맞는 구체적 명칭이어야 하므로, 내부 마케팅 표현을 그대로 지정상품 문구로 옮기는 방식은 피하는 편이 좋다.

### 지정상품 설계표

| 범위 | 의미 | 기본 액션 |
| --- | --- | --- |
| Core | 현재 판매 중이거나 출시 확정인 범위 | 우선 포함 |
| Adjacent | 12~24개월 내 현실적인 확장 범위 | 선택적 포함 |
| Future | 아직 정의가 흐린 먼 확장 범위 | 일단 보류 |

## 수수료와 제출 채널은 예산표에서 먼저 잠근다

출원서 작성이 끝났다고 제출 준비가 끝난 것은 아니다. 비용 승인과 제출 채널이 정리되지 않으면 마지막 단계에서 다시 멈춘다.

2026-03-31 기준 fact verification log에서 잠근 `CN-FEE-001`에 따르면, CNIPA 상표 출원 수수료는 최근 공식 안내 기준으로 종이신청 300위안/류, 온라인신청 270위안/류를 기본으로 본다. 이 사실은 본문에서 단정적으로 길게 설명하기보다, 예산표와 제출 계획에서 "최근 공식 안내 기준"으로 관리하는 편이 안전하다.

### 예산·제출 메모

| 항목 | 이번 건에서 먼저 정할 것 |
| --- | --- |
| 제출 채널 | 온라인 제출 우선인지, 종이 제출 가능성도 열어둘지 |
| 예산 기준 | 최근 공식 안내 기준 수수료와 대리인 비용을 따로 적을지 |
| 승인 흐름 | 사업팀, 법무, 재무 중 누가 최종 제출 신호를 줄지 |

이 메모가 있으면 같은 브랜드 묶음을 여러 류로 나눠 제출할 때도 예산과 우선순위가 한 번에 정리된다.

## 제출 직전 점검할 것

출원서 제출 직전에는 문구보다 정합성을 본다. 출원인, 표장 버전, 지정상품, 내부 사용 구조가 하나의 이야기로 이어지는지 확인해야 한다. 이 정합성이 맞아야 이후 심사 대응이나 등록 후 증거 관리도 쉬워진다.

### 제출 전 체크리스트

- 출원인과 실제 사용 주체의 관계가 설명 가능한가
- 영문, 중국어, 결합표장 가운데 이번 출원의 범위를 고정했는가
- 지정상품이 실제 SKU 또는 계획된 서비스와 연결되는가
- 내부 메모에 출원 목적과 후속 출원 계획을 남겼는가
- 최근 공식 안내 기준 출원 수수료가 온라인 270위안/류, 종이 300위안/류라는 점을 예산표에 반영했는가
- 출원 데이터 패킷이 한 폴더 안에 모여 있고, 대리인에게 그대로 전달 가능한가
