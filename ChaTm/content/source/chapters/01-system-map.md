# 제1장. 중국 상표제도 구조와 판단 프레임

중국 상표제도는 한 번의 신청 절차가 아니라, 출원 설계와 등록 후 운영, 분쟁 대응이 하나의 운영 시스템으로 이어지는 구조로 봐야 한다. 중국 시장에서는 브랜드를 먼저 채택한 사실보다, 어떤 표장을 어떤 범위로 먼저 고정하고 이후 사용과 집행을 어떻게 연결하느냐가 더 직접적인 차이를 만든다.

## 중국 상표 실무를 구성하는 세 개의 층위

### 1. 출원과 심사

첫 번째 층위는 표장을 정하고, 검색과 분류를 거쳐, 출원 경로를 선택하고, 심사와 공고를 통과하는 단계다. 많은 기업이 이 단계를 상표 업무의 전부로 생각하지만 실제로는 운영의 출발점에 가깝다.

이 구간에서 결정되는 항목은 이후 거의 모두 재등장한다. 중국어 표기, 지정상품 범위, 출원인 구조, 대리인 커뮤니케이션 방식은 나중에 사용 증거, 불사용취소, 분쟁 대응에서도 그대로 쟁점이 된다.

### 2. 등록 후 유지와 사용 관리

두 번째 층위는 등록 후 실제 사용과 기록 관리다. 누가 어떤 표장을 어떤 채널에서 쓰는지 흐려지면 권리 자체보다 입증 구조가 먼저 무너진다. 따라서 중국 실무에서는 등록증만 보관하는 것이 아니라 사용의 이야기까지 함께 관리해야 한다.

이 단계의 핵심은 달력보다 구조다. 사용 주체, 거래 문서, 플랫폼 노출, 포장과 라벨, 대리인 업데이트, 집행 이력을 하나의 관리 체계로 묶어야 한다.

### 3. 분쟁 대응과 채널별 집행

세 번째 층위는 침해, 유사표장, 유통 갈등, 플랫폼 신고, 세관 협조, 행정·사법 대응을 포함한다. 중국에서는 이 경로들이 완전히 분리되지 않고 병행되므로, 사건이 발생한 뒤에 처음으로 자료를 모으기 시작하면 대응 속도가 크게 떨어진다.

실무적으로는 "어느 포럼이 옳은가"보다 "현재 확보된 자료와 사업 목적에 맞는 첫 번째 출구가 무엇인가"를 정하는 것이 중요하다.

## 중국에서 반복되는 오판

중국 진출 초기에 반복되는 오판은 거의 비슷하다.

- 한국 등록이나 해외 등록이 있으니 중국도 늦지 않다고 보는 판단
- 영문 브랜드만 확보하면 중국어 표기는 나중에 정리해도 된다는 판단
- 현지 유통 파트너나 제조 파트너가 브랜드 사용 구조를 자연스럽게 설명해 줄 것이라는 기대
- 등록만 되면 플랫폼, 세관, 분쟁 대응은 필요할 때 그때 정리하면 된다는 판단

이 오판들의 공통점은 상표를 "권리 취득 이벤트"로만 본다는 점이다. 중국에서는 상표를 운영 단위로 봐야 한다. 권리 취득, 사용 기록, 채널 통제, 사건 대응이 한 줄로 이어져 있기 때문이다.

## 처음부터 나눠 봐야 하는 브랜드 구조

중국 시장에서는 표장을 하나의 이름으로만 보면 금방 한계가 온다. 최소한 다음 네 묶음으로 나눠 보는 편이 안전하다.

| 묶음 | 질문 | 운영 메모 |
| --- | --- | --- |
| 핵심 표장 | 반드시 먼저 확보해야 하는가 | 사업팀과 법무가 공동 소유하는 우선순위 목록으로 관리 |
| 중국어 표기 | 소비자와 채널에서 실제로 어떻게 불릴 것인가 | 음역, 의역, 병행 사용 시나리오를 함께 검토 |
| 방어 표장 | 짧은 시간 안에 충돌 가능성이 높은 변형이 있는가 | 예산과 중요도에 따라 선별적으로 보호 |
| 사용형 표장 | 포장, 라벨, 플랫폼에서 실제로 보이는 버전은 무엇인가 | 출원본과 사용본 차이를 문서화 |

이 구조를 초기에 나눠 두면 출원 장에서 의사결정이 쉬워지고, 등록 후 단계에서 실제 사용과 증거를 연결하기도 쉬워진다.

## 관할 기관과 공식 출처를 읽는 기준

중국 상표 실무의 핵심 공식 접점은 `CNIPA`다. 제도 총괄 성격의 공지와 운영 정보는 [CNIPA](https://www.cnipa.gov.cn/)와 [SAMR](https://www.samr.gov.cn/)를 우선으로 보고, 국제출원 연계와 해외 경유 절차는 [WIPO](https://www.wipo.int/) 자료를 함께 읽는 편이 좋다. 다만 장별 실무에서 필요한 기한, 수수료, 절차명은 이 가이드 본문보다 사실 검증 로그를 먼저 보는 원칙을 유지해야 한다.

공식 출처를 읽을 때의 기준은 단순하다.

- 제도 설명은 본문이 안내하되
- 사실 확정은 사실 검증 로그가 담당하고
- 실제 실행은 최신 공식 페이지와 내부 사건 자료로 마무리한다

이 세 층을 섞지 않으면 원고는 안정적으로 유지되고, 시점 의존적인 정보는 별도로 업데이트할 수 있다.

## 이 장을 읽고 바로 정해야 할 것

### 조직 기준

- 중국 상표 업무를 본사 단독 업무로 둘지, 사업팀·유통팀·외부 대리인을 포함한 운영 과제로 둘지
- 사건 기록과 사용 증거를 누가 모으고 승인할지

### 브랜드 기준

- 영문 표장만 볼지, 중국어 표기와 변형 표장까지 같은 묶음으로 관리할지
- 핵심 표장과 방어 표장을 같은 우선순위로 볼지, 단계적으로 나눌지

### 운영 기준

- 출원, 등록 후 사용, 분쟁 대응을 각각 다른 폴더와 다른 담당자에게 흩어 둘지
- 아니면 하나의 브랜드 보드로 묶어 관리할지

## 실무 체크리스트

- 중국 시장용 핵심 표장, 중국어 표기, 방어 표장 구분이 되어 있는가
- 출원인과 실제 사용자, 유통·제조 관계가 최소한 문서화돼 있는가
- 사건 발생 시 바로 꺼낼 수 있는 등록증, 사용 증거, 채널 캡처, 내부 메모 체계가 있는가
- 기한과 수수료가 필요한 이슈는 본문이 아니라 사실 검증 로그와 공식 출처로 확인할 준비가 되어 있는가

## system map을 장표로 잠그는 방법

중국 상표 실무를 이해했다고 느껴도, 실제로는 system map이 머릿속에만 있으면 팀 간 handoff에서 곧바로 깨진다. 그래서 이 장의 내용을 장표처럼 남기는 편이 좋다.

| 축 | 최소 질문 | 연결되는 장 |
| --- | --- | --- |
| 브랜드 축 | 어떤 표장을 어떤 이름 구조로 볼 것인가 | 제2장 |
| 검색 축 | 어떤 충돌을 어디까지 볼 것인가 | 제3장 |
| route 축 | direct filing, Madrid, mixed route 중 무엇인가 | 제4장 |
| filing 축 | 누가 packet을 잠그고 제출하는가 | 제5장 |
| evidence 축 | 누가 사용 구조를 설명하는가 | 제7장, 제8장 |
| enforcement 축 | 어떤 채널에서 어떤 첫 출구를 고르는가 | 제10장, 제11장, 제12장 |
| governance 축 | 누가 모니터링하고 승인하는가 | 제13장 |

### system map이 필요한 이유

같은 회사 안에서도 브랜드팀은 제2장과 제3장만, 법무팀은 제5장과 제8장만, 사업팀은 제9장과 제12장만 보는 경우가 많다. system map을 두면 각 팀이 보는 장이 어떻게 이어지는지 설명하기 쉬워진다.

## 중국에서 먼저 잠글 5개의 보드

이 guide 전체를 한 번에 운영 구조로 바꾸려면 아래 5개 보드를 먼저 만드는 편이 좋다.

| 보드 | 목적 | 연결 장 |
| --- | --- | --- |
| naming / search board | 표기 후보와 충돌 범위 잠금 | 제2장, 제3장 |
| route memo | direct filing vs Madrid 판단 고정 | 제4장 |
| filing packet board | owner split, 제출 자료, handoff 정리 | 제5장 |
| evidence / cancellation board | 사용 구조와 취소 방어 준비 | 제7장, 제8장 |
| incident / governance board | 플랫폼·세관·계약 종료 사건 분류 | 제10장~제13장 |

### 보드가 많아 보여도 실제로는 같은 문법이다

각 보드마다 필요한 정보가 완전히 다른 것처럼 보이지만, 실제로는 표장, owner, 상품군, 채널, next action 다섯 항목이 반복된다. 그래서 문서 형식을 통일하면 운영 비용이 크게 줄어든다.

## buyer question 기준으로 system map을 읽는다

중국 실무를 법률 토픽 기준으로 나누면 사용자는 어디서 시작해야 할지 헷갈리기 쉽다. 반대로 buyer question 기준으로 나누면 길이 더 짧아진다.

| buyer question | 먼저 볼 층위 | 바로 이어볼 장 |
| --- | --- | --- |
| 지금 중국어 표기를 잠가야 하나 | 브랜드 구조 + 검색 층위 | 제2장, 제3장 |
| direct filing이 맞나 Madrid가 맞나 | route 층위 | 제4장, 제5장 |
| 등록 뒤 무엇을 미리 쌓아야 하나 | 유지와 사용 관리 층위 | 제7장, 제8장 |
| 위조/플랫폼/도메인 중 어디부터 막아야 하나 | 분쟁 대응 층위 | 제10장, 제11장 |
| OEM/총판 종료 뒤 흔적을 어떻게 거두나 | 운영 + governance 층위 | 제9장, 제13장 |

### buyer question을 먼저 쓰는 이유

제도는 복잡하지만 실제 의사결정은 대부분 짧은 질문으로 시작된다. 이 질문을 장과 연결하는 표가 없으면, 사용자는 검색 결과를 많이 보면서도 다음 액션을 못 정하는 경우가 많다.

## 중국 실무의 누적 리스크

중국 상표에서 비용이 커지는 이유는 단일 사건보다 누적 리스크 때문이다. 초기에는 작은 naming issue였던 것이 launch 지연, cancellation risk, 계약 종료 문제, 세관 incident로 이어질 수 있다.

| 초기 신호 | 누적되면 생기는 문제 | 예방 포인트 |
| --- | --- | --- |
| 중국어 표기 미확정 | search/filling drift | 제2장, 제3장에서 조기 잠금 |
| owner split 미정 | filing/evidence 공백 | 제4장, 제5장 memo |
| 사용 구조 미정리 | cancellation 방어 약화 | 제7장, 제8장 vault |
| seller/OEM 통제 약함 | channel drift와 종료 후 잔류 | 제9장, 제11장 |
| customs signal 분리 저장 | seller tracing 실패 | 제12장, 제13장 |

## 공식 출처를 보는 순서

공식 출처를 한꺼번에 다 읽기보다, 질문 유형에 따라 순서를 나누는 편이 좋다.

| 질문 유형 | 먼저 보는 것 | 그 다음 보는 것 |
| --- | --- | --- |
| 제도 구조 | CNIPA / SAMR overview | guide 본문 판단 프레임 |
| 국제연계 | WIPO Madrid 자료 | 제4장 route memo |
| 기한/수수료 | 사실 검증 로그 | 최신 공식 페이지 |
| 사건별 제출 | 공식 양식/가이드 | 내부 incident memo |

### 본문과 사실 검증 로그의 경계

본문은 판단 프레임을 제공하고, 사실 검증 로그는 시점 의존적 사실을 잠그는 역할을 한다. 이 둘을 섞지 않아야 guide가 오래 유지되고, freshness drift도 추적하기 쉬워진다.

## mature 준비 관점에서 이 장이 하는 역할

ChaTm을 mature 후보로 만들려면 제도 개요 장도 단순 설명을 넘어 search navigation과 governance language를 제공해야 한다. 그래서 이 장은 reader/search 관점에서 아래 역할을 맡는다.

| 역할 | 왜 중요한가 |
| --- | --- |
| 전체 system map 제시 | 초행 사용자 onboarding |
| buyer question과 장 연결 | search result 이후 다음 액션 안내 |
| 공식 출처 읽는 기준 제시 | 본문/로그 역할 분리 |
| 보드 문법 통일 | later chapters reuse |

### 이 장의 onboarding 메모

이 장을 읽고도 "그래서 우리 팀은 어디부터 봐야 하지"가 남아 있으면 아직 밀도가 부족한 것이다. system map은 개요가 아니라 다음 장으로 이동시키는 장치여야 한다.

## quick jump 메모

- naming/search가 비어 있으면 제2장, 제3장으로 간다.
- route choice가 흔들리면 제4장으로 간다.
- filing packet owner가 비어 있으면 제5장으로 간다.
- 취소/무효 걱정이 크면 제7장, 제8장으로 간다.
- seller/OEM/customs incident가 보이면 제9장부터 제13장까지 연결해서 본다.
