# 제8장. 비사용취소, 무효와 취소 리스크

중국에서 등록증을 받았다고 해서 권리가 자동으로 안정화되는 것은 아니다. 등록 후에도 비사용취소, 무효, 취소 리스크가 남아 있고, 실제 운영 구조가 약하면 권리를 오래 유지하기 어렵다. 이 장의 목적은 공격을 두려워하는 것이 아니라, 어떤 약점을 먼저 메워야 하는지 판단하는 데 있다.

### 왜 등록 후에도 권리가 흔들리는가

브랜드팀은 등록을 끝점으로 보는 경향이 있지만, 실무에서는 등록이 시작점인 경우가 많다. 실제 사용이 약하거나, 권리자와 사용 주체가 어긋나거나, 지정상품 범위가 너무 넓으면 공격 여지가 생긴다.

중국에서 특히 주의할 질문은 다음과 같다.

- 핵심 표장이 실제로 시장에 노출되고 있는가
- 등록 범위와 실제 판매 범위가 크게 어긋나지 않는가
- 사용 주체와 권리자가 설명 가능한가
- 공격이 들어왔을 때 바로 꺼낼 자료가 있는가

### 비사용취소 리스크는 증거 구조에서 갈린다

비사용취소 방어는 법리 설명보다 증거 구조 품질이 먼저다. 표장이 실제로 어떻게 쓰였는지, 어느 상품에 붙었는지, 누가 그 사용을 통제했는지 연결되지 않으면 방어력이 약해진다.

2026-03-31 기준 검증 로그에서 확인한 최소 구조는 이렇다. 비사용취소 통지가 오면 2개월 내에 사용 증거나 정당한 불사용 사유를 제출해야 하고, 일부 상품만 증명되면 그 부분과 유사상품 범위만 유지될 수 있다. 따라서 방어 파일은 브랜드 단위뿐 아니라 상품군 단위로도 꺼낼 수 있어야 한다.

#### 비사용취소 통지를 받으면 먼저 무엇부터 볼까

통지를 받자마자 가장 먼저 할 일은 논리 작성이 아니라 증거 범위를 자르는 일이다. 어느 표장인지, 어떤 등록 범위가 문제인지, 최근 3년 사용 흔적이 상품군별로 바로 나오는지부터 정리해야 한다.

이 순서를 놓치면 자료는 많아도 실제 제출 패킷은 약해질 수 있다. 그래서 초기 triage 단계에서 상품군별 증거 유무를 먼저 표시하는 방식이 유용하다.

#### 비사용취소 방어용 점검표

| 점검 항목 | 확인 질문 |
| --- | --- |
| 사용 흔적 | 최근 판매·노출 자료가 있는가 |
| 상품 연결 | 등록 범위와 실제 상품이 이어지는가 |
| 주체 설명 | 권리자와 사용 주체 관계가 설명되는가 |
| 파일 위치 | 채널/거래/제품 자료가 한 폴더에 모여 있는가 |

이 단계에서 중요한 것은 자료를 많이 쌓는 것보다, 공격이 왔을 때 어떤 자료를 어떤 순서로 꺼낼지 정리돼 있는가이다.

#### 일부 상품만 사용한 경우 어떻게 정리할까

모든 지정상품에 대해 같은 강도의 사용 자료가 있는 경우는 드물다. 이때는 전 범위를 막연히 주장하기보다, 실제 사용이 강한 상품군과 보조 설명이 필요한 상품군을 구분해 정리하는 편이 낫다.

현재 검증 로그 기준으로 일부 상품만 증명되면 그 부분과 유사상품 범위만 유지될 수 있으므로, 방어 파일도 상품군별 우선순위 메모와 함께 운영하는 편이 실무적이다.

### 무효와 취소는 포트폴리오 우선순위 문제다

모든 분쟁을 끝까지 다투는 것이 좋은 전략은 아니다. 어떤 표장은 핵심 브랜드이므로 끝까지 방어할 가치가 있고, 어떤 표장은 비용 대비 가치가 낮아 정리하는 편이 낫다. 따라서 무효나 취소 이슈는 사건별 대응이 아니라 포트폴리오 우선순위 기준으로 보아야 한다.

권장되는 내부 질문은 다음과 같다.

- 이 표장이 중국 사업에서 차지하는 가치가 얼마나 큰가
- 같은 역할을 하는 대체 표장이 이미 있는가
- 지정상품 범위를 줄여서 방어할 수 있는가
- 새 출원 또는 표장 조정이 더 싼가

### 공격을 받기 전에 준비할 방어 파일

가장 실용적인 방식은 등록 후부터 방어 파일을 따로 유지하는 것이다. 사건이 생긴 뒤 자료를 모으려 하면 누락과 설명 공백이 생기기 쉽다. 방어 파일은 거창할 필요가 없고, 핵심 표장별로 구조만 통일돼 있으면 충분하다.

#### 계열사나 유통사 사용 자료는 어떻게 묶을까

중국 사업에서는 권리자와 실제 판매 주체가 다를 수 있으므로, 계열사나 유통사 자료를 쓸 가능성을 미리 염두에 두는 편이 좋다. 이때 핵심은 자료의 양보다, 권리자와 사용 주체의 관계를 한 번에 설명할 수 있는 메모가 같이 붙어 있는가이다.

따라서 방어 파일에는 매출 자료나 페이지 캡처만 넣지 말고, 어떤 계약 또는 승인 구조 아래 사용이 이루어졌는지 짧은 설명 메모를 같이 두는 편이 안전하다.

#### 방어 파일 구성

- 권리 기본 정보
- 최근 사용 자료 묶음
- 상품/채널 설명 메모
- 사용 주체 설명 메모
- 계약 또는 승인 구조 자료
- 과거 분쟁 또는 경고장 이력
- 상품군별 유지 우선순위 메모

### 이 장의 운영 메모

비사용취소와 무효 리스크는 단지 "상대방이 공격할 수 있다"는 의미가 아니다. 내부 운영 품질이 낮으면 스스로도 포트폴리오를 설명하기 어려워진다는 뜻이다. 그래서 이 장의 핵심은 방어 기술보다 방어 가능한 파일 구조를 만드는 데 있다.
