# 제7장. 등록 후 유지와 사용 증거

등록이 끝나면 상표 업무가 종료되는 것이 아니라, 이제부터 실제 사용과 권리 유지의 기록 체계를 갖춰야 한다. 중국에서는 본사, 중국 법인, 유통 파트너, 제조 파트너가 동시에 움직이기 쉬워서 사용 주체와 증거 구조가 흐려지기 쉽다.

### 등록 후 첫 번째 질문은 누가 어떻게 쓰는가다

중국에서는 권리자와 실제 사용 주체가 여러 층으로 나뉠 수 있다. 이 구조가 흐리면 나중에 비사용취소, 분쟁 대응, 거래 구조 점검에서 설명이 어려워진다. 그래서 등록 직후에는 사용 자체보다 사용 구조를 먼저 명확히 해야 한다.

정리해야 할 핵심은 다음 네 가지다.

- 누가 표장을 실제로 붙이는가
- 누가 판매 페이지와 라벨을 운영하는가
- 계약상 사용 권한은 누구에게 있는가
- 같은 표장을 영문과 중국어로 어떻게 병행하는가

### 증거는 한 장씩 모으지 말고 이야기로 묶는다

중국에서 실제 사용과 권리 유지 흐름을 방어하려면, 채널별 화면 캡처, 거래 문서, 상품 실물 자료, 사용 주체를 설명하는 내부 메모를 하나의 이야기로 연결해야 한다. 핵심은 “증거가 있는가”보다 “누가 언제 어떤 표장을 어떤 상품에 썼는지 설명되는가”다.

#### 최소 증거 보존 패키지

| 구분 | 예시 | 왜 필요한가 |
| --- | --- | --- |
| 채널 자료 | 상품 페이지, 플랫폼 노출 화면 | 실제 시장 노출 설명 |
| 거래 자료 | 인보이스, 발주서, 출고 문서 | 사용 시점과 상품 연결 |
| 제품 자료 | 포장, 라벨, 실물 사진 | 표장 부착 형태 확인 |
| 운영 자료 | 내부 메모, 승인 기록, 계약서 | 사용 주체와 통제 구조 설명 |

가능하면 분기 단위 폴더를 기준으로 정리하고, 각 폴더에 핵심 표장명, 사용 주체, 채널, 캡처 시점, 관련 계약 문서를 같이 넣어 두는 편이 이후 비사용 주장이나 분쟁 대응 때 가장 실용적이다.

### 상품군별 evidence vault 카드를 만든다

증거를 많이 모아도, 어떤 상품군에 어떤 자료가 연결되는지 바로 꺼낼 수 없으면 실제 대응 속도는 느리다. 그래서 등록 후 운영에서는 표장별 폴더만이 아니라 상품군별 `evidence vault card`를 두는 편이 좋다.

| 카드 항목 | 예시 |
| --- | --- |
| 표장 | 영문 워드마크 / 중국어 표기 / 결합표장 |
| 상품군 | 코어 SKU, 인접 SKU |
| 사용 주체 | 본사, 중국 법인, 유통 파트너 |
| 대표 증거 | 상품 페이지, 인보이스, 포장 사진, 광고 캡처 |
| 마지막 갱신일 | 분기별 업데이트 날짜 |

이 카드가 있으면 비사용취소 통지가 왔을 때 "어떤 상품군 자료가 충분한가"를 바로 볼 수 있다. 일부 상품만 방어가 가능한 경우에도 의사결정이 빨라진다.

### 운영 달력은 등록증이 아니라 포트폴리오 기준으로 짠다

등록 후 관리에서 흔한 실수는 사건번호별 일정만 쫓는 것이다. 실제로는 브랜드 단위로 사용 상황과 증거 상태, 담당자, 대리인 커뮤니케이션을 같이 보아야 한다. 그래야 어느 표장이 살아 있고 어느 표장은 방어가 약한지 한눈에 보인다.

권장 점검 주기는 아래처럼 두는 편이 실무적이다.

#### 분기/반기 점검표

| 주기 | 확인 항목 |
| --- | --- |
| 분기 | 핵심 표장의 실제 노출 여부, 중국어 표기 일치 여부, 신규 SKU 반영 여부 |
| 반기 | 대리인 정보 최신화, 계약상 사용 구조 점검, 집행 이력 저장 위치 확인 |
| 연간 | 비사용취소 방어력 점검, 포트폴리오 정리 여부, 증거 폴더 품질 재검토 |

## 등록 후 양식과 이벤트를 같은 캘린더에 묶는다

등록 후 운영에서 또 하나 자주 빠지는 것은 "양식 제출 이벤트"와 "실제 사용 증거 점검 이벤트"가 서로 다른 달력에 놓이는 일이다. 중국에서는 사용 사실과 제출 대응이 연결돼 있으므로, 두 달력을 분리하면 결국 준비가 늦어진다.

### 권장 캘린더 항목

| 캘린더 항목 | 무엇을 같이 적나 |
| --- | --- |
| 등록일 | 권리자, 대리인, 핵심 표장 묶음 |
| 분기 증거 점검 | 상품군별 evidence vault 카드 상태 |
| 분쟁/플랫폼 대응 발생일 | 관련 증거 폴더 링크 |
| 비사용취소 통지 수령일 | 2개월 제출 창과 담당자 |

이 구조를 쓰면 등록 후 운영팀과 분쟁 대응팀이 같은 기준일을 본다. 결국 유지와 집행이 같은 자료를 공유하기 때문이다.

### 사용 증거는 분쟁 대응 자산이기도 하다

사용 증거는 등록 유지용 파일이면서 동시에 분쟁 대응용 자산이다. 플랫폼 신고, 세관 협조, 경고장, 행정 단속, 법원 분쟁 어느 쪽으로 가더라도 기초 자료는 크게 다르지 않다. 따라서 등록 후 운영팀과 집행 대응팀이 같은 증거 폴더 구조를 쓰는 편이 낫다.

2026-03-31 기준 fact verification log에서 확인한 최소 사실에 따르면, 비사용취소 절차에서는 통지 수령 후 2개월 내 사용 증거나 정당한 불사용 사유를 제출해야 하고, 일부 상품에 대한 증명만 되면 그 범위만 유지될 수 있다. 그래서 증거 폴더도 "전 범위를 막연히 증명"하는 구조보다 상품군별로 바로 꺼낼 수 있는 구조가 더 실무적이다.

#### 운영 체크리스트

- 표장별로 권리자와 실제 사용 주체를 구분해 적는다.
- 영문 표장과 중국어 표기의 실제 사용 버전을 같이 저장한다.
- 채널 자료, 거래 자료, 제품 자료, 운영 자료를 한 폴더에 묶는다.
- 플랫폼·세관·경고장 대응 이력을 사용 증거 폴더와 연결한다.
- 변동성이 큰 유지 관련 기한은 검증 로그에서 다시 확인한다.
- 비사용취소 통지가 오면 2개월 제출 창을 기준으로 상품군별 자료를 바로 꺼낼 수 있는지 점검한다.
- 상품군별 evidence vault 카드가 최신 상태인지 분기마다 확인한다.

### 이 장의 운영 메모

등록 후 운영이 잘 되는 조직은 "증거가 많다"보다 "증거가 설명 가능하다"에 가깝다. 중국에서는 사용 구조가 복잡해지기 쉽기 때문에, 실제 판매와 권리 통제, 계약 구조를 하나의 파일 체계로 묶는 것이 장기적으로 가장 효율적이다.
