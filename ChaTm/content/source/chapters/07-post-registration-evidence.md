# 제7장. 등록 후 유지와 사용 증거

등록이 끝나면 상표 업무가 종료되는 것이 아니라, 이제부터 실제 사용과 권리 유지의 기록 체계를 갖춰야 한다. 중국에서는 본사, 중국 법인, 유통 파트너, 제조 파트너가 동시에 움직이기 쉬워서 사용 주체와 증거 구조가 흐려지기 쉽다.

### 등록 후 첫 번째 질문은 누가 어떻게 쓰는가다

중국에서는 권리자와 실제 사용 주체가 여러 층으로 나뉠 수 있다. 이 구조가 흐리면 나중에 불사용취소, 분쟁 대응, 거래 구조 점검에서 설명이 어려워진다. 그래서 등록 직후에는 사용 자체보다 사용 구조를 먼저 명확히 해야 한다.

정리해야 할 핵심은 다음 네 가지다.

- 누가 표장을 실제로 붙이는가
- 누가 판매 페이지와 라벨을 운영하는가
- 계약상 사용 권한은 누구에게 있는가
- 같은 표장을 영문과 중국어로 어떻게 병행하는가

### 증거는 한 장씩 모으지 말고 이야기로 묶는다

중국에서 실제 사용과 권리 유지 흐름을 방어하려면, 채널별 화면 캡처, 거래 문서, 상품 실물 자료, 사용 주체를 설명하는 내부 메모를 하나의 이야기로 연결해야 한다. 핵심은 “증거가 있는가”보다 “누가 언제 어떤 표장을 어떤 상품에 썼는지 설명되는가”다.

#### 최소 증거 보존 패키지

| 구분 | 예시 | 왜 필요한가 |
| --- | --- | --- |
| 채널 자료 | 상품 페이지, 플랫폼 노출 화면 | 실제 시장 노출 설명 |
| 거래 자료 | 인보이스, 발주서, 출고 문서 | 사용 시점과 상품 연결 |
| 제품 자료 | 포장, 라벨, 실물 사진 | 표장 부착 형태 확인 |
| 운영 자료 | 내부 메모, 승인 기록, 계약서 | 사용 주체와 통제 구조 설명 |

가능하면 분기 단위 폴더를 기준으로 정리하고, 각 폴더에 핵심 표장명, 사용 주체, 채널, 캡처 시점, 관련 계약 문서를 같이 넣어 두는 편이 이후 비사용 주장이나 분쟁 대응 때 가장 실용적이다.

### 상품군별 evidence vault 카드를 만든다

증거를 많이 모아도, 어떤 상품군에 어떤 자료가 연결되는지 바로 꺼낼 수 없으면 실제 대응 속도는 느리다. 그래서 등록 후 운영에서는 표장별 폴더만이 아니라 상품군별 `evidence vault card`를 두는 편이 좋다.

| 카드 항목 | 예시 |
| --- | --- |
| 표장 | 영문 워드마크 / 중국어 표기 / 결합표장 |
| 상품군 | 코어 SKU, 인접 SKU |
| 사용 주체 | 본사, 중국 법인, 유통 파트너 |
| 대표 증거 | 상품 페이지, 인보이스, 포장 사진, 광고 캡처 |
| 마지막 갱신일 | 분기별 업데이트 날짜 |

이 카드가 있으면 불사용취소 통지가 왔을 때 "어떤 상품군 자료가 충분한가"를 바로 볼 수 있다. 일부 상품만 방어가 가능한 경우에도 의사결정이 빨라진다.

### 등록 후 90일 stabilization board

등록 직후 90일은 evidence 구조를 굳히는 데 가장 중요한 구간이다. 이 시점에 owner-user-linkage와 채널 자료 저장 규칙을 잠그지 않으면, 이후 분기 점검이 자료 수집이 아니라 자료 찾기로 바뀐다.

| 시점 | 반드시 끝낼 것 | owner |
| --- | --- | --- |
| 등록 후 30일 | owner-user-linkage 초안, 핵심 표장 버전 잠금 | IP |
| 등록 후 60일 | 채널별 evidence 폴더와 naming rule 고정 | 사업팀 + 운영 |
| 등록 후 90일 | 상품군별 vault card, 약한 상품군 표시 | IP + 사업팀 |

이 보드를 두면 등록 직후 운영이 “서류 보관”에서 “설명 가능한 evidence system 구축”으로 바뀐다.

### 운영 달력은 등록증이 아니라 포트폴리오 기준으로 짠다

등록 후 관리에서 흔한 실수는 사건번호별 일정만 쫓는 것이다. 실제로는 브랜드 단위로 사용 상황과 증거 상태, 담당자, 대리인 커뮤니케이션을 같이 보아야 한다. 그래야 어느 표장이 살아 있고 어느 표장은 방어가 약한지 한눈에 보인다.

권장 점검 주기는 아래처럼 두는 편이 실무적이다.

#### 분기/반기 점검표

| 주기 | 확인 항목 |
| --- | --- |
| 분기 | 핵심 표장의 실제 노출 여부, 중국어 표기 일치 여부, 신규 SKU 반영 여부 |
| 반기 | 대리인 정보 최신화, 계약상 사용 구조 점검, 집행 이력 저장 위치 확인 |
| 연간 | 불사용취소 방어력 점검, 포트폴리오 정리 여부, 증거 폴더 품질 재검토 |

## 등록 후 양식과 이벤트를 같은 캘린더에 묶는다

등록 후 운영에서 또 하나 자주 빠지는 것은 "양식 제출 이벤트"와 "실제 사용 증거 점검 이벤트"가 서로 다른 달력에 놓이는 일이다. 중국에서는 사용 사실과 제출 대응이 연결돼 있으므로, 두 달력을 분리하면 결국 준비가 늦어진다.

### 권장 캘린더 항목

| 캘린더 항목 | 무엇을 같이 적나 |
| --- | --- |
| 등록일 | 권리자, 대리인, 핵심 표장 묶음 |
| 분기 증거 점검 | 상품군별 evidence vault 카드 상태 |
| 분쟁/플랫폼 대응 발생일 | 관련 증거 폴더 링크 |
| 불사용취소 통지 수령일 | 2개월 제출 창과 담당자 |

이 구조를 쓰면 등록 후 운영팀과 분쟁 대응팀이 같은 기준일을 본다. 결국 유지와 집행이 같은 자료를 공유하기 때문이다.

## owner-user-linkage 표

등록 후 가장 자주 생기는 설명 공백은 “권리자는 본사인데 왜 판매자료는 다른 회사 이름으로 나오나”라는 질문이다. 그래서 evidence vault와 별도로 owner-user-linkage 표를 붙여 두는 편이 좋다.

| 항목 | 남겨야 할 메모 | 왜 중요한가 |
| --- | --- | --- |
| owner | 등록 명의자, 권리 보유 목적 | 권리 귀속 설명의 기준 |
| actual user | 판매, 플랫폼 운영, 포장 부착 주체 | 실제 사용 설명의 기준 |
| linkage | 라이선스, 유통계약, 계열사 관계 | 사용이 누구에게 귀속되는지 설명 |
| evidence location | 계약서, 인보이스, 포장, listing 폴더 | 제출 속도 향상 |

이 표가 있으면 등록 후 자료가 많아도 “누구의 사용인가”를 다시 풀지 않아도 된다. 불사용취소나 플랫폼 신고, 행정 대응 모두 이 연결고리를 반복해서 요구하기 때문이다.

### 채널별 evidence ops 표

중국에서는 같은 표장이라도 채널별로 남는 자료의 성격이 다르다. 그래서 evidence vault 카드와 별도로 채널 운영표를 두는 편이 실무적이다.

| 채널 | 기본 증거 | 약해지기 쉬운 지점 | 보강 메모 |
| --- | --- | --- | --- |
| 플랫폼 판매 | 상품 페이지, 주문 화면, 리뷰, 광고 캡처 | seller 계정과 owner 연결이 약함 | owner-user-linkage 메모 필수 |
| 오프라인 유통 | 포장 사진, 진열 사진, 출고 자료 | 시점이 불명확함 | 날짜 보이는 자료와 같이 보관 |
| OEM/제조 | 라벨, 납품 문서, SKU 자료 | 실제 판매 흔적이 약함 | 판매 채널 자료와 같이 묶기 |
| 파트너 판매 | invoice, 계약, listing | partner 사용이 owner 사용으로 안 읽힘 | 계약 linkage 메모 첨부 |

이 표가 있으면 2개월 제출 창에서 “어느 채널 증거가 강한가”를 상품군 판단과 같이 볼 수 있다.

## evidence vault를 incident memo와 같은 구조로 만든다

제7장의 evidence system이 강해질수록 제10장 집행 대응도 빨라진다. 반대로 등록 후 자료와 침해 대응 자료를 따로 관리하면, 플랫폼 신고나 행정 대응이 들어올 때마다 같은 포장 사진과 거래 메모를 다시 모으게 된다.

따라서 상품군별 evidence vault 카드는 "유지용 자료"이면서 동시에 "incident-ready 자료"가 되도록 설계하는 편이 좋다.

| vault 항목 | 유지 관점에서의 의미 | 집행 관점에서의 의미 |
| --- | --- | --- |
| 포장/라벨 사진 | 실제 사용 형태 설명 | 정품 식별 포인트 |
| 거래 자료 | 사용 시점과 상품 연결 | 유통 경로 입증의 출발점 |
| 플랫폼 노출 자료 | 중국 내 판매 흔적 | seller/listing 신고 자료 |
| owner-user-linkage | 권리 귀속과 실제 사용자 설명 | 파트너/계열사 사건 구분 |

이렇게 묶어 두면 불사용취소 2개월 대응과 침해 incident 72시간 대응이 같은 파일 체계를 공유하게 된다.

## 불사용취소 2개월 triage

불사용취소 통지가 오면 자료를 무작정 모으기보다, 2개월 안에 어떤 상품군을 실제로 방어할 수 있는지 먼저 나누는 편이 좋다.

| 구간 | 바로 해야 할 것 | owner |
| --- | --- | --- |
| 1주차 | 대상 표장, 대상 상품군, 최근 3년 자료 존재 여부 확인 | IP |
| 2~3주차 | 상품군별 강한 자료 / 약한 자료 분리 | IP + 사업팀 |
| 4~6주차 | owner-user 설명 메모와 계약 linkage 묶기 | IP |
| 제출 직전 | 유지 우선 상품군과 포기 가능 범위 정리 | 책임자 |

이 triage 표를 두면 방어 파일이 “자료 모음”이 아니라 실제 제출 패킷으로 바뀐다. 일부 상품군만 방어 가능한 사건에서도 빠르게 결론을 낼 수 있다.

### 2개월 제출 패킷 표준안

`CN-EVD-001`을 실제 운영 파일로 바꾸려면, “무엇을 낼 것인가”를 상품군별 제출 패킷 형태로 먼저 고정하는 편이 좋다. 그래야 owner-user-linkage 설명과 채널 자료, 거래 자료가 따로 놀지 않는다.

| 묶음 | 최소 포함 항목 | owner |
| --- | --- | --- |
| 권리 묶음 | 등록 정보, 표장 버전 메모, 권리자 설명 | IP |
| 사용 묶음 | 판매 페이지, 포장 사진, 라벨, 광고 자료 | 사업팀 |
| 거래 묶음 | 인보이스, 주문서, 출고 자료 | 운영/재무 |
| linkage 묶음 | 라이선스, 유통계약, 계열사 관계 메모 | 법무/IP |
| 제출 메모 | 어떤 상품군을 유지 우선으로 볼지에 대한 1페이지 메모 | 리드 |

이 패킷 구조를 먼저 잡아 두면 “자료는 있는데 제출 파일이 없다”는 상황을 줄일 수 있다.

### 부분 유지 판단표

`CN-EVD-001`에서 잠근 중요한 포인트는, 일부 상품에 대한 사용만 증명되면 그 범위만 유지될 수 있다는 점이다. 그래서 불사용취소 대응은 “전부 살릴 수 있나”보다 “어느 상품군까지 설명 가능한가”를 먼저 구분해야 한다.

| 상품군 상태 | 보통 보이는 증거 | 기본 판단 |
| --- | --- | --- |
| 강한 유지 가능 | 포장, 판매 페이지, 인보이스, owner-user linkage가 모두 있음 | 유지 우선 |
| 조건부 유지 가능 | 판매 흔적은 있으나 owner 설명이나 거래 자료가 약함 | 보강 후 유지 검토 |
| 유지 곤란 | 사용 시점 또는 상품 연결이 불명확함 | 부분 포기 또는 범위 축소 검토 |

이 표를 evidence vault 카드 옆에 두면, 2개월 제출 창 안에서 어떤 상품군부터 살릴지 빠르게 정리할 수 있다.

## 2개월 창에서 바로 버릴 상품군도 메모로 남긴다

부분 유지 판단이 어려워지는 이유는, 실제로는 포기할 상품군이 있는데도 회의 기록에는 모두 "추가 검토"로 남겨 두기 때문이다. 이렇게 되면 2개월 창 안에서 owner가 모든 자료를 다 모으려 하다가 핵심 상품군 방어까지 늦어진다.

| 상태 | 지금 남겨야 할 메모 | 기본 액션 |
| --- | --- | --- |
| 유지 우선 | 자료가 충분하고 사업 가치가 큼 | 제출 패킷 최우선 |
| 보강 후 판단 | 채널 자료는 있으나 linkage가 약함 | owner-user 설명 보강 |
| 포기 후보 | 사용 흔적이 약하고 사업 가치도 낮음 | 부분 유지 전제로 정리 |

이 메모를 남기면 부분 유지가 "실패의 기록"이 아니라, 2개월 창 안에서 코어 상품군을 먼저 살리기 위한 운영 판단으로 바뀐다.

### 사용 증거는 분쟁 대응 자산이기도 하다

사용 증거는 등록 유지용 파일이면서 동시에 분쟁 대응용 자산이다. 플랫폼 신고, 세관 협조, 경고장, 행정 단속, 법원 분쟁 어느 쪽으로 가더라도 기초 자료는 크게 다르지 않다. 따라서 등록 후 운영팀과 집행 대응팀이 같은 증거 폴더 구조를 쓰는 편이 낫다.

2026-03-31 기준 사실 검증 로그에서 확인한 최소 사실에 따르면, 불사용취소 절차에서는 통지 수령 후 2개월 내 사용 증거나 정당한 불사용 사유를 제출해야 하고, 일부 상품에 대한 증명만 되면 그 범위만 유지될 수 있다. 그래서 증거 폴더도 "전 범위를 막연히 증명"하는 구조보다 상품군별로 바로 꺼낼 수 있는 구조가 더 실무적이다.

#### 운영 체크리스트

- 표장별로 권리자와 실제 사용 주체를 구분해 적는다.
- 영문 표장과 중국어 표기의 실제 사용 버전을 같이 저장한다.
- 채널 자료, 거래 자료, 제품 자료, 운영 자료를 한 폴더에 묶는다.
- 플랫폼·세관·경고장 대응 이력을 사용 증거 폴더와 연결한다.
- 변동성이 큰 유지 관련 기한은 사실 검증 로그에서 다시 확인한다.
- 불사용취소 통지가 오면 2개월 제출 창을 기준으로 상품군별 자료를 바로 꺼낼 수 있는지 점검한다.
- 상품군별 evidence vault 카드가 최신 상태인지 분기마다 확인한다.

### 이 장의 운영 메모

등록 후 운영이 잘 되는 조직은 "증거가 많다"보다 "증거가 설명 가능하다"에 가깝다. 중국에서는 사용 구조가 복잡해지기 쉽기 때문에, 실제 판매와 권리 통제, 계약 구조를 하나의 파일 체계로 묶는 것이 장기적으로 가장 효율적이다.
