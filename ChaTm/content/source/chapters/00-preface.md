# 서문

중국 상표 업무는 단순히 출원서 하나를 잘 넣는 문제로 끝나지 않는다. 선출원 구조, 중국어 표기 설계, 서브클래스 리스크, 유통 파트너 통제, 플랫폼과 세관 대응이 한 시장 안에서 동시에 연결되기 때문이다. 그래서 중국은 한국 기업에게 중요한 시장일 뿐 아니라, 브랜드 운영 체계가 가장 빨리 시험되는 시장이기도 하다.

## 왜 ChaTm을 별도 운영 가이드로 보는가

한국이나 다른 국가에서 먼저 브랜드를 쓰고 있다는 사실만으로 중국에서 자동 보호가 생기지는 않는다. 중국 실무에서는 누가 어떤 표장을 어떤 상품에 어떤 채널로 먼저 고정하느냐가 이후 등록, 사용 증거, 분쟁 대응의 출발점이 된다.

또한 중국은 출원 단계와 등록 후 단계가 깔끔하게 분리되지 않는다. 출원 설계에서 중국어 표기와 지정상품을 잘못 잡으면, 그 문제는 심사 대응과 집행, 거래 구조 통제까지 이어진다. 반대로 초기 판단을 잘 해 두면 후속 운영 비용을 크게 낮출 수 있다.

이 가이드는 중국 상표를 "제도 설명"이 아니라 "운영 시스템"으로 읽게 만드는 데 목적이 있다. 따라서 장별 구성도 제도 소개보다 의사결정 흐름과 운영 체크포인트를 우선으로 잡는다.

## 이 가이드가 다루는 질문

- 중국 시장에서 먼저 선점해야 하는 표장은 무엇인가
- 영문 브랜드와 중국어 표기를 어떤 구조로 함께 설계해야 하는가
- 검색, 분류, 서브클래스 검토를 어떤 순서로 진행해야 하는가
- 직접출원과 마드리드 연계를 어떤 기준으로 나눌 것인가
- 출원 이후 심사, 공고, 이의, 거절 대응을 어떤 사건 관리 체계로 묶을 것인가
- 등록 후 사용 증거, 불사용취소, 거래 구조, 플랫폼, 세관 대응을 누가 관리할 것인가

## 읽는 순서와 활용 방식

처음부터 끝까지 순서대로 읽을 필요는 없다. 다만 중국 상표를 처음 설계하는 팀이라면 다음 흐름으로 읽는 편이 가장 효율적이다.

| 현재 상황 | 먼저 볼 장 | 다음으로 볼 장 |
| --- | --- | --- |
| 중국 진입 전 브랜드 설계 | 제1장, 제2장, 제3장 | 제4장, 제5장 |
| 출원 직전 또는 출원 중 | 제3장, 제4장, 제5장 | 제6장 |
| 등록 완료 후 운영 정비 | 제7장, 제8장, 제9장 | 제13장 |
| 침해나 유통 문제 대응 | 제10장, 제11장, 제12장 | 제13장, 제14장 |

핵심은 지금 당장 부딪히는 문제만 보더라도, 그 앞단과 뒷단을 함께 확인하는 것이다. 예를 들어 플랫폼 침해만 보더라도 실제로는 등록 상태, 사용 증거, 거래 구조, 세관 연계 가능성까지 같이 봐야 한다.

## 이 가이드를 읽을 때의 운영 원칙

### 본문보다 먼저 확인해야 하는 것

- 기한, 수수료, 공식 절차명, 기관명은 사실 검증 로그
- 실제 사건 대응에서 필요한 최신 공지와 양식은 공식 출처
- 내부 브랜드 구조와 사용 주체는 회사 내부 자료

### 본문에서 바로 가져가야 하는 것

- 판단 프레임
- 체크리스트와 비교표
- 역할 분담 기준
- 사건 기록 형식과 증거 패키지 구조

### 바로 믿지 말고 검증해야 하는 것

- 시점 의존성이 큰 숫자와 공식 기간
- 플랫폼별 세부 제출 요건
- 개별 사건별 심사 또는 집행 결과 예측

## ChaTm v1에서 달성하려는 기준

이 버전의 목표는 중국 상표의 모든 세부 사실을 백과사전처럼 담는 것이 아니다. 대신 한국 기업이 중국 시장에 진입하거나 이미 운영 중일 때, "다음으로 무엇을 정해야 하는가"를 빠르게 판단할 수 있는 기준 문서가 되는 것이다.

따라서 v1의 최소 기준은 다음과 같다.

- 장별로 실행형 체크리스트 또는 판단표가 있다.
- 출원, 등록 후 운영, 분쟁 대응이 서로 끊기지 않고 연결된다.
- 변동성 큰 사실은 사실 검증 로그를 거쳐 본문으로 올린다.
- 루트 `GloTm` 셸 `/china` 리더에서 탐색 가능한 수준의 섹션 밀도를 확보한다.

## 시작 전에 확인할 체크리스트

- 중국에서 실제로 사용할 표장 버전과 중국어 표기가 정리돼 있는가
- 누가 출원인이고 누가 실제 사용자일지 내부 합의가 있는가
- 핵심 상품과 채널, 제조 구조, 유통 구조가 최소한 문서화돼 있는가
- 기한이나 수수료가 필요한 이슈라면 사실 검증 로그와 공식 출처를 먼저 볼 준비가 돼 있는가

## 이 가이드의 기본 입력값

중국 상표 운영을 제대로 읽으려면 제도 설명 이전에 입력값이 있어야 한다. 아래 입력값이 비어 있으면 어느 장을 읽어도 실행력이 떨어진다.

| 입력값 | 왜 필요한가 | 비어 있으면 생기는 문제 |
| --- | --- | --- |
| 핵심 표장과 보조 표장 구분 | 어떤 표장을 먼저 잠글지 판단 | 출원과 방어 우선순위가 흐려진다 |
| 중국어 표기 후보 | 검색·경로 선택·채널 통제 기준 | naming drift가 반복된다 |
| launch market 우선순위 | direct filing / Madrid 판단 | route memo가 추상적으로 남는다 |
| owner-user 구조 | filing, evidence, contract 통제 | 불사용취소와 종료 대응이 약해진다 |
| 채널 구조 | 플랫폼, 총판, OEM, 수출 흐름 확인 | 어떤 장을 먼저 봐야 할지 애매해진다 |

### 최소 입력값을 1페이지로 정리한다

이 가이드에 들어가기 전 1페이지 브리프만 있어도 읽는 순서가 훨씬 명확해진다. 긴 문서가 아니라 아래 네 줄만 적어도 충분하다.

- 핵심 표장 / 중문 표기
- launch 순서와 중국의 우선도
- owner와 실제 사용자
- 핵심 상품군과 채널

## 중국 상표를 운영 시스템으로 보는 이유

중국 시장에서 상표는 등록증이 아니라 운영 인터페이스에 가깝다. 어떤 표장을 먼저 고정했는지, 그 표장이 어느 채널에서 어떻게 쓰이는지, 사건이 생겼을 때 누가 어떤 자료를 꺼내는지가 모두 연결되기 때문이다.

### 제도 설명만으로는 부족한 이유

많은 팀이 출원 기한과 절차명만 정리하면 충분하다고 생각한다. 하지만 실제 병목은 대부분 제도 지식이 아니라 handoff에서 발생한다. naming과 search가 filing으로 안 넘어가거나, filing과 evidence가 연결되지 않거나, contract 종료와 channel cleanup이 끊기면 등록이 있어도 운영은 흔들린다.

### 이 가이드가 해결하려는 handoff

| handoff | 왜 자주 끊기는가 | 이 가이드에서 회수하는 장 |
| --- | --- | --- |
| naming -> search | 표기 후보와 검색 범위가 분리됨 | 제2장, 제3장 |
| search -> route | 충돌 검토와 경로 선택 기준이 따로 놈 | 제3장, 제4장 |
| route -> filing | owner split과 packet owner가 비어 있음 | 제4장, 제5장 |
| filing -> evidence | 실제 사용 구조를 나중에 정함 | 제5장, 제7장 |
| contract -> enforcement | 유통/OEM 종료를 사건으로 안 봄 | 제9장, 제10장, 제11장 |
| customs -> governance | 공급망 incident가 board에 안 올라옴 | 제12장, 제13장 |

## 이 가이드를 읽는 팀별 시작점

같은 `/china` 리더라도 누구에게 필요한지에 따라 시작점이 다르다.

| 역할 | 먼저 볼 장 | 그 다음 장 |
| --- | --- | --- |
| 브랜드팀 | 제2장, 제3장 | 제4장, 제9장 |
| 인하우스 IP팀 | 제1장, 제4장, 제5장 | 제7장, 제8장, 제13장 |
| 사업팀 / 해외영업 | 제4장, 제9장, 제12장 | 제10장, 제13장 |
| 디지털/플랫폼 운영 | 제10장, 제11장, 제13장 | 제7장, 제9장 |
| 공급망 / 품질 | 제9장, 제12장 | 제10장, 제13장 |

### 팀별로 같은 용어를 쓰게 만든다

중국 상표 운영이 어려운 이유 중 하나는 팀마다 같은 상황을 다른 말로 부르기 때문이다. 브랜드팀은 표기 문제로 보고, 법무는 권리 문제로 보고, 사업팀은 launch 지연 문제로 본다. 이 가이드는 같은 incident를 같은 language로 적게 만드는 데도 목적이 있다.

## ChaTm Sprint 2에서 추가로 강화하는 것

Sprint 1이 중국어 표기, search, filing, evidence, enforcement의 핵심 6장을 잠갔다면, Sprint 2는 나머지 장의 밀도를 끌어올려 guide 전체를 같은 밀도로 읽히게 만드는 단계다.

| Sprint 2 대상 | 보강 포인트 |
| --- | --- |
| 제4장 | route memo, local-fit, owner split |
| 제8장 | cancellation heatmap, attack/defense 분기 |
| 제9장 | licensing control sheet, partner exit board |
| 제11장 | channel-first packet, domain linkage |
| 제12장 | customs escalation, SKU별 식별 패킷 |
| 제13장 | mature QA checklist, scorecard input |
| 제14장 | route memo/heatmap/exit appendix |

### 왜 서문에서도 Sprint 2 범위를 명시하나

사용자는 장별 원고를 개별적으로 읽기 전에, 이 guide가 지금 어느 단계까지 성숙했는지를 알고 싶어 한다. 서문에서 현재 강화 포인트를 말해 주면 Gateway 카드와 실제 본문 체감이 더 자연스럽게 맞는다.

## 루트 셸에서 이 가이드를 쓰는 방식

ChaTm은 독립 문서가 아니라 GloTm 포트폴리오의 growth lane guide다. 따라서 독립적인 중국 가이드로만 읽는 것보다, Gateway에서 어떤 buyer 질문을 먼저 중국으로 끌고 와야 하는지 보는 편이 더 유용하다.

### Gateway에서 `/china`로 들어올 때 기대해야 하는 것

- chapter card만 읽어도 현재 강화된 buyer 질문이 보여야 한다
- continue reading이 장 중간 hash로 복귀해야 한다
- search에서 decision table과 checklist가 section 단위로 잡혀야 한다
- Gateway card의 density, QA, maturity note가 실제 본문 체감과 맞아야 한다

### 이 서문을 읽고 나서 바로 해야 할 일

1. 현재 우리 팀의 역할이 위 표에서 어디에 해당하는지 고른다.
2. 입력값 1페이지 브리프를 만든다.
3. launch urgency가 있으면 제4장으로, naming/search가 비어 있으면 제2·3장으로, 운영 통제가 약하면 제9·13장으로 간다.

## 서문 운영 메모

서문은 소개글이 아니라 guide 사용법을 잠그는 페이지여야 한다. 이 장의 목적은 중국 상표가 어렵다는 인상을 주는 것이 아니라, 어떤 팀이 어떤 입력값을 가지고 어느 장부터 읽으면 되는지를 5분 안에 정리하는 데 있다.

## `/china` 첫 방문 5분 체크

| 5분 안에 확인할 것 | 왜 보나 |
| --- | --- |
| chapter card에서 무엇이 우선 장인지 | 현재 guide 강화 포인트 파악 |
| search에서 route / contract / customs가 잡히는지 | search density 체감 확인 |
| continue reading이 장 중간으로 돌아가는지 | 실무 재진입 속도 확인 |
| Gateway 카드 설명과 실제 본문이 맞는지 | buyer-facing trust 확인 |

### 서문을 다 읽지 않아도 되는 기준

입력값 브리프와 팀별 시작점이 정리됐으면 바로 해당 장으로 넘어가도 된다. 이 서문은 배경설명보다, 어떤 순서로 guide를 사용할지 정하는 용도에 더 가깝다.
