# 제6장. 심사, 공고, 이의와 거절 대응

중국 출원 이후 단계는 단순 대기가 아니라, 어떤 결과가 오더라도 내부에서 바로 의사결정을 내릴 수 있게 준비하는 구간이다. 이 장의 목적은 사건이 생긴 뒤 반응하는 것이 아니라, 심사와 공고, 이의, 거절 대응을 하나의 운영 흐름으로 묶는 데 있다. 2026-03-31 기준 `content/research/cn_tm_fact_verification_log.md`에서 잠근 최소 기한은 예비승인 공고일부터 3개월 이내 이의신청, 거절결정 통지 수령일부터 15일 이내 복심 신청이다.

### 심사 단계에서 먼저 읽어야 할 것

심사 결과를 볼 때 가장 흔한 실수는 통지 문구를 그대로 전달하는 것이다. 실제로는 어떤 표장이 충돌 기준인지, 어느 지정상품이 문제인지, 그리고 이 사건을 살릴 이유가 있는지 먼저 분해해야 한다.

중국 실무에서는 다음 세 질문이 먼저 나와야 한다.

- 이번 사건의 핵심 충돌이 문자, 발음, 의미, 상품 범위 중 어디에 있는가
- 브랜드 유지 우선순위가 높은 사건인가
- 지정상품 조정이나 보완 논리로 살릴 수 있는 사건인가

결국 심사 대응은 법률 문서 작성만이 아니라 포트폴리오 정리 작업이다. 살아남아야 할 표장은 끝까지 밀고, 덜 중요한 표장은 구조적으로 정리해야 한다.

### 공고와 이의는 공개 리스크 관리 구간이다

공고 단계는 제3자가 움직일 수 있는 첫 공개 구간이다. 내부적으로는 이의 가능성, 보완 자료, 대리인 커뮤니케이션 일정을 미리 정리해야 한다. 특히 중국어 표기와 영문 표장을 함께 운영하는 브랜드는 어느 표장에서 외부 반응이 먼저 오는지도 같이 봐야 한다.

이 단계에서 유용한 방식은 사건을 "기다리는 출원"이 아니라 "공격 가능성이 있는 공개 자산"으로 보는 것이다. 그러면 공고 직후 아래 자료를 바로 점검하게 된다.

- 상대방이 문제 삼을 수 있는 선행표장 후보
- 실제 사용 계획과 브랜드 유지 우선순위
- 제출 가능한 설명 자료 또는 내부 메모
- 사업팀이 이해할 수 있는 간단한 리스크 설명

공고 단계에서는 공고일과 내부 검토 시작일을 따로 적는 편이 좋다. 이의 기간이 3개월이라고 해도, 실제 조직에서는 외부 대리인 메일 수신일과 내부 승인 개시일 사이에 지연이 생기기 쉽기 때문이다.

#### 공고 단계 대응 테이블

| 질문 | 낮은 부담 | 높은 부담 |
| --- | --- | --- |
| 선행 충돌 | 범위가 멀거나 식별력이 약함 | 핵심 식별부와 직접 충돌 |
| 사업 중요도 | 보조 표장, 방어 표장 | 핵심 브랜드, 출시 예정 표장 |
| 상품 범위 | 좁은 상품군에서만 겹침 | 코어 상품군과 직접 겹침 |
| 대응 선택 | 모니터링 중심 | 즉시 이의/방어 준비 필요 |

### 사건 보드 SLA를 미리 고정한다

기한이 짧은 사건은 좋은 문장보다 좋은 핸드오프가 중요하다. 그래서 중국 사건 보드에는 최소한 아래 세 시점을 항상 따로 적는 편이 좋다.

| 시점 | 왜 따로 적는가 | 책임자 |
| --- | --- | --- |
| 공식 공고일/통지일 | 법정 기한 계산의 기준 | 대리인 또는 IP 담당 |
| 내부 1차 검토 시작일 | 실제 조직이 움직이기 시작한 시점 | 본사 IP / 사업팀 |
| 최종 의사결정일 | 이의, 복심, 포기, 수정 여부를 고정하는 시점 | 책임 임원 또는 리드 |

이 세 줄만 제대로 남겨도 "기한을 놓쳐서 대응이 늦었다"와 "기한 안이었지만 내부 결재가 늦었다"를 구분할 수 있다. 이후 같은 유형의 사건에서 병목을 고치기도 쉬워진다.

### 거절 대응은 사건별이 아니라 포트폴리오 기준으로 본다

거절이 왔을 때는 "이 사건을 이길 수 있는가"만 묻지 말고, "이 표장을 중국에서 계속 가져갈 가치가 있는가"를 같이 물어야 한다. 어떤 사건은 논리를 강화해 살릴 가치가 있고, 어떤 사건은 같은 비용이라면 표장을 수정하거나 상품 범위를 재설계하는 편이 낫다.

실무상 회의체에서는 최소한 다음 항목을 한 번에 정리하는 편이 좋다.

- 사건번호와 현재 단계
- 직접 사유와 충돌 상대
- 브랜드 유지 우선순위
- 보완 논리 가능성
- 지정상품 축소 가능성
- 포기 또는 재출원 대안

거절 대응에서도 같은 원칙이 적용된다. 15일이라는 짧은 창은 "문서에 찍힌 날짜"보다 "조직이 실제로 움직일 수 있는 시간"이 더 중요하므로, 수령일과 첫 내부 검토일을 따로 남겨야 대응 품질이 안정된다.

#### 거절 대응 회의 체크리스트

- 이 사건이 핵심 표장인지, 방어 표장인지 먼저 구분한다.
- 충돌 상대 표장의 지정상품과 자사 범위를 같은 표로 비교한다.
- 문구 보완, 상품 축소, 표장 수정 중 무엇이 가장 비용 효율적인지 판단한다.
- 대리인 의견을 내부 사업 우선순위와 분리해서 그대로 받지 않는다.
- 기한은 반드시 검증 로그와 사건 보드 양쪽에 기록한다.

## 공고 후 72시간 액션

공고나 이의 가능성이 보이는 사건은 첫 72시간 안에 운영 자료를 잠가 두는 편이 좋다. 시간이 지나면 법률 논점보다 내부 승인 지연이 병목이 되기 쉽다.

| 시간대 | 해야 할 일 | 산출물 |
| --- | --- | --- |
| 0~24시간 | 공고일, 대상 표장, 코어 상품군 기록 | 사건 보드 1차 등록 |
| 24~48시간 | 브랜드 중요도와 launch 영향 점검 | 사업 영향 메모 |
| 48~72시간 | 방어 범위, 양보 범위, 대리인 질의 정리 | opposition/response 브리프 |

## 공고 후 3개월 운영 캘린더

공고 후 3개월이라는 기한은 길어 보이지만, 실제 조직에서는 외부 대리인 메일 수신, 내부 사업 검토, 임원 승인, 제출 문안 확정이 순차로 움직여 체감 시간이 짧아진다. 그래서 공고 직후에는 법정 기한만 적지 말고 내부 캘린더를 따로 잘라 두는 편이 낫다.

| 시점 | 반드시 끝내야 할 것 | owner |
| --- | --- | --- |
| 공고 후 1주 이내 | 사건 중요도 분류, 선행표장 1차 검토 | IP |
| 공고 후 3주 이내 | 사업 영향, 양보 가능 범위, 대리인 질의 | 사업팀 + IP |
| 공고 후 6주 이내 | 대응 초안, 내부 승인 라인 확인 | IP + 리드 |
| 공고 만료 전 | 최종 fight / narrow / watch 결정 | 책임자 |

이 캘린더를 남기면 “법정 기한은 남았는데 조직 시간이 없다”는 상황을 줄일 수 있다. 공고 이후 단계는 법률 작업보다 내부 응답 속도 관리에 가깝다.

## 상대방 이의 대응 자료표

중국에서는 이의 가능성이 보이는 사건이라도 어떤 자료가 있느냐에 따라 대응 강도가 크게 달라진다.

| 자료 유형 | 왜 필요한가 | 없을 때의 대안 |
| --- | --- | --- |
| 브랜드 유지 우선순위 메모 | 이 사건을 왜 살려야 하는지 설명 | 보조 표장으로 낮춰 판단 |
| 출시 일정과 채널 메모 | 사업 영향 크기 확인 | launch 영향이 약하면 양보 검토 |
| search 단계 비교표 | 충돌 구조를 빠르게 복기 | 대리인 재질의 필요 |
| 실제 사용 준비 자료 | 표장 유지의 실익 설명 | 재출원/수정안 검토 |

자료표의 목적은 문서를 많이 쌓는 데 있지 않다. 이의나 거절이 왔을 때 어떤 자료가 이미 있고 무엇이 비어 있는지 한눈에 보게 만드는 데 있다.

## 거절 통지 triage 메모

거절이 오면 아래 형식으로 한 페이지 메모를 먼저 만드는 편이 좋다.

| 항목 | 적어 둘 내용 |
| --- | --- |
| 사건 단계 | 심사, 공고, 거절, 복심 가능 여부 |
| 핵심 사유 | 문자, 발음, 의미, 상품 범위 중 무엇이 문제인지 |
| 브랜드 중요도 | core / support / defensive |
| 수정 가능성 | 상품 축소, 보완 논리, 재출원 중 어느 쪽이 유리한지 |
| 최종 제안 | fight / narrow / refile / drop |

## 15일 복심 triage

거절결정 통지 수령 후 15일이라는 창은 아주 짧다. 그래서 복심 여부는 긴 설명보다 즉시 분기표로 나누는 편이 좋다.

| 신호 | 복심 쪽 | 재설계/포기 쪽 |
| --- | --- | --- |
| core 표장 여부 | 반드시 살릴 가치가 큼 | 대체 가능 |
| 충돌 강도 | 논리 보완 여지 있음 | 핵심부 충돌이 너무 큼 |
| 지정상품 조정 가능성 | 조정 후 방어 가능 | 조정해도 실익이 낮음 |
| 출시 일정 영향 | 지금 유지가 필요 | 일정 유연성 있음 |

15일 triage의 목적은 모든 사건을 싸움으로 끌고 가는 것이 아니라, 어떤 사건은 빠르게 접고 어떤 사건은 바로 자원을 집중할지 정하는 데 있다.

### 복심과 재출원 사이의 분기 기준

거절이 오면 조직은 자주 "끝까지 다툴 것인가, 다시 설계할 것인가"에서 시간을 잃는다. 그래서 회의 때는 아래 분기표를 같이 보는 편이 좋다.

| 질문 | 복심 쪽으로 기울 때 | 재설계/재출원 쪽으로 기울 때 |
| --- | --- | --- |
| 핵심성 | 중국에서 반드시 살려야 하는 코어 표장 | 방어 표장 또는 대체 가능한 표장 |
| 충돌 강도 | 논리 보완 여지가 있음 | 지정상품 또는 표장 핵심부 충돌이 큼 |
| 사업 일정 | 출시 일정상 유지 필요 | 조금 늦춰도 사업 영향이 작음 |
| 후속 비용 | 한 번 다투는 편이 전체 비용이 낮음 | 수정 후 다시 가는 편이 단순함 |

이 표가 있으면 대리인 의견과 사업 우선순위를 같은 눈금으로 볼 수 있다. 결국 복심 여부는 법률 승산만이 아니라 포트폴리오 가치 판단이기 때문이다.

### 사건 기록 보드는 반드시 표준화한다

심사·이의·거절 대응은 이메일 체인만 남겨두면 나중에 의사결정 근거가 사라진다. 사건번호, 상대 표장, 제출 논리, 마감기한, 다음 액션을 표준 형식으로 남겨야 같은 실수를 반복하지 않는다.

권장 형식은 간단하다. 사건마다 한 줄 요약과 다음 액션이 바로 보이는 보드를 두고, 상세 자료는 그 밑에 연결한다. 핵심은 "누가 언제 무엇을 결정했는가"가 남는 것이다.

- 사건번호
- 표장명
- 단계
- 직접 사유
- 마감기한
- 현재 결정
- 다음 액션
- 담당자

### 이 장의 운영 메모

심사와 공고, 이의, 거절 대응은 서로 다른 절차로 보이지만 내부 운영에서는 하나의 포트폴리오 필터로 다루는 편이 낫다. 중국에서 중요한 것은 법률 문구의 화려함보다, 어느 사건을 살리고 어느 사건을 접을지 빠르게 정리하는 판단 속도다.
