# 제9장. 라이선스, 유통, OEM/제조 구조 통제

중국 상표 운영에서 계약 구조는 별도 부속 문제가 아니다. 실제로는 권리자, 사용 주체, 제조 주체, 유통 채널이 나뉘는 순간부터 상표 리스크가 생긴다. 이 장의 목적은 라이선스, 유통, OEM/제조 구조를 계약 검토가 아니라 브랜드 통제 문제로 보는 데 있다.

## 브랜드 사용 구조 설계 체크리스트

계약을 시작하기 전에 브랜드 사용 구조가 어느 수준으로 정해져 있는지 확인한다. 이 체크리스트가 절반 이하로 채워지면 계약 조항보다 구조 정리가 먼저다.

| --- | --- | --- |
| 권리자와 실제 사용자 관계 문서화 | | |
| 허용되는 표장 버전 목록(영문/중문/로고) | | |

## OEM 브랜드 통제 구조 설계표

OEM 공장에 브랜드 자산을 넘기기 전에 통제 범위를 명시해야 한다. 아래 구조를 기준으로 각 항목의 허용 범위를 적어 두면 나중에 설명이 끊기지 않는다.

| 항목 | 통제 방식 | 담당 |
| --- | --- | --- |
| 로고 파일 버전 | 승인된 버전만 공식 전달 | 브랜드팀 |
| 포장 도안 | 최종 승인본 발행 번호 관리 | 브랜드팀 + 품질팀 |
| 라벨 문구 | 발행 전 법무·마케팅 검토 | 법무 |
| 재발주 시 자료 | 동일 버전 재확인 절차 | 구매팀 |
| 공장 하청 구조 | 파일 전달 범위 계약서 명시 | 법무 |

### 공장이 직접 서브 공급업체에 파일을 넘기는 경우 대응 기준

OEM 공장이 포장 인쇄를 다른 업체에 맡기는 경우, 로고 파일과 도안이 통제 범위를 벗어날 수 있다. 계약서에는 파일 전달 범위와 재하청 허용 여부를 명시하고, 필요시 서브 공급업체도 같은 보안 기준을 따르게 해야 한다.

### 계약 구조를 먼저 지도처럼 그린다

실무에서 자주 생기는 문제는 문서상 권리자와 실제 사용 구조가 다르다는 점이다. 본사, 중국 법인, 총판, 제조 파트너, 전자상거래 운영사가 각자 다른 역할을 가지면, 상표를 누가 쓰고 누가 통제하는지부터 흔들릴 수 있다.

그래서 계약을 검토할 때는 조항을 먼저 보는 것이 아니라 구조도를 먼저 그리는 편이 낫다.

- 권리자는 누구인가
- 실제 사용 주체는 누구인가
- 제조와 포장은 누가 담당하는가
- 판매 채널은 누가 운영하는가
- 종료 시 표장 사용을 누가 멈출 수 있는가

### 라이선스는 사용 허락보다 품질 통제가 중요하다

라이선스 구조에서 흔한 오해는 사용 허락만 있으면 된다는 생각이다. 실제로는 어떤 품질 기준 아래 어떤 버전의 표장을 누가 어떻게 쓰는지 관리할 수 있어야 한다. 그렇지 않으면 권리 통제 설명이 약해지고, 분쟁이나 종료 시 대응도 어려워진다.

#### 중국 총판이 상세페이지를 운영할 때 무엇을 통제할까

총판이나 운영 대행사가 온라인 상세페이지를 직접 관리하면, 제품 설명문과 이미지뿐 아니라 표장 표기 방식도 현장에서 바뀌기 쉽다. 그래서 라이선스나 유통 계약에서는 판매권만 줄 것이 아니라, 어떤 명칭과 로고 버전을 어떤 채널에서 쓸 수 있는지까지 보이는 형태로 남겨야 한다.

실무적으로는 상품 상세페이지, 배너, 프로필명 같은 채널 자산을 같은 승인 루틴 안에 넣는 편이 재사용성과 통제력 모두에서 낫다.

#### 라이선스 검토 체크리스트

| 항목 | 확인 질문 |
| --- | --- |
| 표장 버전 | 영문/중문/로고 중 무엇을 허락하는가 |
| 상품 범위 | 어떤 SKU와 채널에 한정되는가 |
| 품질관리 | 샘플 승인, 라벨 승인, 마케팅 승인 절차가 있는가 |
| 종료 통제 | 종료 후 재고, 플랫폼 페이지, 포장물 처리가 정해져 있는가 |

#### OEM 공장에 로고 파일을 넘길 때 점검할 것

OEM 구조에서는 공장에 전달하는 로고 파일, 포장 도안, 라벨 문구가 실제 시장 사용의 시작점이 되기 쉽다. 따라서 제조 발주와 별도로, 어떤 파일 버전을 누가 승인했고 어디까지 사용할 수 있는지 기록을 남겨 두는 편이 좋다.

특히 공장이 포장 업체나 인쇄 업체를 다시 쓰는 구조라면, 파일 전달 범위와 회수 가능성을 같이 봐야 종료 시 통제가 쉬워진다.

### 유통과 OEM은 상표 리스크를 외부화하지 않는다

중국에서는 제조와 유통이 분리돼도, 브랜드 리스크까지 외부로 이전되지는 않는다. 특히 OEM이나 제3자 제조 구조에서는 누가 상표를 라벨에 붙이고, 누가 포장 문구를 확정하며, 누가 온라인 채널 이미지를 올리는지 정리되지 않으면 나중에 통제가 무너진다.

유통 구조에서도 비슷한 문제가 생긴다. 총판이 판매 페이지를 운영하거나 현지명 표기를 자체적으로 정하기 시작하면, 등록 내용과 실제 시장 사용이 갈라질 수 있다. 이때 법무가 사후 수정만 하면 이미 증거와 소비자 인식은 어긋나 있을 수 있다.

### 계약 검토는 분쟁 종료 장면까지 본다

계약 초안 단계에서 중요한 질문은 "지금 잘 굴러갈까"보다 "문제가 생기면 끊을 수 있는가"다. 라이선스 종료, 유통계약 종료, OEM 교체 시점에 표장 사용을 멈추게 할 수 있는 조항과 실제 운영 루틴이 같이 있어야 한다.

#### 계약 종료 후 재고와 페이지는 누가 정리할까

종료 조항이 있어도 실제 정리 담당자가 불명확하면 표장 사용 흔적이 오래 남는다. 남은 재고, 플랫폼 페이지, 광고 문구, 포장 자산을 누가 어떤 순서로 회수하거나 비활성화할지 미리 정해 두는 편이 실무적으로 훨씬 중요하다.

이 장면은 분쟁 종료뿐 아니라 불사용취소 방어와도 연결된다. 어떤 시점까지 누구의 통제 아래 사용이 이어졌는지 설명해야 할 수 있기 때문이다.

#### 종료 시점 점검표

- 재고 소진 규칙이 있는가
- 포장재와 라벨 회수 기준이 있는가
- 플랫폼 상세페이지와 광고 문구를 누가 내릴 수 있는가
- 중국어 표기와 로고 파일 접근권한이 통제되는가
- 종료 후 남는 사용 흔적을 누가 정리하는가

## 계약 종료 readiness 보드

파트너 교체 또는 계약 종료 시 브랜드 통제 공백이 생기지 않도록 종료 전에 아래 항목 상태를 점검한다.

| 항목 | 종료 전 확인 사항 | 담당 |
| --- | --- | --- |
| 재고 현황 | 잔여 재고 수량·위치 파악 | 운영팀 |
| 포장재·라벨 | 회수 또는 소각 기준 | 공급망팀 |
| 플랫폼 페이지 | 판매 페이지·광고 문구 비활성화 일정 | 디지털팀 |
| 로고·도안 파일 | 접근 권한 회수 일정 | IT + 브랜드팀 |
| 중국어 표기 자산 | 소셜·채널 계정명 사용 중단 | 브랜드팀 |
| 사건 기록 | 계약 기간 중 분쟁·경고장 이력 정리 | 법무 |

### 종료 후 사용 흔적 잔류 위험 사전 차단

종료 후에도 플랫폼 검색에 구 판매자명이나 중국어 표기가 남는 경우가 있다. 종료 계획에는 플랫폼 검색 결과와 캐시 페이지 확인 일정을 포함시키고, 30일 이내 이상 잔류가 있으면 재확인 루틴을 가동하는 편이 좋다.

### 이 장의 운영 메모

라이선스, 유통, OEM 문제는 계약 부서와 법무팀만의 문제가 아니다. 상표 운영 관점에서는 브랜드 사용 구조를 어디까지 직접 통제할 수 있는지가 핵심이다. 계약서 문구보다 중요한 것은, 실제 현장에서 누가 표장 버전을 고치고 채널에 올리고 포장을 찍는지 보이는 구조다.
