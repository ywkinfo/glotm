# 제3장. 검색, 분류, 서브클래스 리스크 분석

중국에서 검색은 단순 중복 확인이 아니라 "이 이름을 계속 밀고 갈 수 있는가"를 결정하는 과정이다. 특히 중국 실무에서는 국제분류만 맞춘다고 충분하지 않고, 실제 거래 구조와 서브클래스 관점을 같이 봐야 한다. 그래서 검색 단계에서 표장 리스크와 지정상품 설계를 함께 읽는 습관이 필요하다.

## 검색의 목표를 먼저 정한다

검색을 시작할 때 가장 먼저 정해야 할 것은 "무엇을 확인할 것인가"다. 중국어 표기 충돌, 영문 상표 충돌, 유사 발음, 업종 근접성, 서브클래스 접점은 각각 별개로 봐야 한다. 이 구분 없이 한 번에 검색하면 결과는 많아지지만 의사결정에는 도움이 적다.

실무에서는 보통 다음 순서가 효율적이다. 먼저 핵심 표장을 확정하고, 그 다음 중국어 표기 후보를 포함해 1차 스크리닝을 돌린다. 이후 충돌 후보가 보이면 지정상품과 서브클래스 방향을 함께 점검한다. 즉, 검색은 표장만 보는 단계가 아니라 출원 범위를 수정하는 단계이기도 하다.

## 클래스와 서브클래스를 같이 읽어야 하는 이유

중국에서는 클래스 번호만 같다고 같은 위험도는 아니고, 클래스 번호가 다르다고 바로 안전한 것도 아니다. 실제 충돌 판단에서는 어떤 상품이 같은 거래 맥락에서 만나는지, 서브클래스 접점이 있는지, 유통 채널이 얼마나 겹치는지를 함께 보게 된다. 따라서 국제분류 번호만 보고 안심하면 나중에 거절 또는 분쟁 대응에서 다시 비용이 생긴다.

특히 소비재 기업은 하나의 브랜드가 화장품, 소매 서비스, 플랫폼 판매, 라이선스 상품처럼 여러 층으로 확장되기 쉽다. 그래서 현재 판매 품목과 가까운 범위, 12~24개월 내 현실적인 확장 범위, 아직 너무 이른 범위를 나누어 설계하는 편이 좋다.

2026-03-31 기준 사실 검증 로그에서 잠근 `CN-FIL-001`에 따르면, 중국 출원에서는 `类似商品和服务区分表`의 표준명칭과 수리가능 명칭 구조를 먼저 맞추는 편이 안전하다. 그래서 내부적으로는 편의상 "서브클래스"라고 말하더라도, 실제 회의 메모에는 중국 원문 용어와 상품군 묶음을 함께 적어 두는 쪽이 이후 대리인 커뮤니케이션에서 덜 흔들린다.

### 서브클래스 관점의 점검표

| 구간 | 확인 질문 | 기본 판단 |
| --- | --- | --- |
| 현재 판매 범위 | 실제 SKU와 지정상품이 바로 연결되는가 | 우선 확보 대상 |
| 근접 확장 범위 | 1~2년 안에 진입 가능성이 있는가 | 선택적 확보 대상 |
| 먼 확장 범위 | 아직 제품 정의가 불명확한가 | 보류 검토 |

### 표준명칭과 마케팅 표현을 분리하는 기준

`CN-FIL-001`에서 잠근 핵심은 단순히 "서브클래스를 보라"가 아니다. 실제 출원 문구는 `类似商品和服务区分表`의 표준명칭과 공개된 수리가능 명칭 구조를 먼저 맞추고, 내부 마케팅 표현은 그 옆에 번역 메모처럼 붙이는 편이 안전하다는 점이다.

| 입력값 | search 단계에서 보는 질문 | filing 단계로 넘길 메모 |
| --- | --- | --- |
| 표준명칭 | 공식 구분표 용어에 바로 대응되는가 | 우선 문구 후보 |
| 수리가능 명칭 | 비표준이어도 구체성과 분류원칙을 충족하는가 | 대리인 검토 필요 |
| 마케팅 표현 | 소비자에게는 익숙하지만 지정상품으로는 너무 넓지 않은가 | 본문 설명용으로만 유지 |
| 내부 SKU 설명 | 실제 판매 품목과 1:1로 연결되는가 | core/adjacent 구분 근거 |

이 구분을 search 단계에서 먼저 해 두면, 제5장에서 마케팅 카피를 그대로 지정상품에 옮기는 실수를 줄일 수 있다.

## 검색 결과를 읽는 방식

검색 결과를 받으면 정확히 같은 표장만 보는 것은 부족하다. 문자, 발음, 의미, 중국어 표기 연상, 채널 겹침을 같이 봐야 한다. 특히 중국어 표기는 의역형과 음역형이 각각 다른 충돌을 만들 수 있으므로, 하나의 결과표 안에서 구분해 두는 편이 좋다.

이 단계에서 가장 실무적인 출력은 "위험 없음"보다 "어떤 수정으로 위험을 낮출 수 있는가"다. 예를 들어 이름을 유지하되 지정상품을 줄일 수 있는지, 중국어 표기를 조정해야 하는지, 아예 다른 후보를 써야 하는지를 같이 적어 두면 출원 회의가 빨라진다.

### 검색 산출물을 회의 자료로 바꾸는 표준형

검색 결과를 그냥 캡처만 남기면, 다음 회의에서 다시 처음부터 설명해야 한다. 그래서 검색 단계의 산출물은 바로 의사결정 표로 변환하는 편이 좋다.

| 후보 표장 | 검색 축 | 핵심 충돌 | 현재 판단 | 다음 액션 |
| --- | --- | --- | --- | --- |
| 영문 워드마크 | 동일/유사 문자 | 핵심 식별부 직접 충돌 여부 | go / revise / hold | 유지, 수정, 보류 |
| 중국어 표기 A | 발음/의미/표기 | 동일 발음·근접 개념 충돌 | go / revise / hold | 후보 유지 또는 교체 |
| 상품군 묶음 | 类似群 접점 | 코어 상품군 직접 접점 여부 | core / adjacent / defer | 즉시 포함, 선택 포함, 보류 |

이 표를 한 번 만들면 제4장 경로 선택과 제5장 출원 설계가 크게 빨라진다. 이름과 범위를 어디서 줄일지 바로 보이기 때문이다.

## 검색 후 액션을 바로 정한다

검색 이후 결론은 보통 세 갈래다. 그대로 가는 `go`, 손보고 가는 `revise`, 보류하는 `hold`다. 이 분류를 남기지 않으면 검색 메모가 쌓여도 실제 의사결정이 지연된다.

### 검색 결과 해석표

| 등급 | 의미 | 기본 액션 |
| --- | --- | --- |
| Low | 충돌 후보가 약하고 범위 조정 필요가 작다 | 출원 준비로 이동 |
| Medium | 이름 또는 지정상품 수정 여지가 있다 | 수정안 비교 후 결정 |
| High | 핵심 식별부 또는 거래 범위 충돌이 크다 | 출원 보류 또는 이름 재선정 |

### 검색 단계 체크리스트

- 영문 표장과 중국어 표기를 따로 검색 대상으로 분리했는가
- 클래스와 서브클래스를 함께 읽었는가
- 검색 결과를 `go/revise/hold`로 정리했는가
- 지정상품 설계 변경이 필요한 후보를 바로 표시했는가
- `类似商品和服务区分表` 기준 명칭과 내부 상품군 설명을 같은 표에 묶었는가

## 검색 브리프 템플릿

검색 품질은 검색 엔진보다 브리프 품질에 더 크게 좌우된다. 대리인이나 내부 검색 담당자에게 아래 항목을 한 번에 넘기면 표장 검색과 지정상품 설계가 훨씬 덜 흔들린다.

| 브리프 항목 | 포함 내용 | 빠지면 생기는 문제 |
| --- | --- | --- |
| 표장 묶음 | 영문, 중국어, 결합표장, 제외 후보 | 검색 범위가 제각각 됨 |
| 코어 상품군 | 현재 판매 SKU와 가까운 범위 | 서브클래스 접점이 흐려짐 |
| 인접 상품군 | 12~24개월 확장 범위 | 과소/과다 출원 판단이 어려워짐 |
| 위험 채널 | 플랫폼, 오프라인, OEM, 유통 | 실제 충돌 경로를 놓침 |
| 회의 목표 | `go`, `revise`, `hold` 중 무엇을 고를지 | 검색이 보고서로만 끝남 |

## 서브클래스 충돌 escalation 기준

중국 검색 단계에서는 같은 결과라도 어떤 건은 메모만 남기고, 어떤 건은 바로 경영 판단까지 올려야 한다. escalation 기준을 먼저 고정해 두면 제5장 출원 설계가 빨라진다.

| 상황 | 기본 판단 | 다음 액션 |
| --- | --- | --- |
| 코어 상품군과 직접 접점이 없음 | low | 후보 유지, 후속 모니터링 |
| 인접 상품군에서만 근접 접점이 보임 | medium | 지정상품 조정안 비교 |
| 코어 상품군과 식별부가 동시에 겹침 | high | 이름 수정 또는 출원 보류 검토 |
| 중국어 표기만 충돌이 큼 | medium/high | 중국어 후보 재선정 후 재검색 |
| OEM/유통 채널에서 접점이 큼 | high | 사업팀 포함 회의 즉시 소집 |

## 채널별 검색 보정표

중국 검색은 CNIPA 결과만 읽고 끝내기보다, 실제 판매 채널이 무엇인지에 따라 위험 해석을 달리해야 한다. 같은 결과라도 플랫폼 중심 브랜드와 OEM 중심 브랜드는 위험 지점이 다르다.

| 채널 | 검색에서 더 봐야 할 것 | 흔한 오판 | 후속 액션 |
| --- | --- | --- | --- |
| 플랫폼 판매 중심 | 중국어 표기, 검색어 변형, 셀러 상품명 | 등록상표만 보고 안심함 | 플랫폼 사용형과 search 결과를 함께 메모 |
| 오프라인 유통 중심 | 포장·진열에서 보이는 표기, 채널 충돌 | 클래스만 보고 유통 접점을 놓침 | 유통 채널별 충돌 후보 분리 |
| OEM/제조 중심 | 제조품 라벨, 수출입 문서, 부품/완제품 경계 | 실제 거래 명칭과 출원 명칭이 어긋남 | 제조/유통 계약명까지 search 브리프에 반영 |
| 라이선스 확장 중심 | product mark와 sub-brand 구조 | 하위 브랜드를 search 범위에서 누락 | house mark와 product mark를 분리 검색 |

이 표를 장 안에 넣어 두면 "같은 search 결과를 왜 다르게 읽는가"가 설명된다. 중국 실무에서 search는 등록 가능성만 보는 단계가 아니라, 실제 판매와 집행 경로가 어디서 부딪히는지 먼저 보는 단계다.

## 제5장 출원 브리프로 넘기는 handoff 표

검색 단계가 끝난 뒤에도 많은 팀이 다시 회의를 연다. 가장 비효율적인 패턴은 search 메모가 제5장 출원서 작성 입력값으로 바로 이어지지 않는 경우다.

| search 출력값 | 제5장 입력값 | owner | 잠그지 않으면 생기는 문제 |
| --- | --- | --- | --- |
| go 후보 표장 | 출원 대상 표장 | IP | 출원 직전 후보가 다시 바뀜 |
| revise 필요 상품군 | 지정상품 수정안 | IP + 사업팀 | 상품군이 과대/과소로 설계됨 |
| high risk 중국어 표기 | 대체 표기 후보 | 브랜드팀 | 검색 재작업이 반복됨 |
| OEM/유통 접점 메모 | 실제 사용자/채널 메모 | 사업팀 | 사용형과 출원형이 어긋남 |

handoff 표의 목적은 간단하다. search 결과를 "참고 자료"가 아니라 출원 설계를 결정하는 입력값으로 바꾸는 것이다. 그렇게 해야 제5장에서 누가 어떤 자료를 준비할지 빠르게 정리된다.

## 검색·분류 체크리스트

- 검색 대상 표장을 영문, 중국어, 결합표장별로 분리하여 각각 검색했는가
- `类似商品和服务区分表` 기준으로 서브클래스(类似群) 충돌 여부를 확인했는가
- 검색 결과를 go / revise / hold로 분류하고 각 판단 이유를 기록했는가
- 중국어 표기 후보별로 음역·의역 변형까지 검색 범위에 포함했는가
- 플랫폼·OEM·유통 채널별 사용형을 검색 브리프에 반영했는가
- 서브클래스 교차검토(类似群 간 유사 판단)를 확인했는가
- search 결과와 제2장 표기 잠금 결과가 서로 일관되는지 대조했는가
- handoff 표를 제5장 출원 브리프 담당자에게 전달하고 확인받았는가

## 운영 메모

검색은 1회성 단계가 아니라 출원 전까지 반복될 수 있다. 특히 중국어 표기가 바뀌거나 상품군이 추가될 때마다 검색을 다시 돌려야 한다. 이 반복을 줄이려면 처음 검색 때 "어떤 조건이 바뀌면 재검색이 필요한가"를 같이 적어 두는 편이 좋다.

서브클래스 충돌이 발견되면 바로 출원을 포기하지 말고, 지정상품을 좁히거나 서브클래스를 이동하는 방식으로 우회할 수 있는지 먼저 검토한다. 중국의 서브클래스 체계는 다른 국가보다 세분되어 있어, 같은 류(Class) 안에서도 충돌하지 않는 구간이 있을 수 있다.

검색 보고서는 외부 대리인에게 의뢰할 때도 검색 범위와 판단 기준을 명확히 전달해야 한다. "영문 브랜드명으로 검색해 달라"는 요청만으로는 중국어 표기, 음역 변형, 채널 사용형이 빠지기 쉽다.
