# GloTm Harness Constitution

이 문서는 에이전트 종류와 무관하게 유지되는 영구 원칙만 다룬다.
현재 phase, 우선순위, 활성 작업 범위는 `PROJECT-OVERVIEW.md`를 기준으로 보고, 실행 명령과 현재 운영 계약은 `README.md`를 기준으로 확인한다.

## Core Principles

1. 정확성이 완성도보다 우선이고, 완성도는 속도보다 우선이다.
   법률·상표 실무 콘텐츠는 그럴듯한 추정보다 검증된 사실이 더 중요하다.
2. 복잡도는 비용이다.
   1인 운영 전제를 지키기 위해 큰 프레임워크, 과한 추상화, 불필요한 패키지 추가를 피한다.
3. 콘텐츠 무결성은 기능 추가보다 중요하다.
   출처가 민감한 법률 정보, 기한, 수수료, 기관명, 절차 설명은 공식 자료 또는 기존 검증 기록으로 다시 확인한다.
4. 현재 활성 phase 문서를 존중한다.
   새로운 아이디어가 좋아 보여도, 활성 phase 밖 작업은 명시적 지시 없이는 선행하지 않는다.
5. 사용자 흐름을 깨는 개선은 개선이 아니다.
   홈 진입, 챕터 읽기, 검색, 목차 이동, continue reading 같은 핵심 읽기 흐름은 항상 보존한다.

## Working Rules

- 실행 판단은 먼저 현재 저장소의 사실과 스크립트를 확인한 뒤 내린다.
- generated 산출물은 손으로 맞추기보다, 원천 원고와 빌드 스크립트를 통해 재생성하는 방향을 우선한다.
- 기존 문서와 규칙을 바꿀 때는 중복 소스가 생기지 않도록 단일 기준 문서를 유지한다.
- 불확실성이 큰 법률 사실이나 반복 실패가 누적되는 작업은 무리하게 밀어붙이지 않고 멈춰서 보고한다.

## Decision Order

판단이 충돌할 때는 아래 순서를 따른다.

1. 이 문서의 영구 원칙
2. 현재 활성 phase와 우선순위를 적은 진입 문서
3. `Harness/Style-Guide.md`
4. 워크스페이스별 `Harness/Architecture.md`, `Harness/Content-Spec.md`
5. 작업별 로컬 코드와 실제 스크립트 동작
