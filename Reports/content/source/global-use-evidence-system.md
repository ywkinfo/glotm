# 글로벌 사용 증거 수집 운영 시스템 구축

브랜드가 여러 국가에 동시에 진입하기 시작하면, 사용 증거는 분쟁이 생긴 뒤에 급하게 모으는 자료가 아니라 평소 운영 시스템의 일부가 됩니다. 문제는 대부분의 팀이 증거를 법무팀 요청이 올 때마다 개별 폴더에서 찾고, 각 국가가 요구하는 자료 형식과 보관 주기를 따로 관리하지 못한다는 점입니다.

이 리포트는 미국, 중국, 멕시코처럼 실무 요구가 자주 달라지는 국가를 함께 염두에 두고, 어떤 자료를 누가 어떤 이름으로 보관하고 언제 점검해야 하는지 운영 관점에서 정리합니다.

### 왜 지금 사용 증거 운영 시스템이 필요한가

출원은 한 번으로 끝나도, 사용 증거는 시장이 커질수록 계속 쌓여야 합니다. 실제 문제는 자료가 없는 것보다 자료가 흩어져 있다는 데서 더 자주 발생합니다. 판매 스크린샷은 마케팅팀에 있고, 송장과 주문 데이터는 운영팀에 있고, 패키지 변경 이력은 외주 디자이너 메일함에만 남아 있으면, 필요한 시점에 신뢰할 수 있는 증거 묶음을 빠르게 만들기 어렵습니다.

#### 국가별 차이보다 먼저 봐야 하는 것

국가별 요건 차이를 먼저 외우기보다, 공통으로 재사용 가능한 증거 구조를 먼저 설계해야 합니다. 기본 원칙은 간단합니다. 거래 흔적, 상품 표시, 실제 사용 화면, 시점 증명, 권리자와 사용자의 연결 관계가 같은 폴더 체계 안에서 이어져야 합니다.

#### 가장 자주 실패하는 운영 패턴

많은 팀이 "나중에 필요하면 모은다"는 방식으로 움직입니다. 이 방식은 내부 담당자가 바뀌거나, 플랫폼 화면이 교체되거나, 유통 파트너가 판매 기록을 오래 보관하지 않을 때 바로 무너집니다.

### 최소 운영 구조

사용 증거 운영 체계는 거대한 데이터 플랫폼이 아니라, 반복 가능한 폴더 구조와 책임 배분에서 시작할 수 있습니다. 분기별로 같은 자료를 같은 이름으로 쌓게 만들면, 이후 국가별 대응이 훨씬 쉬워집니다.

#### 공통 저장 단위

시장별로 폴더를 나누되, 각 폴더 아래에는 판매 화면, 실제 포장, 거래 증빙, 광고/프로모션 집행 흔적, 내부 승인 로그를 같은 순서로 두는 편이 좋습니다. 이렇게 해야 미국이나 멕시코처럼 사용 시점과 실제 판매 맥락을 함께 설명해야 할 때 자료를 다시 분류하지 않아도 됩니다.

#### 권한과 책임 배분

법무팀만 증거를 책임지는 구조는 오래 유지되지 않습니다. 운영팀은 거래 흔적과 채널 스크린샷, 마케팅팀은 광고와 캠페인 자료, IP 담당은 분쟁 대응용 패키징과 권리자 정보 연결을 각각 owner로 두는 편이 현실적입니다.

#### 분기 점검 리듬

증거는 사건이 생겼을 때가 아니라, 분기 점검 때 정상적으로 수집되는지 보는 편이 훨씬 낫습니다. 각 시장별로 최신 판매 페이지가 살아 있는지, 정품 식별 요소가 바뀌었는지, 보관 위치와 파일명이 유지되는지 확인해야 합니다.

### 여러 국가에서 바로 재사용하려면

한 국가에서 만든 자료를 다른 국가 대응에 쓰려면, 문서 이름과 메타데이터가 먼저 정리돼 있어야 합니다. 국가명이 파일명 뒤에만 붙어 있고 브랜드명, 제품군, 채널, 수집 시점이 빠져 있으면, 재사용성이 크게 떨어집니다.

#### 미국과 중국에서 보는 차이

미국은 실제 상업적 사용과 거래 흐름을 설득력 있게 보여주는 자료 조합이 중요하고, 중국은 표기와 사용 형태가 실제 등록 포트폴리오와 얼마나 정합적인지가 더 민감해질 수 있습니다. 그래서 공통 자료를 쌓더라도 표기 버전과 상품 표시 이미지는 별도 태그로 관리하는 편이 좋습니다.

#### 멕시코와 플랫폼 대응을 위한 준비

멕시코나 플랫폼 신고에서는 거래·유통 문맥과 실제 상품 식별 자료가 함께 요구될 때가 많습니다. 따라서 증거 운영 체계 안에 정품 패키지, 판매 화면, 거래 문서, 채널 owner 정보를 같이 남겨 두는 것이 좋습니다.

### 지금 바로 시작할 수 있는 체크리스트

이번 주 안에 할 일은 복잡하지 않습니다. 첫째, 핵심 수출 시장별로 같은 폴더 구조를 만들고 owner를 정합니다. 둘째, 판매 화면과 거래 증빙, 패키지 이미지, 광고 흔적을 어떤 이름으로 저장할지 규칙을 정합니다. 셋째, 분기 점검 일정을 넣고 누락 여부를 체크합니다.

#### 가장 작은 시작점

처음부터 모든 국가를 동시에 정비할 필요는 없습니다. 미국, 중국, 멕시코처럼 리스크가 빠르게 드러나는 2~3개 시장만 먼저 고르고, 그 시장에서 재사용 가능한 저장 구조를 만든 뒤 확장하면 됩니다.
