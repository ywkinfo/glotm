# 제04장. Filing 전략: 출원 경로 선택 (직접출원 vs 마드리드)

> 이 장의 목적: 중남미 8개국에 대한 출원 경로를 정확하게 분기한다.
> 경로 선택을 잘못하면 비용보다 더 큰 손실, 즉 출원 불가 국가를 놓치거나 실제 필요한 현지 대응을 늦추게 된다.
> 03장(상표 검색) 완료 후 이 장을 진행하고, 완료 후 05장(출원 실무)으로 이동한다.

---

## 1. 핵심 원칙

> "안데스는 공통 규범이고, 출원은 국가별이다. 마드리드는 일부 국가에서만 쓴다."

중남미 8개국을 하나의 경로로 처리할 수 있는 단일 시스템은 없다.
이번 가이드의 대상국 중 마드리드 시스템을 사용할 수 있는 국가는 브라질, 멕시코, 콜롬비아, 칠레뿐이다.
아르헨티나, 페루, 에콰도르, 파나마는 국가별 직접출원이 전제다.

콜롬비아, 페루, 에콰도르는 안데스 공동체 Decision 486이라는 공통 규범을 공유하지만,
그렇다고 하나의 "공동체 상표"를 한 번에 출원하는 구조는 아니다.
검색, 이의, 불사용 취소의 판단 프레임은 유사하게 가져갈 수 있지만,
등록은 각국 상표청에 개별로 확보해야 한다.

따라서 출원 경로는 다음 두 축을 동시에 본다.

1. 해당 국가가 마드리드 체약국인지
2. 그 국가에서 직접출원이 더 유리한지, 아니면 마드리드가 관리상 실익이 있는지

---

## 2. 출원 경로 개요

| 경로 | 설명 | 대상 8개국 적용 범위 |
| --- | --- | --- |
| 직접출원 | 각국 상표청에 개별로 출원 | 8개국 전체 |
| 마드리드 국제출원 | 한국 기본출원/등록을 기반으로 WIPO를 통해 지정 | 브라질, 멕시코, 콜롬비아, 칠레 |
| 안데스 공통 규범(Decision 486) | 콜롬비아·페루·에콰도르에서 검색·이의·불사용 기준을 권역 단위로 설계 | 별도 출원 루트가 아니라 국가별 직접출원의 설계 기준 |

> 정리: 안데스 권역은 "공통 법리"가 있을 뿐이고, 마드리드는 "일부 국가에서만 가능한 국제 절차"다.
> 이 둘을 같은 종류의 출원 경로로 취급하면 설계가 틀어진다.

## 3. 주요 리스크

### 3-1. 마드리드 범위 오인 리스크

| 리스크 유형 | 설명 |
| --- | --- |
| 대상국 오인 | 아르헨티나, 페루, 에콰도르, 파나마까지 마드리드로 처리할 수 있다고 착각 |
| 기본출원 종속 | 기본출원/기본등록이 5년 내 흔들리면 중앙공격 리스크가 발생 |
| 현지 대응 지연 | 거절·이의가 나오면 결국 현지 대리인이 필요하므로 "현지 대응이 없다"는 생각이 위험 |
| 명세 유연성 부족 | 한국 기본출원 범위를 넘지 못해 국가별 맞춤 명세 설계가 어려움 |

### 3-2. 직접출원 과소평가 리스크

| 리스크 유형 | 설명 |
| --- | --- |
| 초기 관리 부담 | 국가별 출원번호, 대리인, 기한을 따로 관리해야 함 |
| 언어별 명세 차이 | 브라질은 포르투갈어, 나머지 대부분은 스페인어로 실무 차이가 큼 |
| 속도 착시 | 한 번에 넣는 마드리드가 더 빨라 보이지만, 실제 핵심국에서는 직접출원이 더 실용적일 수 있음 |

### 3-3. 안데스 구조 오인 리스크

| 리스크 유형 | 설명 |
| --- | --- |
| 단일 권리 환상 | Decision 486이 있으니 한 번의 출원으로 3개국을 덮는다고 착각 |
| 국가별 검색 생략 | 권역 규범만 보고 콜롬비아·페루·에콰도르 DB를 따로 보지 않는 실수 |
| 국가별 이의 대응 누락 | 권역을 묶어 관리하더라도 실제 공고와 절차는 국별로 움직이는 점을 놓침 |

## 4. Decision Box: 출원 경로 선택

이 Decision Box는 국가 확정 후 가장 먼저 실행한다.
결론은 "하나의 경로"가 아니라 국가별 믹스로 나온다.

```
[Step 1] 국가를 마드리드 가능국과 직접출원 전용국으로 나눈다.

  마드리드 가능국: 브라질, 멕시코, 콜롬비아, 칠레
  직접출원 전용국: 아르헨티나, 페루, 에콰도르, 파나마

[Step 2] 파나마·아르헨티나·페루·에콰도르를 먼저 직접출원으로 고정한다.

  IF (파나마 포함)
      → DIGERPI 직접출원 확정

  IF (아르헨티나 포함)
      → INPI-AR 직접출원 확정

  IF (페루 또는 에콰도르 포함)
      → INDECOPI / SENADI 직접출원 확정
        단, 검색·이의·사용증거 설계는 안데스 권역 단위로 함께 설계

[Step 3] 브라질과 멕시코는 직접출원을 기본값으로 본다.

  IF (브라질 포함)
      → 직접출원 우선
        사유: 포르투갈어 명세 현지화, 현지 대리인 대응, 핵심시장 선점이 중요

  IF (멕시코 포함)
      → 직접출원 우선
        사유: 현지 유통 파트너 선점 리스크가 높고, 등록 후 사용선언 관리까지 함께 설계해야 함

[Step 4] 콜롬비아와 칠레는 사업 구조에 따라 직접출원 vs 마드리드를 비교한다.

  IF (콜롬비아 또는 칠레를 포함하고, 같은 기본출원으로 다른 비중남미 Madrid 체약국도 함께 지정할 계획)
      → 마드리드 검토 가능

  IF (국가별 검색 충돌이 높거나 명세 최적화가 중요)
      → 직접출원 우선

[Step 5] 최종 믹스를 문서화한다.

  예시 A: 브라질 + 멕시코 + 아르헨티나 + 파나마
      → 전부 직접출원

  예시 B: 브라질 + 멕시코 + 콜롬비아 + 칠레 + EU/UK 동시 진출
      → 브라질/멕시코 직접출원 + 콜롬비아/칠레는 마드리드 또는 직접출원 비교

  예시 C: 콜롬비아 + 페루 + 에콰도르
      → 3개국 모두 직접출원
        단, Decision 486 기준으로 검색·이의·불사용 취소 전략을 통합
```

## 5. 국가별 권장 경로 요약

| 국가 | 기본 권장 경로 | 대안 | 비고 |
| --- | --- | --- | --- |
| 브라질 | 직접출원 | 마드리드 보조 검토 | 포르투갈어 명세와 현지 대응이 중요 |
| 멕시코 | 직접출원 | 마드리드 보조 검토 | 등록 후 사용선언 관리까지 함께 설계 |
| 콜롬비아 | 직접출원 | 마드리드 검토 가능 | Decision 486 기준으로 안데스 권역과 연동 관리 |
| 아르헨티나 | 직접출원 | 없음 | 마드리드 비체약국 |
| 칠레 | 직접출원 또는 마드리드 | 둘 다 실무 검토 가능 | 남미 남부 확장 전략과 함께 판단 |
| 페루 | 직접출원 | 없음 | 마드리드 비체약국, Decision 486 적용 |
| 에콰도르 | 직접출원 | 없음 | 마드리드 비체약국, Decision 486 적용 |
| 파나마 | 직접출원 | 없음 | 마드리드 비체약국 |

## 6. 실무 비교 포인트

### 6-1. 직접출원 vs 마드리드

| 항목 | 직접출원 | 마드리드 |
| --- | --- | --- |
| 국가 범위 | 대상국 전체 커버 가능 | 체약국만 가능 |
| 기본출원 의존 | 없음 | 한국 기본출원/기본등록 필요 |
| 명세 최적화 | 국가별 조정 가능 | 기본출원 범위 내 제한 |
| 현지 대응 | 출원 초기부터 현지 대리인 연결 | 거절·이의 시 결국 현지 대리인 필요 |
| 중앙공격 리스크 | 없음 | 있음 |

### 6-2. 안데스 권역 운영 포인트

| 항목 | 실무 원칙 |
| --- | --- |
| 검색 | 콜롬비아·페루·에콰도르를 각각 검색 |
| 출원 | 각국 상표청에 직접출원 |
| 법리 설계 | Decision 486을 공통 기준으로 활용 |
| 사용증거 | 국가별 판매·광고·유통 증거를 따로 보관 |

## 7. 실행 절차

### Step 1. 국가별 분기표 작성 (출원 D-60 이전)

1. 대상국을 `직접출원 전용`과 `마드리드 가능`으로 나눈다.
2. 브라질·멕시코는 직접출원을 기본값으로 놓고 별도 예외 사유가 있을 때만 마드리드를 검토한다.
3. 콜롬비아·칠레는 글로벌 동시 지정 여부와 명세 유연성을 비교해 선택한다.
4. 페루·에콰도르·아르헨티나·파나마는 직접출원으로 고정한다.

### Step 2. 안데스 3국 전략 메모 작성 (출원 D-45 이전)

안데스 국가가 포함되면 아래를 별도 메모로 정리한다.

- 국가별 직접출원 여부
- Decision 486 관련 공통 논점
- 국가별 검색 결과
- 국가별 공고/이의 캘린더
- 국가별 사용증거 수집 계획

### Step 3. 마드리드 사용 여부 최종 승인 (출원 D-30 이전)

마드리드를 쓰려면 다음 4개 조건을 모두 확인한다.

1. 대상국이 실제로 체약국인가
2. 한국 기본출원/기본등록 상태가 안정적인가
3. 기본출원 명세 범위가 현지 보호 목표와 맞는가
4. 거절·이의 발생 시 현지 대리인 체계를 이미 확보했는가

## 8. 체크리스트

### 공통

- [ ] 대상 8개국을 마드리드 가능국과 직접출원 전용국으로 분리했다
- [ ] 아르헨티나, 페루, 에콰도르, 파나마는 직접출원으로 고정했다
- [ ] 브라질·멕시코는 직접출원을 기본값으로 검토했다
- [ ] 콜롬비아·칠레는 직접출원과 마드리드를 비교 검토했다
- [ ] 안데스 3국은 "공통 규범 + 국가별 직접출원" 구조로 정리했다
- [ ] 최종 출원 경로 믹스를 포트폴리오 관리 대장에 기록했다

### 멕시코 해당 기업 추가 확인

- [ ] 등록 후 사용선언 기한을 `등록일 3주년 후 3개월` 기준으로 관리하기로 확정했다
- [ ] 유통 파트너 계약 일정이 출원 선행 원칙과 충돌하지 않는지 점검했다

### 안데스 권역 해당 기업 추가 확인

- [ ] 콜롬비아·페루·에콰도르를 각각 직접출원 대상으로 분리했다
- [ ] Decision 486을 검색·이의·불사용 취소의 공통 기준으로 문서화했다
- [ ] 국가별 공고 모니터링을 별도로 운영하기로 정했다

### 마드리드 경로 선택 국가 추가 확인

- [ ] 체약국 여부를 WIPO 공식 회원국 페이지 기준으로 재확인했다
- [ ] 한국 기본출원/기본등록 상태를 확인했다
- [ ] 5년 중앙공격 리스크 종료일을 캘린더에 반영했다

## 9. 실패 사례 및 주의사항

### 실패 패턴 1: "마드리드 하나로 8개국을 커버하려 했다"

상황: 파나마, 아르헨티나, 페루, 에콰도르까지 한 번에 지정할 수 있다고 보고 일정과 예산을 짰다.

결과: 실제 착수 단계에서 직접출원 전용국을 다시 분리하느라 일정이 늦어지고, 우선순위가 무너졌다.

교훈: 먼저 체약국 여부를 확정하고, 직접출원 전용국을 즉시 분리한다.

### 실패 패턴 2: "안데스 권역을 하나의 상표로 처리할 수 있다고 오해했다"

상황: Decision 486을 이유로 콜롬비아·페루·에콰도르를 단일 출원처럼 관리했다.

결과: 국가별 검색과 공고 대응이 누락되어 한 국가에서 발생한 충돌을 늦게 인지했다.

교훈: 안데스 권역은 공통 규범일 뿐이며, 실제 등록과 모니터링은 국가별로 운영한다.

### 실패 패턴 3: "브라질·멕시코도 관리 편의만 보고 마드리드로 넣었다"

상황: 본사에서 행정 단순화를 이유로 브라질·멕시코를 마드리드로만 처리하려 했다.

결과: 현지 명세 조정과 파트너 리스크 대응이 늦어져 핵심시장 선점 속도가 떨어졌다.

교훈: 핵심시장일수록 직접출원을 기본값으로 보고, 마드리드는 보조 경로로만 평가한다.
