# 제7장 등록 후 유지관리: 갱신·권리 유지 캘린더

등록이 끝이 아니고 여기서부터 운영이 시작된다. 일본은 설정등록일부터 10년 단위로 권리를 이어가므로, 등록료·갱신·명의 변경·예산 승인 흐름이 하나의 캘린더로 묶여 있어야 한다. 일정 누락보다 더 큰 문제는 누가 무엇을 관리하는지 모르는 상태다. 등록 직후의 정리 품질이 이후 10년의 유지관리 품질을 좌우한다.

이 장은 일본 상표 운영을 “권리 보유”가 아니라 “권리 유지 시스템”으로 보는 장이다. 등록된 권리를 실제 사업 포트폴리오와 계속 맞추고, 조직 개편과 거래 구조 변화를 반영하며, 갱신과 분쟁 대응에 필요한 자료를 잃지 않게 만드는 것이 핵심이다.

### lighter-track maintenance owner board

JapTm의 유지관리 장은 full expansion보다 “누가 무엇을 놓치지 않을까”를 빠르게 다시 보는 용도에 가깝다. 그래서 등록 직후 아래 네 줄을 먼저 적어 두면, lighter track에서도 유지관리 공백을 크게 줄일 수 있다.

| 등록 직후 잠글 것 | 왜 필요한가 | owner | 다음 handoff |
|---|---|---|---|
| 등록번호 / 핵심 상품군 | 어떤 권리를 유지하는지 한 줄로 보여 줌 | IP | 분기 점검표 |
| owner / actual user | 권리자와 실제 사용자 관계 설명 | 법무 + 사업팀 | evidence memo |
| 다음 점검일 / 갱신 horizon | 일정 누락 방지 | IP + 재무 | 갱신 준비표 |
| 일본어 표기 drift 여부 | 등록 포트폴리오와 실제 사용 연결 | 브랜드팀 | evidence hygiene 패킷 |

### 등록 직후 챙겨야 할 것

등록증 확보, 내부 포트폴리오 등록, 지정 범위 요약, 현지 사용 계획 연결, 대리인 연락처 정리를 한 번에 해둔다. 등록 직후 이 다섯 가지가 정리되어 있지 않으면 몇 달 뒤 “권리는 있는데 아무도 내용을 모르는 상태”가 된다. 특히 일본 사업팀이 실제로 어떤 표기를 쓸지, 어느 SKU와 서비스에 연결되는지도 함께 기록해야 한다.

또한 등록된 표장의 일본어 표기와 영문 표기가 실제 판매물에 어떻게 붙는지도 확인해야 한다. 등록이 끝났다고 자동으로 사용 통제가 되는 것은 아니므로, 마케팅 자산과 판매 페이지가 등록 포트폴리오와 같은 방향으로 움직이는지 초기에 맞춰 두는 편이 좋다.

### 갱신 달력과 분할 납부 판단

존속기간 10년, 갱신 가능, 일부 수수료 분할 구조 가능이라는 큰 틀은 [JPO FAQ](https://www.jpo.go.jp/e/faq/yokuaru/trademark.html)와 fee schedule에서 확인할 수 있다. 실제 운용에서는 각 class의 사업 중요도와 예산 주기를 함께 보며 갱신 결정을 준비한다. 일본 포트폴리오가 커질수록 “언제 갱신인가”보다 “어떤 자산을 유지할 것인가”가 더 중요한 질문이 된다.

실무적으로는 갱신 달력을 재무 캘린더와 연결하는 편이 좋다. 법무/IP팀이 혼자 기억하는 방식은 인사 이동이나 조직 개편에 매우 약하다. 최소한 분기 단위 점검표에 일본 상표 갱신 예정 건과 사업 중요도를 같이 넣어 두어야 예산과 권리 유지가 따로 놀지 않는다.

### 포트폴리오 정합성 유지

등록 목록이 실제 시장 사용과 어긋나지 않는지 분기별로 확인한다. 브랜드가 바뀌었는데 포트폴리오가 그대로면 분쟁 대응과 거래 문서가 꼬이기 쉽다. 특히 일본 시장에서는 병행 표기, 한정 SKU, 한시적 캠페인 이름이 자주 생길 수 있으므로, 실제 사용 현황과 등록 포트폴리오를 주기적으로 대조해야 한다.

또한 조직 재편이나 법인 구조 변화가 있을 때는 상표 포트폴리오도 같이 보아야 한다. 명의 변경이 늦거나 실제 사용 주체와 권리자 관계가 흐려지면, 나중에 양도·라이선스·집행 단계에서 설명 비용이 크게 늘어난다.

#### use evidence를 놓치지 않는 저장 루틴

등록 이후에는 사용증거를 꾸준히 남기는 습관이 중요하다. 패키지, 상세페이지, 판매 캡처, 캠페인 자산, 일본어 표기 노출 화면을 분기별로 저장해 두면 나중에 non-use나 분쟁 이슈가 생겨도 대응이 훨씬 빠르다. 유지관리의 상당 부분은 사실상 evidence hygiene다.

증거 저장은 법무팀만의 일이 아니라 사업팀과 마케팅팀의 반복 업무여야 한다. 운영 루틴으로 내려오지 않으면 몇 년 뒤 필요한 자료가 비어 있게 된다.

#### 유지관리용 최소 증거 패킷

일본 lane에서 evidence hygiene를 실제 운영 기준으로 바꾸려면, 최소 패킷 단위를 더 분명히 두는 편이 좋다. 화면 캡처만 있거나 패키지 사진만 있으면 설명이 반쪽짜리가 되기 쉽다.

| 항목 | 최소 예시 | 빠지면 생기는 문제 |
|---|---|---|
| 화면 | 자사몰, marketplace, 일본어 상세페이지, 앱 화면 | 실제 소비자 노출 문맥이 약해짐 |
| 거래 문맥 | 주문 확인, 출하 기록, 유통 문서, 판매 채널 메모 | 실제 상업적 사용 설명이 약해짐 |
| 표기 버전 | 영문 표장, 일본어 표기, 패키지·라벨 버전 | 표기 drift와 등록 포트폴리오 연결이 흔들림 |
| owner-user memo | 권리자, 실제 사용자, 총판·라이선스 구조 | 누가 어떤 자격으로 사용했는지 설명 비용이 커짐 |
| 날짜 | 캡처일, 촬영일, 집행 기간, 최근 업데이트일 | 사용 시점과 갱신 준비의 기준점이 흐려짐 |

일본에서는 특히 영문 표장과 일본어 표기가 따로 굴러가기 쉬우므로, 두 버전을 같은 패킷 안에서 연결해 두는 편이 안전하다. 총판이나 대리점이 별도 표기를 만들기 시작한 경우에는 그 흔적도 별도 메모가 아니라 같은 패킷 안에 넣어야 한다.

#### owner 변경이 없어도 점검해야 할 것

법인 명의가 그대로여도 실제 owner-user 관계는 흔들릴 수 있다. 일본 판매 구조가 바뀌거나 총판 역할이 커지면, 권리자와 실제 사용자의 간격이 넓어지고 설명 비용이 커진다. 그래서 명의 변경이 없더라도 owner-user memo는 주기적으로 다시 읽어야 한다.

이 점검은 겉으로는 작아 보여도 분쟁 대응과 갱신 준비의 품질을 크게 좌우한다.

### 이 장의 실무 체크리스트

- 등록 직후 권리 요약표, 담당자, 사용 계획, 대리인 연락처를 한 번에 정리한다.
- 일본 상표 갱신 예정 건을 예산 캘린더와 연결한다.
- 분기별로 실제 사용 표기와 등록 포트폴리오의 정합성을 점검한다.
- 조직 개편, 법인 변경, 총판 구조 변경이 있으면 상표 포트폴리오도 같이 점검한다.

| 거래 문맥과 화면이 같은 패킷 안에 묶여 있는가 | 실제 사용 설명과 분쟁 대응이 쉬워짐 |

## 분기 운영 quick triage

lighter track에서는 완벽한 포트폴리오 리뷰보다 “정상 / 보완 / 공백”을 빠르게 나누는 편이 더 유효하다.

| 상태 | 의미 | 기본 액션 |
|---|---|---|
| 정상 | 최근 캡처, owner-user memo, 점검일이 모두 최신 | 현 상태 유지 |
| 보완 | 자료는 있으나 표기 drift 또는 거래 문맥이 약함 | 다음 분기 전 linkage 보강 |
| 공백 | 최근 사용 자료나 owner memo가 비어 있음 | 즉시 evidence packet 재구성 |

이 triage 표가 있으면 JapTm은 standard QA 유지와 reader utility 점검이라는 현재 lane 목적에 더 잘 맞게 작동한다.
