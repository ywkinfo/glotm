# 제5장 출원서 작성 실무: 출원인 구조·대리인·전자출원 준비

출원서 작성 단계에서는 법률 지식보다 정보 정합성이 더 중요하다. 출원인, 주소, 표장 버전, 지정상품, 제출 채널이 서로 어긋나지 않아야 이후 보정 비용이 줄어든다. 일본 절차가 일본어 중심으로 돌아간다는 점도 미리 반영해야 한다. 준비가 느슨하면 실제 제출 시점에 문제가 터지고, 그때는 수정 비용이 훨씬 커진다.

이 장의 실무 포인트는 “무엇을 써 넣는가”보다 “누가 어떤 자료를 어느 순서로 준비하는가”에 있다. 출원 직전이 아니라 네이밍 확정 단계에서부터 자료를 모으고, 출원 전 최종 체크에서 한 번 더 맞추는 구조를 만들어야 한다. 결국 좋은 출원서는 좋은 운영표에서 나온다.

### 출원인과 실제 사용자 정리

한국 본사, 일본 법인, 총판, 라이선시 가운데 누가 권리자이고 누가 사용자일지 먼저 정리해야 한다. 출원은 본사 명의로 하되 일본 시장 사용은 현지 파트너가 하는 구조라면, 이후 계약과 집행에서 어떤 문서를 제시할지까지 함께 설계해야 한다. 출원인 명의와 실제 사용 주체의 관계를 나중에 설명하려 하면 대부분 늦다.

또한 내부 결재 구조도 같이 확인해야 한다. 표장 자체는 브랜드팀이 들고 있고, 출원 예산은 법무/IP가 들고 있고, 일본 사업팀이 출시 일정을 밀고 있다면, 누가 최종본을 승인하는지 모르는 상태로는 출원서가 자주 흔들린다. 출원인 정보와 실사용 정보는 단순 입력값이 아니라 내부 책임 구조를 보여 주는 데이터다.

### 지정상품 문구와 제출 자료 준비

표장 파일, 지정상품 초안, 내부 승인자, 일본 내 대리인 커뮤니케이션 채널을 한 묶음으로 관리한다. 특히 지정상품·서비스 문구는 사업팀과 IP팀이 서로 다른 언어로 작성하기 쉬우므로, “실제 팔 것”, “가까운 확장 예정”, “당장 안 하는 것”을 나눠 정리하는 편이 좋다. 이 정리가 없으면 보정 단계에서 줄였다 늘렸다를 반복하게 된다.

또한 실무자는 외부 대리인에게 “알아서 일본어로 정리해 달라”고만 넘기지 않는 편이 좋다. 어떤 채널에서 어떤 제품을 누구 명의로 팔지, 일본에서 어떤 표기를 쓸지, 론칭 시점은 언제인지까지 적어 줘야 지정 범위와 설명 방향이 더 빨리 정리된다. 잘 정리된 브리프는 대리인 비용도 줄여 준다.

### 전자출원과 비용 체크

[JPO step-by-step guide](https://www.jpo.go.jp/e/system/professionals/step-by-step-trademark.html)와 [JPO fee schedule](https://www.jpo.go.jp/e/system/process/tesuryo/hyou.html)을 기준으로 제출 경로와 class 구조를 확인한다. 정확 금액은 바뀔 수 있으므로 본문에서는 수치보다 `class별 정부수수료 구조`와 `비거주자의 직접 납부 제약`을 기억하는 것이 더 안전하다.

실무적으로는 비용표보다 결재 루틴이 더 중요하다. class 수가 늘어날수록 정부수수료뿐 아니라 외부 대리인 검토량도 늘어나므로, 출원 전 단계에서 “우선 출원 범위”와 “후속 확장 범위”를 나눠 두면 예산 승인과 속도 관리가 쉬워진다.

#### filing 직전 마지막 owner check

출원 직전에는 법인명, 주소, 실제 사용자, 일본 시장 판매 구조를 한 번 더 대조하는 편이 좋다. 이 값들이 초안 단계와 달라지는 경우가 의외로 많고, 제출 이후에 발견되면 보정과 설명 비용이 커진다. owner check를 checklist로 고정하면 이런 실수를 줄일 수 있다.

특히 launch 직전 조직 변경이나 총판 구조 변경이 있었다면, 출원 브리프가 여전히 최신인지 재확인해야 한다. filing packet의 품질은 마지막 1회 재확인에서 크게 갈린다.

#### 표장 버전 drift를 잡는 법

출원 직전에는 로고 파일, 워드마크, 일본어 표기, 패키지 시안이 서로 다른 버전을 참조하는 drift가 자주 생긴다. 이때는 어느 파일이 최신인지 묻기보다, “실제 일본 시장에서 먼저 노출될 버전이 무엇인가”를 기준으로 잠그는 편이 좋다.

표장 버전 drift를 그대로 두면 검색 메모, 출원 서류, 판매 자산이 서로 다른 표장을 가리키게 된다. 이 문제는 filing 이후보다 filing 직전에 잡는 것이 훨씬 싸다.

### 이 장의 실무 체크리스트

- 출원인, 실제 사용자, 결재권자, 일본 현지 연락 창구를 한 표로 정리한다.
- 표장 파일, 일본어 표기, 지정상품 초안, 출시 일정, 승인 상태를 하나의 브리프로 묶는다.
- 비거주자 실무 제약과 수수료 구조는 공식 페이지 기준으로만 확인하고, 금액은 출판 직전 재검증 원칙을 유지한다.
- 외부 대리인에게 넘길 때는 “문구 초안”이 아니라 “운영 구조 메모”까지 같이 넘긴다.

## 출원 브리프 표준안

일본 출원은 브리프를 잘 만들면 절반은 끝난다.

| 묶음 | 포함 항목 |
| --- | --- |
| owner pack | 출원인, 실제 사용자, 승인권자 |
| mark pack | 영문, 일본어 표기, 로고 버전 |
| goods/services pack | 현재 판매, 가까운 확장, 보류 범위 |
| filing pack | 대리인, 제출 채널, 예산 승인 상태 |

## 대리인 handoff 직전 red flag

| red flag | 기본 대응 |
| --- | --- |
| 일본어 표기가 아직 흔들림 | 네이밍과 출원 범위 동시 재점검 |
| 실제 사용자와 출원인이 설명되지 않음 | owner-user memo 보강 |
| 지정상품이 출시 계획과 연결되지 않음 | core/adjacent 재구분 |

## filing day packet owner

| packet | owner |
| --- | --- |
| final mark set | 브랜드팀 |
| owner-user memo | 법무/IP |
| launch timing note | 일본 사업팀 |
| agent submission bundle | 외부 대리인 + 내부 owner |
